## What Have I Learned?

I learned that a tag is a fixed label on a commit (for releases like `v1.0`).

`git tag v1.0` will work fine.

If you want annotated which is generally preferred for releases use `git tag -a v1.0.1 -m "..."`.
`-a` stands for `annotated`.

Tags cannot push with `git push`, you must use `git push origin v1.0`.

Also I learned that push order does not reorder history. Tag push uploads commits which have not pushed before it.

And `git diff` shows what changed line by line before you commit.
You can look at the difference between two branches `git diff main feature`.
You can look staged edits `git diff --staged`.
