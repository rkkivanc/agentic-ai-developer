## What Have I Learned?

I learned that `git reset <file>` (or `git restore --staged <file>`, new way) can unstage the files. Edits will stay, just not be staged.

`git reset --hard HEAD~1` moves branch back one commit and discard those changes from the branch and log. It is ok for local and unshared changes.

`git revert` creates new commit that undoes an old one. 
It is more preferred and accurate to use if commit is already on GitHub and shared. 
write `--no-edit` to skip writing revert commit message. It will be something like `Revert "bla bla"`. Bla bla stands for your old commit message :).

If you want to erase from GitHub history, after you reset locally you can use `git push --force-with-lease`. It makes GitHub's branch match with yours, even if history changed.

`git push --force-with-lease` is safer than `git push --force` because it refuses if someone else pushed new commits that you don't have yet.
