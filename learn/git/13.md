## What Have I Learned?

I learned that `git rebase -i` is for interactive history cleanup.

`squash` combines several commits into one.
`reword` changes a commit message.

You should not do this on shared main that others already pulled. Do this on your local.

Stuck rebase?? => `git rebase --abort`, you will be fine.

`:wq` to save and quit.

if you use git rebase without -i like `git rebase main`, it makes (it is more clear with picture):

Git takes your branch’s commits and replays them on top of main.

main itself usually doesn’t move
your current branch moves forward, now based on latest main

Picture:
```
Before:  main:     A---B
         feature:  A---C---D
After:   main:     A---B
         feature:  A---B---C'---D'
```

Your work (C, D) is rebuilt on top of B.
To put that into main, you still merge or open a PR later — rebase alone doesn’t send commits onto main.
