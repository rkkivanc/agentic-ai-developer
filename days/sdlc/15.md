# Day 15: Operate, Improve & Capstone - Retrospectives and Continuous Improvement

## 📝 Today's Tasks

1.  **Retro Format:** Run a personal retro on a recent project (Went well / Improve / Actions).
2.  **Actionable Outcomes:** Convert 2 complaints into specific improvement actions with owners/dates (even if owner is you).
3.  **Measure Once:** Pick one process metric (cycle time, escaped bugs) to watch next month.
4.  **Feedback Loops:** Map how user feedback re-enters requirements.

## 📖 Task Explanations

Retrospectives close the SDLC loop: learn → adjust → build better. Continuous improvement needs actions and feedback into requirements—not only venting.

## 💡 What You'll Gain

You will run a simple retrospective with actionable outcomes and connect feedback to the next requirements cycle.

## 📚 Dictionary for Developers

*   **Retrospective:** A meeting/practice to inspect process and improve.
*   **Continuous Improvement:** Ongoing small changes that raise quality and speed.
*   **Cycle Time:** How long work takes from start to done/release.
*   **Escaped Defect:** A bug found in production that testing missed.
