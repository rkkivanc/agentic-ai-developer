# Day 11: Build, Quality & Release - Testing in the Life Cycle

## 📝 Today's Tasks

1.  **Testing Levels:** Explain unit, integration, and end-to-end tests with one example each.
2.  **Map to Stories:** For a user story, list which tests you'd write at each level.
3.  **Bug Life Cycle:** Outline steps from bug report → triage → fix → verify → close.
4.  **Quality != Only QA:** Note developer responsibilities for quality before handoff.

## 📖 Task Explanations

Testing is a continuous life-cycle activity, not a final surprise phase. Different levels catch different failures. Everyone owns quality—QA deepens it, developers don't outsource it entirely.

## 💡 What You'll Gain

You will map testing levels to stories and describe a healthy bug life cycle.

## 📚 Dictionary for Developers

*   **Unit Test:** Tests a small unit of code in isolation.
*   **Integration Test:** Tests how components work together.
*   **E2E Test:** Tests a full user flow through the system.
*   **Triage:** Assessing severity/priority of a bug or issue.
