# Day 10: Build, Quality & Release - Code Review and Collaboration

## 📝 Today's Tasks

1.  **Review Checklist:** Create a 8–10 item code review checklist (correctness, tests, naming, security).
2.  **Give Feedback:** Practice writing one kind comment and one necessary critical comment on a sample diff.
3.  **Receive Feedback:** Write how you'd respond to a review you disagree with professionally.
4.  **PR Hygiene:** List what belongs in a good pull request description.

## 📖 Task Explanations

Code review is a quality and knowledge-sharing gate in the life cycle. Good reviews focus on risk and clarity; good responses stay curious and professional.

## 💡 What You'll Gain

You will use a practical review checklist and practice constructive review communication.

## 📚 Dictionary for Developers

*   **Code Review:** Peer examination of changes before merge.
*   **Pull Request (PR):** A proposed change set for review and merge.
*   **Knowledge Sharing:** Spreading system understanding across the team via reviews and docs.
*   **Blocking Issue:** A review finding that must be fixed before merge.
