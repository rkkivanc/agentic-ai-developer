# Day 12: Build, Quality & Release - Release Readiness and CI Basics

## 📝 Today's Tasks

1.  **Release Checklist:** Draft a go-live checklist (migrations, Feature flags, rollback, smoke test).
2.  **CI Purpose:** Explain how continuous integration supports SDLC quality gates.
3.  **Rollback Plan:** Write a simple rollback/mitigation plan for a bad release.
4.  **Version Note:** Practice writing a short changelog entry for a sample release.

## 📖 Task Explanations

Shipping is a deliberate phase: readiness checks, automation, and rollback thinking. CI enforces repeatable checks so humans focus on judgment, not forgotten steps.

## 💡 What You'll Gain

You will produce a release checklist, explain CI's role, and draft a basic rollback and changelog approach.

## 📚 Dictionary for Developers

*   **Release:** Delivering a version of software to users/environments.
*   **CI (Continuous Integration):** Automatically building/testing changes as they merge.
*   **Rollback:** Returning to a previous good version after a bad release.
*   **Smoke Test:** A quick check of critical paths after deploy.
*   **Changelog:** A human-readable list of noteworthy changes in a release.
