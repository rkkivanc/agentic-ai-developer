# Day 6: Requirements & Design - User Stories and Acceptance Criteria

## 📝 Today's Tasks

1.  **Write User Stories:** Create 5 stories in 'As a… I want… So that…' form.
2.  **Add Acceptance Criteria:** For 2 stories, write testable acceptance criteria (Given/When/Then or bullets).
3.  **Split Large Stories:** Take an epic-sized idea and break it into smaller shippable stories.
4.  **Priority Pass:** Order stories by user value and risk.

## 📖 Task Explanations

User stories capture intent from a user perspective. Acceptance criteria make 'done' testable. Small stories reduce risk and improve feedback speed—central to modern SDLC practice.

## 💡 What You'll Gain

You will write clear user stories with acceptance criteria and split large work into smaller slices.

## 📚 Dictionary for Developers

*   **User Story:** A short requirement from a user's point of view.
*   **Acceptance Criteria:** Conditions that must be true for a story to be accepted as done.
*   **Epic:** A large body of work that must be broken into smaller stories.
*   **Prioritization:** Ordering work by value, risk, urgency, or dependencies.
