# Day 9: Build, Quality & Release - Implementation Discipline

## 📝 Today's Tasks

1.  **Slice Work:** Break one user story into implementation tasks (API, UI, tests, docs).
2.  **Branch Mentality:** Describe a safe workflow for coding the story (branch, small commits).
3.  **Definition of Done Coding:** List coding DoD items (tests, lint, reviewed, documented).
4.  **Spike vs Build:** Decide when you'd spike (research) before committing to an approach.

## 📖 Task Explanations

Implementation is only one SDLC phase—but it benefits from discipline: small slices, clear DoD, and knowing when to research first. This keeps delivery predictable.

## 💡 What You'll Gain

You will plan implementation as task slices with a coding Definition of Done and a sense for spikes.

## 📚 Dictionary for Developers

*   **Implementation:** The phase of writing and integrating working code.
*   **Spike:** A timeboxed research task to reduce uncertainty.
*   **Definition of Done (DoD):** Checklist that defines when implementation work is complete.
*   **Vertical Slice:** A thin end-to-end piece of functionality across layers.
