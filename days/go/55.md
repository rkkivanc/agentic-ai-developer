# Day 55: Auth & Security - Practice

## 📝 Today's Tasks

1.  **Secure Your MVP:** Add registration, login, JWT auth, and role checks to your API.
2.  **Test Auth Flows:** Write tests for success, wrong password, missing token, and forbidden roles.
3.  **Document Auth:** Explain how clients obtain and send tokens in your README.
4.  **Review Threats:** List remaining risks honestly, such as token revocation or CSRF scope.

## 📖 Task Explanations

Today secures the MVP end-to-end. Tests for auth flows prevent regressions that are embarrassing and dangerous.

Document token usage clearly for API consumers. Honest threat review shows maturity—note what you have not solved yet.

Auth is never done, but this is a strong baseline.

## 💡 What You'll Gain

By the end of today, your MVP will have tested authentication, authorization, and documented client usage.

## 📚 Dictionary for Developers

*   **Threat Model:** A structured view of assets, attackers, and mitigations for a system.
*   **Auth Regression:** A bug re-breaking login or permission checks after changes.
*   **Bearer Token:** An HTTP Authorization scheme sending tokens in the `Authorization` header.
*   **Security Baseline:** Minimum practices expected before exposing a service publicly.
*   **CSRF:** Cross-Site Request Forgery, mainly relevant for cookie-based browser flows.
