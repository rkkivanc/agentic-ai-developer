# Day 85: Caching & Messaging - Practice

## 📝 Today's Tasks

1.  **Add Caching Layer:** Cache a read endpoint and invalidate on updates.
2.  **Publish Domain Events:** Emit events on create/update and process them in a worker.
3.  **Test Idempotency:** Deliver the same event twice and verify correct outcome.
4.  **Document Async Flow:** Diagram HTTP, DB, cache, and messaging interactions.

## 📖 Task Explanations

Today extends your MVP with async and cache paths—real systems rarely are CRUD-only. Verify idempotency explicitly.

Diagrams help teammates reason about race windows and failure modes. Keep workers small and focused.

Async introduces complexity—earn it with measurable benefit.

## 💡 What You'll Gain

By the end of today, your MVP will use caching and messaging with documented, idempotent async flows.

## 📚 Dictionary for Developers

*   **Async Flow:** Processing that continues outside the originating HTTP request lifecycle.
*   **Race Window:** A brief period where cache and database may disagree during updates.
*   **Worker Process:** A separate runnable binary consuming queue messages.
*   **System Diagram:** A visual map of components and data flows between them.
*   **End-to-End Idempotency:** Safety when the same logical operation or message appears multiple times.
