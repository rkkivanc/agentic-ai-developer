# Day 31: First REST API Project (MVP) - Project Setup and API Design

## 📝 Today's Tasks

1.  **Scaffold the Project:** Create module layout with `cmd/api` and `internal/` packages.
2.  **Define Resources:** Choose an MVP domain such as bookmarks or tasks and sketch endpoints.
3.  **Model Types:** Define request/response structs with JSON tags and validation rules.
4.  **Plan Errors:** Decide on a consistent JSON error format and status code mapping.

## 📖 Task Explanations

Your first REST MVP needs a clear layout before code accumulates. Separating `cmd` entrypoints from `internal` packages keeps dependencies directed and readable.

Spend time on API design: resources, methods, and response shapes. Cheap changes now prevent client-breaking churn later.

Error format decisions matter equally—clients depend on stable failure responses.

## 💡 What You'll Gain

By the end of today, you will scaffold an MVP REST project with domain models and a documented endpoint plan.

## 📚 Dictionary for Developers

*   **cmd/:** A conventional directory housing main packages that build executables.
*   **internal/:** A directory for packages not meant to be imported by external modules.
*   **DTO (Data Transfer Object):** A struct shaped for API requests or responses rather than database rows.
*   **Status Code Mapping:** Consistent rules translating domain failures into HTTP status codes.
*   **MVP:** The smallest useful version of a product that demonstrates core value.
