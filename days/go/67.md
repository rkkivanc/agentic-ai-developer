# Day 67: Advanced Testing & Quality - Test Fixtures and Containers

## 📝 Today's Tasks

1.  **Use testcontainers-go Optionally:** Start database containers for realistic tests.
2.  **Abstract Setup:** Centralize container or database startup in test helpers.
3.  **Parallel Caution:** Isolate resources when running integration tests in parallel.
4.  **Teardown Reliably:** Stop containers and drop schemas even when tests fail.

## 📖 Task Explanations

Containers make integration tests resemble production without shared dev databases. `testcontainers-go` is popular though optional—manual test DBs work too.

Central setup/teardown helpers prevent copy-paste drift. Always teardown, even on failures, or CI machines accumulate junk.

Parallelism requires unique ports or schemas—plan for it.

## 💡 What You'll Gain

By the end of today, you will manage repeatable integration test environments with reliable setup and teardown.

## 📚 Dictionary for Developers

*   **testcontainers:** A library starting real services in Docker containers during tests.
*   **Teardown:** Cleanup logic run after tests, releasing external resources.
*   **Ephemeral Database:** A short-lived database instance created per test run.
*   **Port Collision:** Failure when parallel tests attempt to bind the same port.
*   **CI Runner:** The machine executing tests in continuous integration.
