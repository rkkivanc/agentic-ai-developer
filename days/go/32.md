# Day 32: First REST API Project (MVP) - CRUD Endpoints

## 📝 Today's Tasks

1.  **Implement Create:** Add `POST /items` decoding JSON bodies into new records.
2.  **Implement Read:** Add `GET /items` and `GET /items/{id}` with proper 404 handling.
3.  **Implement Update:** Add `PUT` or `PATCH` endpoints updating existing records.
4.  **Implement Delete:** Add `DELETE /items/{id}` and return appropriate status codes.

## 📖 Task Explanations

CRUD endpoints are the skeleton of most services. Implement them in order, testing each before moving on.

Use explicit status codes: 201 on create, 404 when IDs are missing, 204 or 200 on delete depending on your contract. Keep handlers thin by delegating to a store layer.

Thin handlers plus tested storage functions mirror how production codebases stay maintainable.

## 💡 What You'll Gain

By the end of today, you will expose full CRUD operations for your MVP resource over HTTP.

## 📚 Dictionary for Developers

*   **201 Created:** HTTP status indicating a resource was successfully created.
*   **404 Not Found:** HTTP status when a requested resource identifier does not exist.
*   **PATCH:** An HTTP method for partial updates compared to full replacement with PUT.
*   **Handler Layer:** HTTP-specific code translating requests into domain operations.
*   **Idempotent:** An operation that produces the same result when applied multiple times, like DELETE.
