# Day 83: Caching & Messaging - Working with NATS or Kafka Basics

## 📝 Today's Tasks

1.  **Publish Events:** Emit a domain event when a resource is created or updated.
2.  **Subscribe Handlers:** Process events in a separate worker process.
3.  **Handle Failures:** Nack or retry failed messages with limits.
4.  **Observe Lag:** Log consumer lag or pending message counts.

## 📖 Task Explanations

Hands-on messaging makes concepts concrete. Publish events from your API; handle them in workers that can scale independently.

Plan failure handling—unbounded retries poison systems. Monitor lag so you know when consumers fall behind.

Workers can be separate Go binaries in `cmd/`.

## 💡 What You'll Gain

By the end of today, you will publish and consume domain events with a real message broker.

## 📚 Dictionary for Developers

*   **Domain Event:** A message announcing something happened in the business domain.
*   **Worker:** A background process handling tasks outside the request path.
*   **Consumer Lag:** How far behind consumers are relative to producers.
*   **Nack:** Negative acknowledgment indicating a message should be retried or dead-lettered.
*   **Topic:** A named channel to which producers publish and from which consumers subscribe.
