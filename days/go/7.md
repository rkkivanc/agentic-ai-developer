# Day 7: Structs, Methods & Interfaces - Methods and Receivers

## 📝 Today's Tasks

1.  **Write Value Receivers:** Define methods with value receivers and observe copy semantics.
2.  **Write Pointer Receivers:** Define methods with pointer receivers to mutate struct state.
3.  **Choose Receiver Type:** Practice deciding when pointer receivers are necessary for consistency.
4.  **Method Sets:** Summarize which methods are available on values versus pointers.

## 📖 Task Explanations

Methods are functions with a receiver argument. They let types carry behavior alongside data, which is how Go achieves object-like patterns without inheritance.

Value receivers copy the struct; pointer receivers can modify it and avoid copying large values. The Go community strongly prefers consistent receiver choices within a type.

Understanding method sets also prepares you for interfaces: the set of methods attached to a type determines which interfaces it satisfies.

## 💡 What You'll Gain

By the end of today, you will be able to attach methods to types, choose appropriate receivers, and explain how method sets work.

## 📚 Dictionary for Developers

*   **Method:** A function with a receiver parameter that binds behavior to a type.
*   **Receiver:** The parameter before a method name that indicates which type the method belongs to.
*   **Value Receiver:** A receiver that operates on a copy of the value.
*   **Pointer Receiver:** A receiver that operates on a pointer, allowing mutation and avoiding copies.
*   **Method Set:** The collection of methods available on a type, used to determine interface satisfaction.
