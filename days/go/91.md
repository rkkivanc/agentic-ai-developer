# Day 91: Team Practices & Tooling - Code Review and Git Workflow

## 📝 Today's Tasks

1.  **Trunk-Based Habits:** Use short-lived branches and small pull requests.
2.  **Write Good PR Descriptions:** Explain why, test plan, and risks in each PR.
3.  **Review Checklist:** Look for correctness, tests, security, and readability.
4.  **Respond Constructively:** Give and receive feedback focused on code, not people.

## 📖 Task Explanations

Professional Go work happens in teams. Small PRs review faster and ship safer. Descriptions should explain motivation and how you validated changes.

Review checklists catch missing tests and security issues. Constructive feedback builds trust and better systems.

Git workflow is a skill—practice it daily.

## 💡 What You'll Gain

By the end of today, you will use pull requests and reviews with clear descriptions and constructive feedback.

## 📚 Dictionary for Developers

*   **Pull Request:** A proposed change set reviewed before merging to main.
*   **Trunk-Based Development:** Frequent integration to main with small branches.
*   **Test Plan:** Steps reviewers or QA can follow to verify a change.
*   **Nit:** A minor style comment, optional and non-blocking in reviews.
*   **Constructive Feedback:** Comments aimed at improving code while respecting authors.
