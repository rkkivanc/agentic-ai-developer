# Day 29: HTTP Basics & Handlers - Middleware Patterns

## 📝 Today's Tasks

1.  **Wrap Handlers:** Write middleware functions that accept and return `http.Handler`.
2.  **Log Requests:** Add logging middleware recording method, path, status, and duration.
3.  **Recover Panics:** Catch panics in handlers and return 500 responses instead of crashing.
4.  **Chain Middleware:** Compose logging, recovery, and auth-placeholder middleware cleanly.

## 📖 Task Explanations

Middleware cross-cuts concerns like logging, metrics, authentication, and recovery. In Go, middleware is often a function wrapping `http.Handler`.

You'll build small wrappers and chain them explicitly—no magic. Recovery middleware keeps one bad request from taking down the whole process.

This handler-wrapping style reappears with `context`, configuration, and security later.

## 💡 What You'll Gain

By the end of today, you will implement and chain HTTP middleware for logging, recovery, and cross-cutting concerns.

## 📚 Dictionary for Developers

*   **Middleware:** A layer wrapping handlers to add shared behavior before or after request processing.
*   **http.Handler:** An interface with `ServeHTTP`, the building block for handlers and middleware chains.
*   **Panic:** A runtime error that stops normal execution unless recovered.
*   **Recovery:** Middleware catching panics and converting them into controlled HTTP 500 responses.
*   **Cross-cutting Concern:** Behavior affecting many handlers, such as logging or authentication.
