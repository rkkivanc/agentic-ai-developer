# Day 80: Containers & CI/CD - Practice

## 📝 Today's Tasks

1.  **Dockerize MVP:** Build and run your full MVP stack in containers.
2.  **Complete CI Pipeline:** Test, lint, vet, build, and optional integration in GitHub Actions.
3.  **Document Deploy Steps:** Write how to build image, run container, and required env vars.
4.  **Tag Release:** Create a version tag and note artifacts produced by CI.

## 📖 Task Explanations

Today delivers deployable artifacts: container image plus green CI. Document env vars and ports for operators.

Tag a release—even `v0.1.0`—to practice versioning discipline. CI should produce the same binary Docker runs.

You now have a path from laptop to running container.

## 💡 What You'll Gain

By the end of today, your MVP will build in CI, run in Docker, and ship with documented deploy steps.

## 📚 Dictionary for Developers

*   **Deployable Artifact:** A container image or binary ready to run in a target environment.
*   **Version Tag:** A git tag marking a release point such as `v1.0.0`.
*   **Environment Parity:** Keeping dev, CI, and production as similar as practical.
*   **Operator Docs:** Instructions for people running the service, not only developing it.
*   **Release Candidate:** A build considered potentially ready after passing CI checks.
