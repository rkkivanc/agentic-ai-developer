# Day 43: Databases (I) - Queries and Prepared Statements

## 📝 Today's Tasks

1.  **Use Prepared Statements:** Prepare repeated queries for performance and clarity.
2.  **Insert Rows:** Insert records with parameterized queries—never string-concatenate SQL.
3.  **Handle sql.ErrNoRows:** Translate missing rows into domain-level not-found errors.
4.  **Log Query Errors:** Wrap DB errors with context while avoiding logging secrets.

## 📖 Task Explanations

Parameterized queries prevent SQL injection and help the database cache plans. `sql.ErrNoRows` is not a catastrophe—it means your query succeeded but found nothing.

Map `sql.ErrNoRows` to domain errors handlers understand. Wrap low-level errors with operation context (`insert user`, `select order`) for debugging.

Never interpolate user input into SQL strings.

## 💡 What You'll Gain

By the end of today, you will run safe parameterized queries and handle empty result sets idiomatically.

## 📚 Dictionary for Developers

*   **Prepared Statement:** A precompiled SQL statement reused with different parameters.
*   **SQL Injection:** An attack embedding malicious SQL via unescaped user input.
*   **Parameter Binding:** Supplying user values separately from SQL text so they are treated as data.
*   **sql.ErrNoRows:** An error returned by `Scan` when a query returns no rows.
*   **Query Plan:** How the database executes a SQL statement, affected by indexes and parameters.
