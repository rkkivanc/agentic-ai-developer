# Day 53: Auth & Security - Authorization and RBAC

## 📝 Today's Tasks

1.  **Define Roles:** Model roles like `admin` and `member` in your domain.
2.  **Authorize Actions:** Check permissions before destructive or sensitive operations.
3.  **Return 403:** Use Forbidden when authenticated users lack permission.
4.  **Centralize Policy:** Write helper middleware or functions for authorization checks.

## 📖 Task Explanations

Authorization decides what an authenticated user may do. Role-based access control maps users to roles with permissions.

Return 403 when identity is known but action is not allowed—distinct from 401. Central policy helpers prevent copy-pasted checks drifting apart.

Tests should cover allowed and denied roles explicitly.

## 💡 What You'll Gain

By the end of today, you will enforce role-based authorization on sensitive endpoints.

## 📚 Dictionary for Developers

*   **Authorization:** Determining whether an authenticated principal may perform an action.
*   **RBAC:** Role-Based Access Control assigning permissions via roles.
*   **403 Forbidden:** HTTP status when identity is valid but permission is denied.
*   **Policy:** Rules defining who can perform which operations on which resources.
*   **Principal:** The authenticated user or service making a request.
