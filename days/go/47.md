# Day 47: Databases (II) & Repositories - Query Organization

## 📝 Today's Tasks

1.  **Centralize SQL:** Keep SQL strings in one file or use a query constant pattern.
2.  **Explore sqlc Optionally:** Generate type-safe Go from SQL if you want stronger compile-time checks.
3.  **Name Queries Clearly:** Use descriptive names matching business operations.
4.  **Avoid N+1 Queries:** Fetch related data efficiently with joins or batch queries.

## 📖 Task Explanations

Scattered SQL becomes unmaintainable. Centralize queries so reviewers can audit data access quickly.

Tools like `sqlc` generate Go from SQL—optional but popular on Go teams. Whether handwritten or generated, name queries after operations: `GetUserByID`, not `query1`.

Watch for N+1 patterns when listing related data; they are a common performance footgun.

## 💡 What You'll Gain

By the end of today, you will organize SQL clearly and recognize basic query performance pitfalls.

## 📚 Dictionary for Developers

*   **sqlc:** A tool generating Go code from SQL queries for type-safe data access.
*   **N+1 Query Problem:** Executing one query per item in a list instead of a single efficient query.
*   **Join:** A SQL operation combining rows from multiple tables based on related keys.
*   **Query Constant:** A named constant holding SQL text, grouped for discoverability.
*   **Data Access Layer:** The part of the codebase responsible for reading and writing persistent data.
