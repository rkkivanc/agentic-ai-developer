# Day 37: Context, Config & Middleware - Environment Config

## 📝 Today's Tasks

1.  **Read env Vars:** Load configuration from environment variables with sensible defaults.
2.  **Support .env Locally:** Use a simple loader for local development without committing secrets.
3.  **Validate on Startup:** Fail fast when required settings like `PORT` or `DATABASE_URL` are missing.
4.  **Struct Config:** Map environment variables into a typed `Config` struct.

## 📖 Task Explanations

Configurable services run in dev, staging, and production without code changes. Environment variables are the default contract for twelve-factor style apps.

Validate configuration at startup—crashing early with a clear message beats mysterious runtime failures. Typed config structs beat scattered `os.Getenv` calls.

Never commit secrets; local `.env` files stay out of version control.

## 💡 What You'll Gain

By the end of today, you will load typed configuration from the environment and validate it on startup.

## 📚 Dictionary for Developers

*   **Environment Variable:** A key/value configuration provided by the OS or container runtime.
*   **Fail Fast:** Exiting immediately on startup misconfiguration with a clear error.
*   **Secret:** Sensitive data like API keys or passwords, never committed to source control.
*   **Default Value:** A fallback used when an optional setting is not provided.
*   **Twelve-Factor App:** A methodology encouraging config via environment, not code.
