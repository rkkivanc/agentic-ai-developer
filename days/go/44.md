# Day 44: Databases (I) - Transactions

## 📝 Today's Tasks

1.  **Begin Transactions:** Start transactions with `BeginTx` and commit or rollback explicitly.
2.  **Defer Rollback:** Rollback unless committed, even when errors occur.
3.  **Multi-Step Updates:** Move money between accounts or update parent/child rows atomically.
4.  **Isolation Awareness:** Read about isolation levels and anomalies at a high level.

## 📖 Task Explanations

Transactions group changes into all-or-nothing units. Use them when partial success would leave inconsistent business state.

Always `defer tx.Rollback()` after beginning—Rollback becomes a no-op after successful Commit. Multi-step examples make the value obvious.

You do not need expert isolation knowledge today, but know transactions exist to prevent torn state.

## 💡 What You'll Gain

By the end of today, you will execute multi-step database changes safely inside transactions.

## 📚 Dictionary for Developers

*   **Transaction:** A unit of work that commits all changes or rolls back on failure.
*   **Commit:** Persisting all changes in a transaction permanently.
*   **Rollback:** Undoing all changes in a transaction after an error.
*   **Atomicity:** The guarantee that either all operations in a transaction succeed or none do.
*   **Isolation Level:** Rules governing how concurrent transactions see each other's changes.
