# Day 99: Capstone & Professional Delivery - Deployment and CI/CD

## 📝 Today's Tasks

1.  **Containerize Capstone:** Build production-like images with multi-stage Dockerfiles.
2.  **Extend CI Pipeline:** Test, lint, build, and optionally push images on main.
3.  **Document Deploy:** Write environment variables, migrations, and rollback steps.
4.  **Run Staging Deploy:** Deploy to a staging environment or local prod-like stack.

## 📖 Task Explanations

Day 99 is delivery mechanics: containers, CI, deploy docs, staging run. Reuse patterns from Days 76–80 instead of inventing new chaos.

Document rollback—deploys fail sometimes. Staging deploys reveal config mistakes before users see them.

CI should gate what you ship.

## 💡 What You'll Gain

By the end of today, your capstone will deploy via containers with CI and documented rollback.

## 📚 Dictionary for Developers

*   **Staging Environment:** A production-like environment for final validation before release.
*   **Rollback:** Reverting to a previous known-good deployment after a failed release.
*   **Image Push:** Uploading built container images to a registry for deployment.
*   **Deployment Runbook:** Steps operators follow to release and verify new versions.
*   **Production-Like:** Configuration resembling real operations, not simplified dev defaults.
