# Day 60: Project Layout & Architecture - Practice

## 📝 Today's Tasks

1.  **Refactor MVP Architecture:** Reorganize your MVP into domain, service, repository, and HTTP packages.
2.  **Add Service Tests:** Test use cases with fake repositories.
3.  **Draw Package Diagram:** Sketch imports to verify dependency direction.
4.  **Update Docs:** Explain architecture and package layout in README.

## 📖 Task Explanations

Today is a refactor milestone—behavior should stay the same while structure improves. Move logic out of handlers into services and domain types.

Drawing import arrows catches accidental cycles early. Architecture docs help future contributors respect boundaries.

Refactoring with tests green is a professional skill—run them often.

## 💡 What You'll Gain

By the end of today, your MVP will follow layered architecture with tested services and clear package docs.

## 📚 Dictionary for Developers

*   **Refactor:** Improving structure without changing external behavior.
*   **Import Cycle:** Circular package dependencies that Go forbids or that indicate design problems.
*   **Package Diagram:** A visual map of packages and their allowed dependencies.
*   **Use Case Test:** A test verifying application logic through the service layer.
*   **Maintainability:** How easily code can be understood, changed, and extended over time.
