# Day 51: Auth & Security - Authentication Basics

## 📝 Today's Tasks

1.  **Hash Passwords:** Use `bcrypt` or `argon2`—never store plaintext passwords.
2.  **Registration Flow:** Create users with unique emails and hashed passwords.
3.  **Login Flow:** Verify credentials and issue session tokens or JWTs.
4.  **Protect Routes:** Require authentication on selected handlers.

## 📖 Task Explanations

Authentication proves who the caller is. Password handling is security-critical: strong hashing, unique emails, and constant-time comparisons where libraries provide them.

Registration and login flows are classic REST exercises. Protected routes should fail with 401 when credentials are missing or invalid—not with generic 500s.

Build auth in small, testable steps.

## 💡 What You'll Gain

By the end of today, you will implement basic registration, login, and protected routes with hashed passwords.

## 📚 Dictionary for Developers

*   **Authentication:** Verifying the identity of a user or service.
*   **bcrypt:** A password hashing function designed to be slow against brute force.
*   **Plaintext Password:** An unhashed password that must never be stored.
*   **401 Unauthorized:** HTTP status when authentication is missing or invalid.
*   **Salt:** Random data combined with passwords before hashing to defeat rainbow tables.
