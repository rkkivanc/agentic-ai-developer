# Day 65: gRPC & Protocol Buffers - Practice

## 📝 Today's Tasks

1.  **Expose gRPC Alongside HTTP:** Add gRPC endpoints for one MVP use case while keeping HTTP.
2.  **Reuse Services:** Ensure business logic lives in shared services, not duplicated handlers.
3.  **Add Interceptors:** Log RPCs and reject missing auth metadata.
4.  **Document RPCs:** Describe how to generate stubs and call RPCs locally.

## 📖 Task Explanations

Today adds a second transport to your MVP—common in real systems. Keep business logic in services; HTTP and gRPC should be thin adapters.

Interceptors provide consistent logging and auth. Document codegen steps so teammates regenerate after `.proto` changes.

Two transports, one domain layer—that is professional structure.

## 💡 What You'll Gain

By the end of today, your project will expose shared use cases over both HTTP and gRPC with interceptors.

## 📚 Dictionary for Developers

*   **Multi-Transport:** Exposing the same capabilities via more than one protocol.
*   **Codegen:** Generating Go source from `.proto` definitions as part of development.
*   **Adapter:** Transport-specific code translating between protocol and service layer.
*   **Service Boundary:** A clear API surface between components or teams.
*   **Local Development:** Running services on a developer machine with documented commands.
