# Day 20: Concurrency Basics - Practice

## 📝 Today's Tasks

1.  **Concurrent Downloader:** Fetch several URLs concurrently and collect results with a WaitGroup.
2.  **Pipeline:** Build a three-stage pipeline: generate numbers, square them, print results.
3.  **Timeout Fetch:** Add a timeout to network calls using `select` and `time.After`.
4.  **Race Check:** Run your solutions with `-race` and fix any detected issues.

## 📖 Task Explanations

Today puts goroutines, channels, `select`, and sync primitives together. The downloader exercise mirrors real I/O-bound work; the pipeline teaches staged processing.

Add timeouts to network operations even in exercises—blocking forever is a common beginner mistake. Finish by running the race detector and fixing issues until it is clean.

## 💡 What You'll Gain

By the end of today, you will have built small concurrent programs that use coordination, timeouts, and safe shared-state patterns.

## 📚 Dictionary for Developers

*   **Pipeline:** A series of stages connected by channels, each performing part of a transformation.
*   **I/O-bound:** Work limited by waiting on input/output such as network or disk rather than CPU.
*   **Fan-out:** Starting multiple goroutines to process tasks in parallel.
*   **Fan-in:** Combining results from multiple goroutines into one stream or collection.
*   **Clean Shutdown:** Ending goroutines cooperatively without leaks or dangling work.
