# Day 15: Errors, Collections & I/O - Practice

## 📝 Today's Tasks

1.  **Config Loader:** Read a JSON config file into a struct and validate required fields.
2.  **Word Counter:** Read a text file and count word frequencies using a `map[string]int`.
3.  **Error Path Tests:** Manually test missing files, malformed JSON, and empty inputs.
4.  **Combine Skills:** Write a small CLI that loads config, reads data, and writes a JSON summary file.

## 📖 Task Explanations

Today combines errors, collections, file I/O, and JSON into practical utilities you might ship in a real tool.

The config loader reinforces struct tags and explicit error handling. The word counter exercises maps and file reading. Testing unhappy paths is as important as the happy path—professional Go code assumes inputs can fail.

Take time to keep functions small and errors informative. This block's habits will carry into HTTP and database work soon.

## 💡 What You'll Gain

By the end of today, you will have built small file-and-JSON utilities that demonstrate idiomatic error handling and collection usage.

## 📚 Dictionary for Developers

*   **CLI Utility:** A focused command-line tool that automates one task well, often composed of small packages.
*   **Validation:** Checking inputs or configuration for required values and acceptable ranges before processing.
*   **Frequency Map:** A map counting occurrences of keys, commonly used in text processing and analytics.
*   **Malformed Input:** Data that does not match the expected schema or format, which robust programs must handle.
*   **Summary Report:** A condensed output derived from processed data, often serialized as JSON for other systems.
