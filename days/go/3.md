# Day 3: Go Fundamentals - Functions and Packages

## 📝 Today's Tasks

1.  **Write Functions:** Define functions with parameters and return values, including multiple return values.
2.  **Export and Import:** Create a helper package and import it from `main` using your module path.
3.  **Use Named Returns:** Experiment with named result parameters and naked returns in small functions.
4.  **Read godoc Style:** Add brief comments above exported functions so `go doc` can describe your API.

## 📖 Task Explanations

Functions are the primary unit of behavior in Go. You'll learn how to define them, return multiple values (a Go hallmark), and organize related functions into packages.

Packages are how Go code is structured and reused. By splitting code into packages, you create clear boundaries and reusable libraries. Understanding what is exported (capitalized) versus unexported (lowercase) is essential for designing maintainable APIs.

Named return values and documentation comments are small details that become important as your projects grow. Today you build habits that professional Go teams rely on every day.

## 💡 What You'll Gain

By the end of today, you will understand how to structure Go code into functions and packages, return multiple values idiomatically, and document exported APIs.

## 📚 Dictionary for Developers

*   **Function:** A named block of code that performs a task. Go functions can take parameters and return one or more values.
*   **Package:** A directory of Go files that compile together. The package name appears in the `package` declaration.
*   **Export:** Making an identifier available outside its package by capitalizing its first letter.
*   **Import:** Bringing another package into scope with the `import` keyword.
*   **godoc:** Go's documentation convention: comments immediately preceding declarations become package documentation.
