# Day 27: HTTP Basics & Handlers - Routing and Handlers

## 📝 Today's Tasks

1.  **Use ServeMux:** Register patterns like `/health` and `/users/` on a mux.
2.  **Path Parameters:** Parse IDs from paths manually or with small helpers.
3.  **Method Checks:** Return 405 Method Not Allowed when verbs do not match.
4.  **Organize Handlers:** Split handlers into separate functions and files by resource.

## 📖 Task Explanations

Routing maps URLs and methods to handler logic. Go 1.22+ improved `ServeMux` pattern matching, but explicit method checks remain important for REST design.

You'll organize handlers by resource so the project stays navigable as it grows. Even without a framework, consistent structure matters.

Clear routing early prevents spaghetti `switch` statements inside one giant handler.

## 💡 What You'll Gain

By the end of today, you will route requests to focused handlers and enforce HTTP methods per endpoint.

## 📚 Dictionary for Developers

*   **ServeMux:** An HTTP request multiplexer matching URL patterns to handlers.
*   **Route:** A mapping from a URL pattern and HTTP methods to a handler function.
*   **405 Method Not Allowed:** A status code indicating the verb is not supported for the resource.
*   **Resource:** A logical entity exposed by the API, such as users or orders.
*   **REST:** An architectural style using HTTP methods and resources, common for JSON APIs.
