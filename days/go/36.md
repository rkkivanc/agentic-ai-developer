# Day 36: Context, Config & Middleware - context.Context Fundamentals

## 📝 Today's Tasks

1.  **Pass Context:** Thread `context.Context` as the first parameter in service and store functions.
2.  **Use context.WithCancel:** Cancel background work when requests end or clients disconnect.
3.  **Set Deadlines:** Apply timeouts with `context.WithTimeout` around slow operations.
4.  **Respect ctx.Done:** Check cancellation in loops and before expensive steps.

## 📖 Task Explanations

`context.Context` carries deadlines, cancellation signals, and request-scoped values across API boundaries. In Go services, it is the standard way to stop work when it is no longer needed.

You'll propagate context from HTTP handlers into storage and outbound calls. Honoring cancellation prevents goroutine leaks and wasted CPU.

Deadlines translate product requirements into code: every outbound call should have a bounded wait time.

## 💡 What You'll Gain

By the end of today, you will propagate and honor `context.Context` for cancellation and timeouts.

## 📚 Dictionary for Developers

*   **context.Context:** A standard interface carrying deadlines, cancellation, and request-scoped values.
*   **Cancellation:** Signaling that work should stop because a request ended or timed out.
*   **Deadline:** A point in time after which an operation should abort.
*   **ctx.Done:** A channel closed when the context is canceled or times out.
*   **Leak:** A goroutine or resource that continues after it is no longer needed.
