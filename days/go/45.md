# Day 45: Databases (I) - Practice Project & Review

## 📝 Today's Tasks

1.  **Persist MVP Data:** Replace in-memory storage with database-backed CRUD for your MVP.
2.  **Apply Migrations:** Check migrations into the repo and document how to run them.
3.  **Transaction Where Needed:** Use a transaction for multi-row updates if your domain requires it.
4.  **Integration Smoke Test:** Manually verify CRUD against a real database instance.

## 📖 Task Explanations

Today connects HTTP handlers to a real database—the biggest leap since starting the MVP. Keep repository functions small and test them with real SQL against a dev database.

Document migration commands for teammates. Smoke-test every CRUD path; database bugs often appear only with real persistence.

Celebrate this milestone: your API now survives process restarts.

## 💡 What You'll Gain

By the end of today, your MVP will persist data in a SQL database with migrations and tested CRUD flows.

## 📚 Dictionary for Developers

*   **Persistence:** Storing data outside process memory so it survives restarts.
*   **Integration Smoke Test:** A quick end-to-end check against real infrastructure components.
*   **Repository Function:** A data-access function hiding SQL details from handlers.
*   **Dev Database:** A local database instance for development, separate from production.
*   **Stateful Service:** A service retaining data across requests and restarts.
