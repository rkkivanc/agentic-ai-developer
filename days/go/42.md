# Day 42: Databases (I) - Migrations and Schema Design

## 📝 Today's Tasks

1.  **Choose a Migration Tool:** Use `goose`, `golang-migrate`, or SQL files checked into the repo.
2.  **Design Tables:** Model your MVP entities with primary keys, timestamps, and indexes.
3.  **Write Up/Down Migrations:** Create reversible migrations for schema changes.
4.  **Run Migrations in Dev:** Apply migrations locally and verify schema matches expectations.

## 📖 Task Explanations

Schema changes should be versioned like code. Migrations let teams evolve databases reproducibly across environments.

Design tables with clear primary keys and timestamps from day one. Indexes support queries you already know you need—do not wait for production slowness.

Reversible migrations save you during development when experiments go wrong.

## 💡 What You'll Gain

By the end of today, you will design a schema and manage it with versioned migrations.

## 📚 Dictionary for Developers

*   **Migration:** A versioned script applying or reverting database schema changes.
*   **Primary Key:** A unique identifier column for each row in a table.
*   **Index:** A database structure speeding lookups at the cost of extra storage and write work.
*   **Timestamp Column:** Fields recording creation or update times for auditing and sorting.
*   **Schema:** The structure of tables, columns, constraints, and indexes in a database.
