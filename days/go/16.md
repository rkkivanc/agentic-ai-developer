# Day 16: Concurrency Basics - Goroutines

## 📝 Today's Tasks

1.  **Launch Goroutines:** Start concurrent work with `go` and observe interleaved output.
2.  **Use sync.WaitGroup:** Wait for a group of goroutines to finish before exiting `main`.
3.  **Share Data Safely:** Identify a data race in a naive counter and note why it is unsafe.
4.  **Run the Race Detector:** Execute tests or programs with `-race` to catch unsafe concurrent access.

## 📖 Task Explanations

Goroutines are lightweight threads managed by the Go runtime. They make concurrent programming approachable, but concurrency still requires discipline.

You'll launch goroutines for parallel tasks and coordinate completion with `sync.WaitGroup`. A deliberate race exercise shows why unsynchronized shared memory fails.

The race detector is one of Go's best teaching tools—use it early and often when learning concurrency.

## 💡 What You'll Gain

By the end of today, you will be able to launch goroutines, wait for completion, and recognize unsafe shared-state access.

## 📚 Dictionary for Developers

*   **Goroutine:** A lightweight concurrent function execution managed by the Go runtime.
*   **Concurrency:** Structuring a program to handle multiple tasks that may overlap in time.
*   **WaitGroup:** A counter from `sync` that waits for a collection of goroutines to finish.
*   **Data Race:** Undefined behavior when two goroutines access the same memory concurrently and at least one writes.
*   **Race Detector:** A Go tool enabled with `-race` that finds data races at runtime.
