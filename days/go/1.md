# Day 1: Go Fundamentals - Variables, Types, and Your First Program

## 📝 Today's Tasks

1.  **Install Go:** Install the latest stable Go toolchain from go.dev and verify with `go version`.
2.  **Initialize a Module:** Create a working directory and run `go mod init` to start a new Go module.
3.  **Write Hello World:** Create `main.go` that prints a greeting and run it with `go run .`.
4.  **Explore Core Types:** Declare variables with `var` and `:=`, and experiment with `int`, `float64`, `string`, and `bool`.
5.  **Use go fmt:** Format your code with `go fmt` and notice how Go enforces a consistent style.

## 📖 Task Explanations

Today is about laying the groundwork for your Go journey. A proper development environment is essential for a smooth learning experience. Installing the Go toolchain gives you the compiler, formatter, and module tools you need, while a good editor with Go support will provide syntax highlighting, completion, and debugging.

Your Hello World program is a classic first step. It confirms that your environment works and that you understand the basic structure of a Go program: package declaration, imports, and a `main` function.

The core of today's lesson is understanding how Go handles data. You'll learn how to declare variables, use type inference with `:=`, and work with Go's built-in types. Every program you write will build on these fundamentals.

## 💡 What You'll Gain

By the end of today, you will have a functional Go development setup and a core understanding of variables and fundamental data types. This knowledge is the bedrock for all future Go concepts.

## 📚 Dictionary for Developers

*   **SDK (Software Development Kit):** A collection of development tools in one installable package. The Go SDK includes the compiler, standard library, and `go` command.
*   **Module:** A unit of versioned Go code identified by a module path in `go.mod`. Modules manage dependencies and package paths.
*   **Package:** A way to organize Go source files. Every Go file belongs to a package; executable programs use `package main`.
*   **Variable:** A named storage location that holds a value. Go supports explicit types and short declaration with `:=`.
*   **Data Type:** A classification specifying what kind of value a variable can hold, such as `int`, `string`, or `bool`.
*   **go fmt:** A tool that formats Go source into the standard style, keeping code consistent across projects and teams.
