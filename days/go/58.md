# Day 58: Project Layout & Architecture - Dependency Injection

## 📝 Today's Tasks

1.  **Constructor Wiring:** Build services and handlers with explicit dependencies.
2.  **Interface Boundaries:** Depend on interfaces for repositories, clocks, and ID generators.
3.  **Avoid Service Locator:** Do not hide dependencies in global registries.
4.  **Test with Fakes:** Inject fakes to test services without databases.

## 📖 Task Explanations

Explicit dependency injection makes components testable and readable. Constructors state what a type needs; readers see coupling immediately.

Interfaces at boundaries enable fakes. Avoid service locators and hidden globals—they obscure dependencies and encourage spaghetti.

Service tests with fakes are fast and precise.

## 💡 What You'll Gain

By the end of today, you will wire dependencies explicitly and test services with injected fakes.

## 📚 Dictionary for Developers

*   **Dependency Injection:** Passing dependencies into a component rather than creating them internally.
*   **Fake:** A test double implementing an interface with simple in-memory behavior.
*   **Service Locator:** An anti-pattern hiding dependency lookup in globals.
*   **Clock Interface:** An injectable time source making time-dependent logic testable.
*   **Coupling:** How tightly components depend on concrete implementations.
