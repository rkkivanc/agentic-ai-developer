# Day 92: Team Practices & Tooling - Documentation and API Contracts

## 📝 Today's Tasks

1.  **Write Package Docs:** Add package-level godoc comments explaining purpose.
2.  **Maintain OpenAPI or Proto:** Keep HTTP or gRPC contracts updated with implementation.
3.  **Add ARCHITECTURE.md:** Summarize layers, dependencies, and key decisions.
4.  **Changelog Entries:** Record user-visible changes per release.

## 📖 Task Explanations

Documentation is part of the deliverable. godoc helps teammates discover packages; OpenAPI or proto files are contracts clients rely on.

ARCHITECTURE.md captures decisions README should not drown in. Changelogs communicate impact to consumers.

Stale docs harm trust—update them with behavior changes.

## 💡 What You'll Gain

By the end of today, you will improve godoc, API contracts, and architecture documentation for your project.

## 📚 Dictionary for Developers

*   **godoc:** Go documentation generated from comments above packages and exported symbols.
*   **OpenAPI:** A specification format describing REST HTTP APIs.
*   **ARCHITECTURE.md:** A document explaining system structure and major technical decisions.
*   **Changelog:** A file listing notable changes per version for consumers.
*   **Contract:** A formal description of API behavior both sides must honor.
