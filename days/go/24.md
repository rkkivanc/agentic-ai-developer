# Day 24: Testing Fundamentals - httptest and Test Helpers

## 📝 Today's Tasks

1.  **Use httptest.NewRecorder:** Test HTTP handlers without opening real network ports.
2.  **Build Request Fixtures:** Construct `http.Request` values with bodies and headers for handler tests.
3.  **Extract Helpers:** Create helpers for common setup and mark them with `t.Helper()`.
4.  **Table-Test Handlers:** Apply table-driven patterns to HTTP status codes and JSON bodies.

## 📖 Task Explanations

HTTP handlers are easy to test in isolation with `net/http/httptest`. Recorders capture status, headers, and bodies without flakiness from real ports.

You'll construct requests programmatically and assert responses—skills you'll reuse throughout the REST API project. Helpers keep handler tests short and focused.

Table-driven handler tests document your API contract: method, path, payload, expected status.

## 💡 What You'll Gain

By the end of today, you will test HTTP handlers with `httptest` and organize reusable test helpers.

## 📚 Dictionary for Developers

*   **httptest.ResponseRecorder:** A fake `http.ResponseWriter` recording handler output for assertions.
*   **Handler:** A function with signature `http.HandlerFunc` that processes requests and writes responses.
*   **Fixture:** Predefined test data or objects reused across multiple tests.
*   **Contract Test:** A test verifying an API's expected inputs and outputs for consumers.
*   **Flaky Test:** A test that sometimes passes and sometimes fails, often due to timing or real network use.
