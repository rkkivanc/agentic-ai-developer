# Day 39: Context, Config & Middleware - Graceful Shutdown

## 📝 Today's Tasks

1.  **Trap Signals:** Listen for `SIGINT` and `SIGTERM` using `os/signal`.
2.  **Use http.Server Shutdown:** Call `Shutdown` with a context to drain in-flight requests.
3.  **Set Timeouts:** Configure read/write timeouts on `http.Server`.
4.  **Close Resources:** Flush logs and close database connections on exit in later projects.

## 📖 Task Explanations

Production servers must stop without dropping active requests. Graceful shutdown listens for signals, stops accepting new work, waits for in-flight requests, then exits.

Configure server timeouts to protect against slow clients. Even before databases, practice shutdown in your MVP so the habit is automatic later.

Operators notice services that drain cleanly during deploys.

## 💡 What You'll Gain

By the end of today, you will implement graceful HTTP shutdown and basic server timeouts.

## 📚 Dictionary for Developers

*   **Graceful Shutdown:** Stopping a server without abruptly terminating active requests.
*   **SIGTERM:** A signal asking a process to terminate, commonly sent by orchestrators.
*   **Drain:** Allowing in-flight work to finish before process exit.
*   **http.Server:** A configurable HTTP server type supporting timeouts and graceful shutdown.
*   **Slowloris:** An attack pattern exploiting connections kept open without completing requests.
