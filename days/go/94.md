# Day 94: Team Practices & Tooling - Release and Versioning

## 📝 Today's Tasks

1.  **Semantic Versioning:** Tag releases with MAJOR.MINOR.PATCH meaning.
2.  **Generate Release Notes:** Summarize changes between tags for deployers.
3.  **Build Reproducibly:** Ensure CI builds the same artifacts from a tag.
4.  **Deprecate Safely:** Mark APIs deprecated before removal with migration notes.

## 📖 Task Explanations

Releases communicate stability expectations. Semver signals breaking changes; release notes explain operational impact.

Reproducible builds from tags let you redeploy old versions confidently. Deprecate with timelines—clients need migration paths.

Versioning discipline separates hobby projects from products.

## 💡 What You'll Gain

By the end of today, you will version releases, write release notes, and plan safe API deprecations.

## 📚 Dictionary for Developers

*   **Semantic Versioning:** A scheme where version numbers convey compatibility expectations.
*   **Release Notes:** Human-readable summary of changes in a deployment.
*   **Reproducible Build:** Building the same artifact from the same source and dependencies.
*   **Deprecation:** Marking features for future removal with guidance to migrate.
*   **Breaking Change:** A change requiring clients to update their integration.
