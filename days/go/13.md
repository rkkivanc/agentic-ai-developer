# Day 13: Errors, Collections & I/O - Reading and Writing Files

## 📝 Today's Tasks

1.  **Read Files:** Use `os.ReadFile` and `bufio` to read text files line by line.
2.  **Write Files:** Create and write files with `os.WriteFile` and buffered writers.
3.  **Work with io:** Use `io.Reader` and `io.Writer` interfaces in small utilities.
4.  **Handle File Errors:** Check for `os.IsNotExist` and return wrapped errors with useful context.

## 📖 Task Explanations

File I/O is where programs meet the outside world. Go's `io` package provides abstractions that make it easy to test and compose readers and writers.

You'll practice reading and writing both small and large files, using buffering where appropriate. Good error handling matters here: missing files, permission errors, and partial writes should all be reported clearly.

These skills transfer directly to loading configuration, exporting reports, and processing data pipelines.

## 💡 What You'll Gain

By the end of today, you will be able to read and write files using Go's `os` and `io` packages with solid error handling.

## 📚 Dictionary for Developers

*   **io.Reader:** An interface with a `Read` method for consuming byte streams from many sources.
*   **io.Writer:** An interface with a `Write` method for sending byte streams to many destinations.
*   **bufio:** A package providing buffered I/O wrappers that reduce system calls and simplify line reading.
*   **File Descriptor:** An operating system handle representing an open file used by `os.File`.
*   **os.IsNotExist:** A helper that reports whether an error indicates a missing file or directory.
