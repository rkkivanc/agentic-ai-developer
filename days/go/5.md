# Day 5: Go Fundamentals - Practice

## 📝 Today's Tasks

1.  **CLI Calculator:** Build a tiny command-line program that reads two numbers and an operator, then prints the result.
2.  **Temperature Converter:** Write functions to convert between Celsius and Fahrenheit and call them from `main`.
3.  **Package Refactor:** Move helper functions into a separate package and import them cleanly.
4.  **Self-Review:** Run `go fmt`, `go vet`, and manually test edge cases like division by zero.

## 📖 Task Explanations

Today is about putting Days 1–4 into practice. Small programs solidify syntax, packages, and control flow better than reading alone.

Your calculator exercise combines input, branching, and functions. The temperature converter reinforces multiple return values and reusable packages. Take time to test edge cases—empty input, invalid operators, and boundary values—so you build the habit of thinking beyond the happy path.

## 💡 What You'll Gain

By the end of today, you will have written several small Go programs that combine variables, control flow, functions, and packages. This practice prepares you for modeling real domain data with structs.

## 📚 Dictionary for Developers

*   **CLI (Command-Line Interface):** A text-based program that runs in the terminal and accepts user input as arguments or prompts.
*   **Edge Case:** A scenario at the boundary of normal operation, such as invalid input or division by zero.
*   **go vet:** A static analysis tool that reports suspicious constructs in Go code.
*   **Refactor:** Restructuring code to improve clarity or organization without changing external behavior.
*   **Happy Path:** The expected successful flow through a program when inputs and conditions are normal.
