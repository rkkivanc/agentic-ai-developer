# Day 62: gRPC & Protocol Buffers - Defining gRPC Services

## 📝 Today's Tasks

1.  **Define RPC Methods:** Add service definitions with unary RPCs in `.proto` files.
2.  **Generate gRPC Stubs:** Produce server and client interfaces with Go gRPC plugins.
3.  **Version APIs:** Plan package names and backward-compatible field additions.
4.  **Document RPCs:** Comment proto files describing behavior and error expectations.

## 📖 Task Explanations

gRPC services declare RPC methods with request and response messages. Unary RPCs are the starting point before streaming.

Generated interfaces push you toward clean boundaries—servers implement generated interfaces; clients call stubs.

Proto comments are API docs—write them for humans too.

## 💡 What You'll Gain

By the end of today, you will define gRPC services in protobuf and generate server/client interfaces.

## 📚 Dictionary for Developers

*   **gRPC:** A high-performance RPC framework using HTTP/2 and Protocol Buffers.
*   **Unary RPC:** A single request followed by a single response.
*   **Stub:** Generated client code wrapping RPC calls.
*   **Backward Compatibility:** Adding fields without breaking existing clients using protobuf rules.
*   **RPC:** Remote Procedure Call invoking a method on another service over the network.
