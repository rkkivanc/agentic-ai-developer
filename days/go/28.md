# Day 28: HTTP Basics & Handlers - Request and Response Details

## 📝 Today's Tasks

1.  **Read JSON Bodies:** Decode request bodies with `json.NewDecoder` and validate required fields.
2.  **Write JSON Responses:** Set `Content-Type: application/json` and encode structs safely.
3.  **Use Headers:** Read and set headers for auth placeholders, request IDs, and caching hints.
4.  **Return Problem Details:** Send consistent JSON error bodies with codes and messages.

## 📖 Task Explanations

Production APIs live in the details: correct content types, structured errors, and careful body decoding. Partial reads and silent failures cause painful client bugs.

You'll decode JSON from request bodies and encode responses with explicit headers. Consistent error JSON helps frontend and mobile clients immensely.

These patterns flow directly into your MVP REST project next week.

## 💡 What You'll Gain

By the end of today, you will read JSON request bodies, write structured JSON responses, and return consistent error payloads.

## 📚 Dictionary for Developers

*   **Content-Type:** An HTTP header declaring the media type of the request or response body.
*   **Decoder:** A JSON decoder reading sequentially from an `io.Reader`, suitable for HTTP bodies.
*   **Problem Details:** A structured error response format describing what went wrong for clients.
*   **Request ID:** A unique identifier propagated through logs and responses to trace a single request.
*   **Validation:** Checking that input fields are present, correctly typed, and within acceptable constraints.
