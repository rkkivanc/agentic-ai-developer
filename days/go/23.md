# Day 23: Testing Fundamentals - Benchmarks and Examples

## 📝 Today's Tasks

1.  **Write Benchmarks:** Create `BenchmarkXxx` functions and run `go test -bench`.
2.  **Reset Timer:** Use `b.ResetTimer()` to exclude expensive setup from measurements.
3.  **Add Examples:** Write `ExampleXxx` functions with `// Output:` comments for documentation tests.
4.  **Read ns/op:** Interpret benchmark output and compare runs after small changes.

## 📖 Task Explanations

Benchmarks measure performance changes over time. They are not profiling, but they help you notice when a refactor unexpectedly regresses speed.

Example tests double as documentation that must compile and match expected output—great for libraries and utility packages.

Learn to read `ns/op` and allocations columns so later performance work with `pprof` feels familiar.

## 💡 What You'll Gain

By the end of today, you will benchmark Go code, document behavior with examples, and read basic benchmark results.

## 📚 Dictionary for Developers

*   **Benchmark:** A test measuring performance, reported as nanoseconds per operation by default.
*   **b.ResetTimer:** A benchmark helper excluding setup time from measured iterations.
*   **Example Test:** A doctest-style function verifying printed output matches comments.
*   **ns/op:** Nanoseconds per operation, the default benchmark timing unit.
*   **Allocation:** Heap memory allocated per benchmark iteration, shown when `-benchmem` is enabled.
