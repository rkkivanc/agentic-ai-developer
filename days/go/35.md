# Day 35: First REST API Project (MVP) - Finalization & Review

## 📝 Today's Tasks

1.  **Final Polish:** Review your MVP API for naming consistency, status codes, and JSON shapes.
2.  **Manual Testing:** Walk through all CRUD flows with `curl` including edge cases.
3.  **Code Review:** Remove dead code, run `go fmt` and `go vet`, and improve package boundaries.
4.  **Reflection:** Write notes on what you learned and what was challenging during the MVP.

## 📖 Task Explanations

Today mirrors Flutter's MVP review day: polish, test, review, reflect. A working CRUD API is a milestone—take time to make it presentable.

Try edge cases: duplicate creates, invalid IDs, empty JSON bodies. Code review your own work as a teammate would.

Reflection solidifies learning and highlights what to study before adding databases and authentication.

## 💡 What You'll Gain

By the end of today, you will have completed your first substantial Go REST MVP with tested endpoints and thoughtful review habits.

## 📚 Dictionary for Developers

*   **MVP Review:** A milestone pass verifying the smallest useful product meets agreed behavior.
*   **Edge Case Testing:** Manually or automatically verifying unusual inputs and failure scenarios.
*   **Package Boundary:** A separation between layers such as HTTP handlers and storage.
*   **Technical Debt:** Shortcuts taken for speed that should be revisited, documented honestly during review.
*   **Portfolio Piece:** A project demonstrating skills to employers or collaborators.
