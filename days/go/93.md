# Day 93: Team Practices & Tooling - Makefile and Developer Experience

## 📝 Today's Tasks

1.  **Add Makefile Targets:** Provide `make test`, `make lint`, `make run`, and `make migrate`.
2.  **One-Command Setup:** Document bootstrap from clone to running service.
3.  **Use direnv Optionally:** Load local env vars automatically for contributors.
4.  **Pre-commit Hooks Optionally:** Run fmt and fast tests before commits.

## 📖 Task Explanations

Developer experience determines how fast newcomers contribute. Makefiles or `mage` scripts encode tribal knowledge into commands everyone can run.

One-command setup reduces onboarding friction. Optional direnv and pre-commit hooks keep local workflows consistent.

DX is productivity for the whole team.

## 💡 What You'll Gain

By the end of today, you will provide Makefile-driven workflows and clear setup instructions.

## 📚 Dictionary for Developers

*   **Makefile:** A file defining command shortcuts like test and lint targets.
*   **Developer Experience (DX):** How easy and pleasant it is to work on a codebase day to day.
*   **Bootstrap:** Steps to go from fresh clone to running application.
*   **Pre-commit Hook:** A script running checks automatically before git commits.
*   **Tribal Knowledge:** Undocumented practices that should be encoded in scripts and docs.
