# Day 21: Testing Fundamentals - Writing Your First Tests

## 📝 Today's Tasks

1.  **Create _test.go Files:** Place tests alongside code in `something_test.go` files within the same package.
2.  **Use testing Package:** Write `TestXxx` functions and run them with `go test`.
3.  **Assert with Comparisons:** Compare got/want values and use `t.Fatalf` or `t.Errorf` on failures.
4.  **Test Exported Behavior:** Focus tests on public API behavior consumers rely on.

## 📖 Task Explanations

Testing is first-class in Go. The standard library's `testing` package needs no extra frameworks for unit tests, which keeps projects simple and portable.

You'll write table-free starter tests, run them locally, and see how failures report clearly. Testing exported behavior encourages stable APIs and documents expected usage.

From here forward, testing is part of every feature—not an afterthought.

## 💡 What You'll Gain

By the end of today, you will write and run basic Go unit tests alongside your production code.

## 📚 Dictionary for Developers

*   **Unit Test:** A test verifying a small unit of code, usually a function or method, in isolation.
*   **testing.T:** The type passed to test functions carrying state, logging, and failure methods.
*   **go test:** The command that compiles and runs tests in the current module or package.
*   **Fatalf:** A testing helper that logs a formatted message and marks the test failed immediately.
*   **Test Package:** Tests in the same package as code (`package foo`) can access unexported symbols.
