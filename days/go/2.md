# Day 2: Go Fundamentals - Control Flow

## 📝 Today's Tasks

1.  **Learn if/else:** Write conditional logic with `if`, `else if`, and `else` to branch based on boolean conditions.
2.  **Use for Loops:** Iterate with `for` over ranges, slices, and classic C-style loop headers.
3.  **Try switch:** Use `switch` on values and expressions, including switch without a tag for cleaner condition chains.
4.  **Handle break and continue:** Control loop execution with `break` and `continue`, and label loops when needed.

## 📖 Task Explanations

Today is about making your programs smart. Control flow statements let your code make decisions and repeat actions. `if/else` is the foundation of branching logic in Go.

Go's `for` loop is the only looping construct, but it is flexible enough to replace `while` and `do-while` patterns from other languages. You'll use it to process collections and repeat work until a condition changes.

`switch` in Go is powerful and readable. It can replace long `if/else` chains and supports expression-less switches that read like a series of conditions.

## 💡 What You'll Gain

By the end of today, you will be able to write programs that react to different situations and perform repetitive tasks efficiently. This is a crucial step toward building applications that handle real input and data.

## 📚 Dictionary for Developers

*   **Control Flow:** The order in which a program executes its statements. Statements like `if`, `for`, and `switch` alter this order.
*   **Conditional Statement:** A statement that performs different actions depending on whether a condition is true or false.
*   **Loop:** A control flow construct that executes a block of code repeatedly until a condition is met.
*   **Range:** A Go construct used with `for` to iterate over slices, arrays, maps, strings, and channels.
*   **switch:** A control flow statement that selects among multiple cases, often cleaner than long if/else chains.
