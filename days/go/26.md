# Day 26: HTTP Basics & Handlers - net/http Basics

## 📝 Today's Tasks

1.  **Start an HTTP Server:** Use `http.ListenAndServe` with a handler function or `ServeMux`.
2.  **Write HandlerFuncs:** Respond with plain text and set status codes explicitly.
3.  **Inspect Requests:** Log method, path, headers, and remote address in a handler.
4.  **Serve Locally:** Call your API with `curl` and observe request/response details.

## 📖 Task Explanations

Go's standard library includes a production-capable HTTP server. You do not need a framework to learn fundamentals—`net/http` is enough.

Handlers are functions that receive a response writer and request. You'll practice setting status codes and reading request metadata, which every REST endpoint relies on.

Using `curl` trains you to think like a client, which is invaluable when debugging APIs.

## 💡 What You'll Gain

By the end of today, you will stand up a basic HTTP server and write handlers that inspect requests and return responses.

## 📚 Dictionary for Developers

*   **net/http:** Go's standard library package for HTTP clients and servers.
*   **Handler:** Code that processes an `http.Request` and writes an `http.Response`.
*   **Status Code:** A numeric code like 200 or 404 indicating the outcome of an HTTP request.
*   **ListenAndServe:** A function binding a TCP address and serving HTTP with a handler.
*   **curl:** A command-line tool for making HTTP requests, widely used for API debugging.
