# Day 86: Performance - Profiling with pprof

## 📝 Today's Tasks

1.  **Enable pprof:** Expose `net/http/pprof` on a debug port or import in main.
2.  **CPU Profile:** Capture CPU profiles under load and view top functions.
3.  **Heap Profile:** Inspect allocations and retained memory.
4.  **Benchmark Before/After:** Compare benchmarks after a small optimization.

## 📖 Task Explanations

`pprof` is Go's built-in profiler. CPU profiles show hot functions; heap profiles reveal allocation sources.

Generate load representative of reality—empty servers profile poorly. Pair profiling with benchmarks to confirm wins.

Optimize with evidence, not guesses.

## 💡 What You'll Gain

By the end of today, you will capture and interpret CPU and heap profiles with pprof.

## 📚 Dictionary for Developers

*   **pprof:** Go's profiling tool for CPU, memory, goroutine, and other runtime data.
*   **CPU Profile:** Samples showing where CPU time is spent during a window.
*   **Heap Profile:** Data about memory allocations and retained objects.
*   **Hot Function:** A function consuming disproportionate CPU or allocations.
*   **Profile-Guided Optimization:** Improving code based on measured bottlenecks, not assumptions.
