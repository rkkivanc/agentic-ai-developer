# Day 30: HTTP Basics & Handlers - Practice Project & Review

## 📝 Today's Tasks

1.  **Build a Notes API:** Create endpoints for listing, creating, and deleting in-memory notes.
2.  **Add Middleware:** Apply logging and recovery middleware to all routes.
3.  **Write Handler Tests:** Cover success and error cases with `httptest`.
4.  **Document Endpoints:** Write a short README listing routes, methods, and example `curl` commands.

## 📖 Task Explanations

Today you build a compact HTTP service combining routing, JSON, middleware, and tests—similar to Flutter's practice milestones.

Keep storage in memory for now; persistence comes later. Tests should cover invalid JSON, wrong methods, and missing IDs.

A README with `curl` examples is early API documentation practice that professional teams expect.

## 💡 What You'll Gain

By the end of today, you will have a tested mini HTTP service demonstrating routing, JSON, middleware, and basic documentation.

## 📚 Dictionary for Developers

*   **In-Memory Store:** A data structure inside the process used temporarily instead of a database.
*   **API Documentation:** Descriptions of endpoints, payloads, and responses so clients can integrate correctly.
*   **Smoke Test:** A quick manual or automated check that the service starts and core routes respond.
*   **README:** A project overview file explaining setup, usage, and examples.
*   **CRUD:** Create, Read, Update, Delete—the core operations most REST APIs expose.
