## 📖 Task Explanations

Slices and maps are the workhorse collections in Go. Slices are flexible views over arrays; maps provide fast key-based lookup.

Understanding capacity and append behavior prevents subtle bugs when multiple parts of a program share the same backing array. Maps require care too: they are reference types, and iteration order is intentionally randomized.

Mastering these collections prepares you for JSON decoding, in-memory stores, and repository layers later in the curriculum.

## 📚 Dictionary for Developers

*   **Slice:** A dynamically-sized view into an underlying array, defined by pointer, length, and capacity.
*   **Map:** An unordered associative container that maps keys to values with fast lookup.
*   **Capacity:** The number of elements a slice can hold before needing to reallocate its backing array.
*   **append:** A built-in function that adds elements to a slice, reallocating when capacity is exceeded.
*   **Reference Semantics:** Behavior where copying a header still points at shared underlying data, as with slices and maps.
