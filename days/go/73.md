# Day 73: Observability & Resilience - Tracing and OpenTelemetry

## 📝 Today's Tasks

1.  **Add OTel SDK:** Initialize OpenTelemetry tracing in your service.
2.  **Create Spans:** Wrap HTTP handlers and repository calls in spans.
3.  **Propagate Trace Context:** Pass trace headers through outbound HTTP or gRPC calls.
4.  **Export Locally:** Send traces to a local collector or console exporter for learning.

## 📖 Task Explanations

Tracing shows where time goes across functions and services. Spans nest to form traces you can visualize.

Propagate context so multi-service requests stay connected. Start with console or local collectors before production vendors.

Tracing complements logs and metrics—it answers why latency spiked.

## 💡 What You'll Gain

By the end of today, you will create and propagate basic OpenTelemetry traces through your service.

## 📚 Dictionary for Developers

*   **OpenTelemetry:** A standard for traces, metrics, and logs across languages and vendors.
*   **Span:** A single timed operation within a distributed trace.
*   **Trace:** A collection of spans representing one end-to-end request.
*   **Propagation:** Forwarding trace identifiers across network calls.
*   **Collector:** A component receiving telemetry data for storage or visualization.
