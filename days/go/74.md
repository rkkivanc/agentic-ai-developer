# Day 74: Observability & Resilience - Retries, Circuit Breakers, and Timeouts

## 📝 Today's Tasks

1.  **Retry with Backoff:** Retry idempotent operations with exponential backoff and jitter.
2.  **Set Client Timeouts:** Apply timeouts on outbound HTTP and database calls.
3.  **Introduce Circuit Breaker:** Use a breaker library to fail fast when dependencies are unhealthy.
4.  **Document Policies:** Write when retries are safe versus dangerous.

## 📖 Task Explanations

Resilience patterns prevent cascading failures. Retries help transient errors; backoff avoids hammering sick dependencies.

Timeouts bound waiting forever. Circuit breakers stop calling dependencies likely to fail, giving them time to recover.

Only retry idempotent operations unless you have deduplication.

## 💡 What You'll Gain

By the end of today, you will apply retries, timeouts, and circuit breaking thoughtfully on outbound calls.

## 📚 Dictionary for Developers

*   **Exponential Backoff:** Increasing delay between retries to reduce load on failing systems.
*   **Jitter:** Random variation added to backoff to prevent synchronized retries.
*   **Circuit Breaker:** A pattern stopping calls to a failing dependency until it recovers.
*   **Idempotent Operation:** Safe to repeat without unintended side effects.
*   **Cascading Failure:** When one outage triggers others through retries and overload.
