# Day 90: Performance - Practice

## 📝 Today's Tasks

1.  **Profile MVP:** Load test a hot endpoint and capture CPU and heap profiles.
2.  **Fix One Bottleneck:** Apply an optimization guided by data, not instinct.
3.  **Document Results:** Record before/after latency and allocation metrics.
4.  **Guard Against Regressions:** Add a benchmark or perf test threshold if practical.

## 📖 Task Explanations

Today is evidence-based performance work on your MVP. Load test realistically, profile, fix one real bottleneck, document results.

Even modest improvements with data beat random micro-optimizations. Optional benchmark thresholds catch regressions early.

Performance is iterative—this is one solid cycle.

## 💡 What You'll Gain

By the end of today, you will have improved a measured bottleneck and documented the impact.

## 📚 Dictionary for Developers

*   **Load Test:** Generating artificial traffic to observe behavior under stress.
*   **Baseline:** Measurements before changes used for comparison.
*   **Regression Threshold:** A limit in CI failing builds when performance worsens significantly.
*   **Latency:** Time taken to complete a request or operation.
*   **Bottleneck:** The limiting factor preventing higher throughput or lower latency.
