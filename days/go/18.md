# Day 18: Concurrency Basics - Select and Timeouts

## 📝 Today's Tasks

1.  **Use select:** Wait on multiple channel operations with `select` and `default` where appropriate.
2.  **Add Timeouts:** Use `time.After` or context deadlines to avoid waiting forever.
3.  **Cancel Work:** Stop goroutines cooperatively using done channels or contexts.
4.  **Handle Backpressure:** Drop, buffer, or block intentionally when consumers are slow.

## 📖 Task Explanations

`select` is Go's switch for channel operations. It lets one goroutine wait on many communications at once, which is essential for responsive servers and pipelines.

Timeouts turn stuck goroutines into controlled failures. You'll see how deadlines prevent resource leaks when downstream systems hang.

Thinking about backpressure early helps you design services that degrade gracefully under load.

## 💡 What You'll Gain

By the end of today, you will multiplex channel operations with `select` and implement timeouts and cooperative cancellation.

## 📚 Dictionary for Developers

*   **select:** A control structure that waits on multiple channel operations, running the first ready case.
*   **Timeout:** A time limit after which an operation should stop waiting and return an error or fallback.
*   **Backpressure:** A mechanism slowing producers when consumers cannot keep up, preventing unbounded memory growth.
*   **Done Channel:** A channel closed to broadcast a shutdown signal to multiple goroutines.
*   **Cooperative Cancellation:** Design where goroutines periodically check for cancel signals instead of being force-killed.
