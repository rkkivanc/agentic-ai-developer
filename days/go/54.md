# Day 54: Auth & Security - Security Best Practices

## 📝 Today's Tasks

1.  **Validate All Input:** Treat client data as untrusted even after authentication.
2.  **Use HTTPS:** Terminate TLS in production and avoid sending tokens over plaintext.
3.  **Rate Limit:** Add basic rate limiting middleware to sensitive routes.
4.  **Audit Dependencies:** Run `govulncheck` and update modules regularly.

## 📖 Task Explanations

Security is a set of habits: validate input, least privilege, secure transport, and dependency hygiene.

HTTPS protects tokens and passwords on the wire. Rate limits slow brute-force attacks on login endpoints.

`govulncheck` scans your module graph for known vulnerabilities—run it regularly.

## 💡 What You'll Gain

By the end of today, you will apply foundational API security practices beyond authentication alone.

## 📚 Dictionary for Developers

*   **TLS/HTTPS:** Encrypted HTTP protecting data in transit from eavesdropping.
*   **Rate Limiting:** Restricting how many requests a client can make in a time window.
*   **govulncheck:** A Go tool reporting known vulnerabilities in dependencies.
*   **Least Privilege:** Granting only the permissions required for a task.
*   **Input Sanitization:** Ensuring user input is safe and expected before use.
