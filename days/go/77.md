# Day 77: Containers & CI/CD - Multi-Stage Builds and Optimization

## 📝 Today's Tasks

1.  **Cache Modules:** Copy `go.mod` and `go.sum` before source to leverage Docker layer cache.
2.  **Disable CGO if Possible:** Build static binaries for simpler containers when CGO is unnecessary.
3.  **Strip Debug Symbols:** Use `-ldflags` to reduce binary size for production images.
4.  **Scan Images:** Run a basic image vulnerability scan with available tools.

## 📖 Task Explanations

Docker layer caching speeds rebuilds—copy dependency files before source. Static Go binaries simplify deployment when CGO is off.

Smaller images deploy faster and reduce attack surface. Scan images for known CVEs as part of release hygiene.

Optimization matters once you deploy often.

## 💡 What You'll Gain

By the end of today, you will optimize Docker builds for speed, size, and basic security scanning.

## 📚 Dictionary for Developers

*   **Layer Cache:** Docker reusing unchanged build steps to speed image builds.
*   **CGO:** Enabling Go to call C libraries, complicating static cross-compilation.
*   **ldflags:** Linker flags passing options such as stripping debug info from binaries.
*   **Image Scan:** Checking container images for known vulnerabilities.
*   **Release Hygiene:** Practices keeping deployable artifacts safe and reproducible.
