# Day 14: Errors, Collections & I/O - JSON Encoding and Decoding

## 📝 Today's Tasks

1.  **Marshal Structs:** Convert structs to JSON with `json.Marshal` and inspect struct tags.
2.  **Unmarshal JSON:** Parse JSON into structs and handle unknown fields gracefully.
3.  **Use json.Encoder/Decoder:** Stream JSON from readers and writers for HTTP-sized payloads.
4.  **Model Optional Fields:** Represent optional JSON fields with pointers or `omitempty` tags.

## 📖 Task Explanations

JSON is the lingua franca of web APIs. Go's `encoding/json` package maps between JSON and structs using reflection and struct tags.

You'll learn how tags control field names and omission, and how pointers represent optional values. Streaming encoders and decoders become important when you build HTTP handlers in upcoming days.

Getting JSON right early saves painful debugging when clients send unexpected payloads.

## 💡 What You'll Gain

By the end of today, you will serialize and deserialize JSON confidently, including optional fields and streaming APIs.

## 📚 Dictionary for Developers

*   **JSON:** JavaScript Object Notation, a text format commonly used for HTTP APIs and configuration.
*   **Struct Tag:** Metadata on struct fields, such as `json:"name,omitempty"`, guiding encoding behavior.
*   **Marshal:** Converting a Go value into a serialized format like JSON bytes.
*   **Unmarshal:** Parsing serialized JSON into a Go value.
*   **omitempty:** A JSON tag option that omits a field when it holds a zero value.
