# Day 95: Team Practices & Tooling - Practice

## 📝 Today's Tasks

1.  **Polish Repo DX:** Add Makefile, improve README, ARCHITECTURE.md, and PR template.
2.  **Mock Code Review:** Review your own MVP as if you were a teammate using a checklist.
3.  **Tag v0.2.0:** Cut a release with notes reflecting recent features.
4.  **Identify Gaps:** List documentation or process gaps to fix post-100.

## 📖 Task Explanations

Today focuses on the human side of software—docs, reviews, releases. A polished repo signals professionalism.

Self-review with a checklist often finds issues you overlooked. Tagging a release practices the full delivery loop.

Note remaining gaps honestly for future improvement.

## 💡 What You'll Gain

By the end of today, your repository will read like a team-maintained project with releases and review habits.

## 📚 Dictionary for Developers

*   **PR Template:** A default pull request description guiding authors through required info.
*   **Release Tag:** A git tag marking a specific released version.
*   **Checklist Review:** Structured review ensuring important aspects are not missed.
*   **Professional Repository:** A project with docs, automation, and conventions welcoming collaborators.
*   **Process Gap:** Missing workflow or documentation that slows contributors down.
