# Day 9: Structs, Methods & Interfaces - Composition and Embedding

## 📝 Today's Tasks

1.  **Embed Structs:** Use anonymous struct embedding to promote fields and methods.
2.  **Compose Behavior:** Build larger types from smaller ones instead of using inheritance.
3.  **Override Methods:** Shadow embedded methods by defining methods on the outer struct.
4.  **Design a Small Domain:** Model entities like `User` and `Admin` using composition.

## 📖 Task Explanations

Go favors composition over inheritance. Embedding lets you reuse fields and methods by including one struct inside another, but the relationship remains explicit and simple.

You'll learn how promoted fields look like direct members and when to override embedded behavior. This pattern appears constantly in production Go, from HTTP middleware wrappers to repository decorators.

Designing a tiny domain today helps you see how structs, methods, and interfaces work together as a toolkit.

## 💡 What You'll Gain

By the end of today, you will be able to compose types with embedding and design small domain models without classical inheritance.

## 📚 Dictionary for Developers

*   **Composition:** Building complex types by combining simpler ones rather than inheriting from a base class.
*   **Embedding:** Including a struct anonymously inside another struct to promote its fields and methods.
*   **Promotion:** The mechanism by which embedded fields and methods become accessible on the outer type.
*   **Shadowing:** Defining a method on the outer type that hides an embedded type's method with the same name.
*   **Domain Model:** A code representation of business entities and their relationships.
