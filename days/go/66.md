# Day 66: Advanced Testing & Quality - Integration Testing

## 📝 Today's Tasks

1.  **Test HTTP End-to-End:** Spin up handlers with real repositories against a test database.
2.  **Use httptest.Server:** Call real HTTP URLs in integration tests when useful.
3.  **Seed Data:** Insert fixtures before tests and clean up after.
4.  **Tag Slow Tests:** Separate long integration tests with build tags if needed.

## 📖 Task Explanations

Integration tests verify layers together: HTTP, services, repositories, database. They catch wiring mistakes unit tests miss.

Seed minimal fixtures and clean up reliably. Slow tests are fine—label them so developers choose when to run.

A balanced pyramid still includes many unit tests; integration tests cover critical paths.

## 💡 What You'll Gain

By the end of today, you will write HTTP integration tests against real infrastructure components.

## 📚 Dictionary for Developers

*   **End-to-End Test:** A test covering multiple layers from API boundary to storage.
*   **Fixture Data:** Predefined rows or objects inserted to establish known test state.
*   **Build Tag:** A compile-time label excluding tests unless explicitly enabled.
*   **Test Pyramid:** A model favoring many unit tests and fewer broad integration tests.
*   **Wiring Bug:** A mistake connecting components, often caught only by integration tests.
