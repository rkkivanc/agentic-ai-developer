# Day 63: gRPC & Protocol Buffers - Client and Server Implementation

## 📝 Today's Tasks

1.  **Implement Server:** Register a service implementation on a gRPC server.
2.  **Implement Client:** Call RPCs from a client with connection setup and teardown.
3.  **Map Errors:** Return `status.Error` with appropriate gRPC codes.
4.  **Share Domain Logic:** Reuse service layer code from HTTP handlers where possible.

## 📖 Task Explanations

Implement servers by satisfying generated interfaces; clients dial, call RPCs, and handle errors. Map domain failures to gRPC status codes analogously to HTTP mapping.

Reuse inner service layers so HTTP and gRPC are transports, not duplicate business logic.

Run client and server locally to build intuition before deploying.

## 💡 What You'll Gain

By the end of today, you will run a gRPC server and client sharing core domain logic.

## 📚 Dictionary for Developers

*   **gRPC Server:** A process registering service implementations and listening for RPCs.
*   **gRPC Client:** A program dialing a server and invoking RPC methods.
*   **status.Code:** A gRPC error classification such as `NotFound` or `InvalidArgument`.
*   **Dial:** Opening a client connection to a gRPC server address.
*   **Transport:** The mechanism carrying RPCs or HTTP, separate from business logic.
