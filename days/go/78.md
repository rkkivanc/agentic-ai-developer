# Day 78: Containers & CI/CD - GitHub Actions Basics

## 📝 Today's Tasks

1.  **Create Workflow File:** Add `.github/workflows/ci.yml` triggered on push and pull request.
2.  **Setup Go Action:** Use `actions/setup-go` with your module's Go version.
3.  **Cache Modules:** Enable module caching to speed CI runs.
4.  **Run Tests:** Execute `go test ./...` on each push.

## 📖 Task Explanations

CI automates feedback on every change. GitHub Actions is a common choice for Go projects on GitHub.

Pin Go versions to match production. Cache modules and build artifacts to keep pipelines fast.

Start simple: test on push, expand later.

## 💡 What You'll Gain

By the end of today, you will have a GitHub Actions workflow running Go tests on each push.

## 📚 Dictionary for Developers

*   **GitHub Actions:** GitHub's CI/CD platform running workflows defined in YAML.
*   **Workflow:** An automated pipeline triggered by repository events.
*   **actions/setup-go:** A GitHub Action installing a specific Go toolchain version.
*   **Module Cache:** Stored dependencies speeding subsequent CI runs.
*   **Pull Request:** A proposal to merge changes, often gated by CI checks.
