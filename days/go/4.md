# Day 4: Go Fundamentals - Pointers and Memory Basics

## 📝 Today's Tasks

1.  **Understand Addresses:** Use `&` to take addresses and `*` to dereference values through pointers.
2.  **Pass by Value vs Pointer:** Compare functions that accept values versus pointers and observe mutation behavior.
3.  **Use new:** Allocate values with `new` and contrast it with taking the address of a composite literal.
4.  **Avoid Nil Panics:** Check for `nil` before dereferencing pointers in small exercises.

## 📖 Task Explanations

Go gives you pointers without pointer arithmetic, which keeps memory manipulation safe while still allowing efficient sharing of data. Understanding when to pass a pointer versus a copy is one of the most practical skills in Go.

You'll see how pointers let functions mutate state in place, which matters for structs, slices, and maps. You'll also learn that `nil` pointers are valid but dangerous if dereferenced without checks.

This day connects language fundamentals to later topics like methods with pointer receivers and efficient API design.

## 💡 What You'll Gain

By the end of today, you will understand how pointers work in Go, when to use them, and how to avoid common nil-related mistakes.

## 📚 Dictionary for Developers

*   **Pointer:** A variable that holds the memory address of another value. Declared with `*` in types and created with `&`.
*   **Dereference:** Accessing the value stored at a pointer's address using the `*` operator.
*   **Pass by Value:** Copying a value into a function parameter; mutations do not affect the caller's variable.
*   **Pass by Reference (via pointer):** Passing a pointer so the function can modify the original value.
*   **nil:** The zero value for pointers, slices, maps, channels, interfaces, and function values.
