# Day 79: Containers & CI/CD - CI Pipelines for Go

## 📝 Today's Tasks

1.  **Add Lint Job:** Run golangci-lint in CI and fail on issues.
2.  **Add Vet and Build:** Run `go vet` and `go build ./...` for compile safety.
3.  **Optional Integration Job:** Run integration tests with services containers in CI.
4.  **Upload Coverage:** Publish coverage summaries as CI artifacts or comments.

## 📖 Task Explanations

Mature Go CI runs tests, vet, build, and lint on every change. Integration jobs are slower—mark them clearly.

Coverage artifacts help reviewers see test impact. Keep pipelines fast enough that developers wait willingly.

CI is part of the product, not an afterthought.

## 💡 What You'll Gain

By the end of today, you will extend CI with lint, build, and optional integration coverage reporting.

## 📚 Dictionary for Developers

*   **Pipeline Stage:** A step in CI such as test, lint, or deploy with its own job.
*   **Artifact:** A file saved from CI runs, such as coverage output or binaries.
*   **Compile Safety:** Ensuring all packages build even if tests are skipped locally.
*   **Gating:** Blocking merges when required CI checks fail.
*   **Feedback Loop:** How quickly developers learn whether a change is safe.
