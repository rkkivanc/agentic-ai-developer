# Day 64: gRPC & Protocol Buffers - Interceptors and Metadata

## 📝 Today's Tasks

1.  **Server Interceptors:** Add logging and auth checks before RPC handlers run.
2.  **Client Interceptors:** Attach metadata like request IDs on outgoing calls.
3.  **Propagate Metadata:** Pass auth or tracing headers across service boundaries.
4.  **Test Interceptors:** Verify rejected unauthenticated calls in tests.

## 📖 Task Explanations

gRPC interceptors mirror HTTP middleware. They handle logging, auth, metrics, and retries around RPC handlers.

Metadata carries cross-cutting headers analogously to HTTP. Propagation is essential for tracing and security contexts.

Test interceptors—auth bugs in RPCs are as serious as in REST.

## 💡 What You'll Gain

By the end of today, you will implement gRPC interceptors and propagate metadata across calls.

## 📚 Dictionary for Developers

*   **Interceptor:** gRPC middleware running before or after RPC handlers.
*   **Metadata:** Key/value pairs attached to gRPC calls, similar to HTTP headers.
*   **Propagation:** Forwarding metadata or context across service boundaries.
*   **Unary Interceptor:** An interceptor handling single-request RPCs.
*   **Auth Context:** Identity information attached to a request for authorization decisions.
