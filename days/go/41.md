# Day 41: Databases (I) - database/sql Introduction

## 📝 Today's Tasks

1.  **Open a DB Connection:** Connect to PostgreSQL or SQLite with the right driver import.
2.  **Ping the Database:** Verify connectivity on startup and fail fast if unreachable.
3.  **Query Rows:** Run `SELECT` queries with `QueryContext` and scan into variables.
4.  **Defer Close:** Close rows and connections diligently to avoid leaks.

## 📖 Task Explanations

`database/sql` is Go's standard SQL interface. Drivers plug in beneath it; your code depends on the interface, not a specific database brand.

Always use context-aware methods in servers. Scanning rows into variables is verbose but explicit—ORMs come later as optional tools, not replacements for understanding SQL.

Closing rows and connections is non-negotiable in long-running processes.

## 💡 What You'll Gain

By the end of today, you will connect to a SQL database and query rows with `database/sql`.

## 📚 Dictionary for Developers

*   **database/sql:** Go's generic interface for SQL databases and connection pooling.
*   **Driver:** A package implementing `database/sql` for a specific database like Postgres.
*   **Scan:** Copying column values from a row into Go variables.
*   **Connection Pool:** A cache of database connections managed by `database/sql`.
*   **Ping:** A lightweight check verifying the database is reachable.
