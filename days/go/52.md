# Day 52: Auth & Security - JWT and Sessions

## 📝 Today's Tasks

1.  **Issue JWTs:** Sign tokens with a server secret and reasonable expiration.
2.  **Validate JWTs:** Parse and verify signatures on protected requests.
3.  **Store Secrets Safely:** Load signing keys from environment variables.
4.  **Compare Sessions:** Summarize trade-offs between stateless JWTs and server-side sessions.

## 📖 Task Explanations

JWTs are popular for stateless APIs. They carry claims and expire, but revocation is harder than server-side sessions.

Validate algorithms and signatures—never accept `none`. Keep secrets out of source control and rotate them with a plan.

Understanding session trade-offs helps you choose appropriately per product.

## 💡 What You'll Gain

By the end of today, you will issue and validate JWTs securely and articulate session trade-offs.

## 📚 Dictionary for Developers

*   **JWT (JSON Web Token):** A signed token encoding claims used for stateless authentication.
*   **Claim:** A key/value assertion inside a JWT, such as user ID or role.
*   **Expiration:** A time limit after which a token must be rejected.
*   **Signing Key:** A secret used to create and verify token signatures.
*   **Stateless Auth:** Authentication without server-side session storage, using self-contained tokens.
