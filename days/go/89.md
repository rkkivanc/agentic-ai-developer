# Day 89: Performance - Database and HTTP Performance

## 📝 Today's Tasks

1.  **Explain Query Plans:** Use EXPLAIN on slow SQL and add missing indexes.
2.  **Batch Writes:** Group inserts or updates to reduce round trips.
3.  **HTTP Keep-Alive:** Reuse connections on clients with proper transport settings.
4.  **Set Timeouts Everywhere:** Prevent slow dependencies from stalling handlers.

## 📖 Task Explanations

Many slowness issues live in databases and network calls, not CPU. EXPLAIN shows whether indexes are used; batching reduces latency multiplication.

HTTP keep-alive lowers TLS and TCP overhead for chatty clients. Timeouts prevent one slow query from freezing requests.

Measure query latency separately from handler latency.

## 💡 What You'll Gain

By the end of today, you will optimize database and HTTP client performance using plans, batching, and timeouts.

## 📚 Dictionary for Developers

*   **EXPLAIN:** A SQL command showing how the database executes a query.
*   **Index Scan:** Reading rows via an index, usually faster than full table scans for selective queries.
*   **Batching:** Grouping multiple operations into fewer round trips.
*   **Keep-Alive:** Reusing TCP connections for multiple HTTP requests.
*   **Round Trip:** A full request-response cycle to the database or remote service.
