# Day 61: gRPC & Protocol Buffers - Protocol Buffers Basics

## 📝 Today's Tasks

1.  **Install protoc:** Set up the protobuf compiler and Go plugins.
2.  **Write a .proto File:** Define messages and package options for your domain.
3.  **Generate Go Code:** Run `protoc` to generate structs and serialization code.
4.  **Serialize Messages:** Marshal and unmarshal protobuf messages in a small program.

## 📖 Task Explanations

Protocol Buffers define schema-first contracts efficient on the wire. They are common in microservices alongside JSON HTTP.

`.proto` files are the source of truth; generated Go keeps types aligned with other languages. Learn generation commands early—they become part of your build.

Protobuf complements REST; it does not replace everything.

## 💡 What You'll Gain

By the end of today, you will define protobuf messages and generate Go code from `.proto` files.

## 📚 Dictionary for Developers

*   **Protocol Buffers:** A language-neutral serialization format with schema definitions in `.proto` files.
*   **protoc:** The Protocol Buffers compiler generating code for multiple languages.
*   **Message:** A structured data type defined in a `.proto` file.
*   **Schema-First:** Defining data contracts before implementation code.
*   **Serialization:** Converting structured data to bytes for storage or transport.
