# Day 38: Context, Config & Middleware - Middleware Chaining

## 📝 Today's Tasks

1.  **Context Middleware:** Attach request IDs and loggers to context for downstream use.
2.  **Config Middleware:** Inject shared dependencies like config and stores into handlers.
3.  **Compose Chains:** Build a helper that orders middleware predictably.
4.  **Test Middleware:** Use `httptest` to assert headers, status codes, and side effects.

## 📖 Task Explanations

Middleware is where cross-cutting setup happens: request IDs, authentication placeholders, metrics, and shared dependencies. Ordering matters—recovery should wrap handlers early; auth before business logic.

Inject dependencies via constructor functions or middleware rather than global variables. Tests should cover middleware behavior, not only final handlers.

Clean chaining helpers keep `main` readable as middleware grows.

## 💡 What You'll Gain

By the end of today, you will chain middleware that enriches context and injects dependencies with tests.

## 📚 Dictionary for Developers

*   **Dependency Injection:** Providing dependencies from the outside rather than hard-coding globals.
*   **Request Scoped:** Values tied to a single HTTP request, not shared across requests.
*   **Middleware Order:** The sequence wrappers run in, affecting logging, auth, and recovery.
*   **Constructor:** A function building handlers or services with required dependencies.
*   **Global State:** Package-level variables accessible everywhere, discouraged for request-specific data.
