# Day 8: Structs, Methods & Interfaces - Interfaces and Polymorphism

## 📝 Today's Tasks

1.  **Define Small Interfaces:** Create interfaces with one or two methods following Go's preference for narrow contracts.
2.  **Implement Implicitly:** Make types satisfy interfaces without explicit `implements` declarations.
3.  **Use Interface Values:** Store concrete types behind interface variables and call methods dynamically.
4.  **Handle nil Interfaces:** Learn the difference between a nil interface value and an interface holding a nil concrete value.

## 📖 Task Explanations

Interfaces in Go are satisfied implicitly. If a type has the required methods, it implements the interface—no ceremony required. This encourages small, focused interfaces like `io.Reader` and `fmt.Stringer`.

Polymorphism in Go is compile-time safe and runtime flexible. You'll pass different concrete types through the same interface-shaped APIs, which is the foundation of testable service design.

Be careful with nil interfaces; they are a common source of subtle bugs in production code.

## 💡 What You'll Gain

By the end of today, you will understand Go's implicit interfaces and be able to write polymorphic functions that accept multiple concrete types.

## 📚 Dictionary for Developers

*   **Interface:** A type defined by a set of method signatures. Any type with those methods implements it.
*   **Implicit Implementation:** Go's approach where types satisfy interfaces automatically when they have the required methods.
*   **Polymorphism:** The ability to treat different concrete types uniformly through a shared interface.
*   **Concrete Type:** A specific struct or other type, as opposed to an interface abstraction.
*   **Nil Interface Gotcha:** An interface value can be non-nil while the dynamic value inside it is nil, affecting type assertions and comparisons.
