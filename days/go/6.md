## 📖 Task Explanations

Structs are Go's primary way to group related data. Unlike classes in some languages, structs are lightweight value types you compose into larger designs.

You'll learn several initialization styles and when each is clearest. Structs become the backbone of APIs, database models, and configuration objects throughout this curriculum.

Today focuses on data modeling. Tomorrow you'll add behavior with methods and receivers.

## 💡 What You'll Gain

By the end of today, you will be able to define structs, initialize them safely, and model simple domain entities in idiomatic Go.

## 📚 Dictionary for Developers

*   **Struct:** A composite type that groups named fields under a single type.
*   **Field:** A named member of a struct that holds a value of a specific type.
*   **Composite Literal:** A concise syntax for creating struct, slice, map, or array values inline.
*   **Zero Value:** The default value Go assigns when a variable is declared without an explicit initializer.
*   **Value Type:** A type whose assignment copies the entire value, such as structs by default.
