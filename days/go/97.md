# Day 97: Capstone & Professional Delivery - Building Core Features

## 📝 Today's Tasks

1.  **Implement Core APIs:** Build the primary use cases from your capstone plan.
2.  **Wire Persistence:** Connect repositories and migrations for capstone data.
3.  **Add Auth Where Needed:** Protect sensitive capstone routes appropriately.
4.  **Keep Tests Green:** Add unit and integration tests as you implement.

## 📖 Task Explanations

Execution day: turn plans into working code. Focus on core user journeys first—happy paths with tests—before optional polish.

Wire persistence and auth early so you do not bolt them on fragile prototypes. Green tests give confidence to refactor.

Ship vertical slices, not half-layers.

## 💡 What You'll Gain

By the end of today, your capstone will have working core features with persistence, auth, and tests.

## 📚 Dictionary for Developers

*   **Vertical Slice:** Implementing one feature through all layers end to end.
*   **Core Journey:** The most important user or client flow your service must support.
*   **Green Tests:** A test suite passing completely, enabling safe changes.
*   **Incremental Delivery:** Shipping small complete pieces rather than big-bang integration.
*   **Use Case:** A specific behavior the system provides to its users or clients.
