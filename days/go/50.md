# Day 50: Databases (II) & Repositories - Practice Project & Review

## 📝 Today's Tasks

1.  **Refactor to Repositories:** Ensure all persistence goes through tested repository interfaces.
2.  **Organize SQL:** Group queries and remove duplicated statements.
3.  **Add Integration Tests:** Cover happy and not-found paths for each repository method.
4.  **Review Data Layer:** Document schema, migrations, pool settings, and test instructions.

## 📖 Task Explanations

Today strengthens the data layer like Flutter Day 50 strengthens API integration. Refactor until handlers contain no raw SQL.

Integration tests become part of your standard workflow. Document how to run migrations and tests so collaborators succeed on day one.

A clean data layer is a portfolio highlight—show it with confidence.

## 💡 What You'll Gain

By the end of today, you will have a well-tested repository layer with organized SQL and clear documentation.

## 📚 Dictionary for Developers

*   **Data Layer Review:** A pass over persistence code for clarity, safety, and test coverage.
*   **Duplicated SQL:** Repeated query strings that should be centralized to reduce drift.
*   **Test Instructions:** Documentation explaining how to run automated tests locally and in CI.
*   **Portfolio Quality:** Code organized and documented well enough to show employers.
*   **Separation of Concerns:** Keeping HTTP, business logic, and SQL in distinct layers.
