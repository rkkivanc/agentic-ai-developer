# Day 28: Modules, Tooling & tsconfig - Path Aliases and Project References

## 📝 Today's Tasks

1.  **Configure paths:** Add `@/` or `@utils/*` aliases in `tsconfig` and use them in imports.
2.  **Sync Bundler Aliases:** Mirror path aliases in Vite, webpack, or tsx so runtime resolution matches.
3.  **Set Up Project References:** Create a multi-package workspace with `references` and `composite`.
4.  **Build with tsc --build:** Compile referenced projects in dependency order.

## 📖 Task Explanations

Path aliases improve import readability: `@/components/Button` beats `../../../components/Button`. TypeScript `paths` must be mirrored in your bundler or runtime tool—otherwise compilation succeeds but execution fails.

Project references enable monorepos: shared packages compile independently with incremental builds. `composite: true` and `references` array tell TypeScript about dependency graphs. `tsc --build` compiles packages in topological order, caching outputs.

These patterns appear in every medium-to-large TypeScript codebase. Learning them now prevents painful migrations later.

## 💡 What You'll Gain

By the end of today, you will configure path aliases, align bundler resolution, and set up project references for incremental multi-package builds.

## 📚 Dictionary for Developers

*   **Path Mapping (`paths`):** tsconfig option mapping alias prefixes to filesystem locations.
*   **Project References:** A tsconfig feature linking dependent TypeScript projects.
*   **`composite`:** Enables a project to be referenced and built incrementally.
*   **`tsc --build`:** Build mode compiling projects and dependencies in order.
*   **Monorepo:** A repository containing multiple related packages or apps.
