# Day 25: Advanced Types (I) - Practice

## 📝 Today's Tasks

1.  **Code Challenge 1: API Layer Types:** From a `DbUser` type, derive `PublicUser`, `CreateUserInput`, and `UpdateUserInput` using utilities only.
2.  **Code Challenge 2: Typed Config Map:** Build `Record<Environment, Config>` where `Environment` is `'dev' | 'staging' | 'prod'`.
3.  **Code Challenge 3: Wrapper Types:** Write `withLogging` preserving `Parameters` and `ReturnType` of any async function.
4.  **Review:** Audit one of your earlier projects for duplicated types replaceable with `Pick`/`Omit`/`Partial`.

## 📖 Task Explanations

Practice day for Advanced Types I. The API layer challenge mirrors real backend work: database models carry fields clients must never see. Deriving `PublicUser` with `Omit` and `CreateUserInput` with `Omit` plus `Partial` keeps layers synchronized when `DbUser` evolves.

The config map enforces exhaustive environment keys—adding `'preview'` without updating the `Record` should error. The logging wrapper applies `Parameters` and `ReturnType` together, a pattern seen in middleware and instrumentation.

The review step reinforces that utility types are not academic—they reduce maintenance in every typed codebase.

## 💡 What You'll Gain

By the end of today, you will derive multi-layer API types from a single source, build exhaustive config maps, and write type-preserving wrappers—production patterns, not textbook exercises.

## 📚 Dictionary for Developers

*   **Layered Types:** Distinct types per architectural layer derived from a canonical model.
*   **Exhaustive Mapping:** Ensuring every key in a union has a corresponding entry.
*   **Type-Preserving Wrapper:** A function wrapper maintaining the wrapped function's signature.
*   **Type Audit:** Reviewing code for redundant definitions replaceable by utilities.
*   **Maintenance Reduction:** Fewer places to update when domain shapes change.
