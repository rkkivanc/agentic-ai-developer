## 📖 Task Explanations

`tsconfig.json` is the control panel for every TypeScript project. `target` and `lib` determine emitted JavaScript version and available APIs. `rootDir`/`outDir` shape your output folder structure. `strict` bundles safety flags you should rarely disable.

`include` and `exclude` glob patterns define compilation scope—misconfigured globs cause "file not found" or accidentally compiled test fixtures. Extending a shared base config (`"extends": "@repo/tsconfig/base.json"`) keeps monorepo packages consistent.

Professional developers do not memorize every flag—they know where to look and which options affect their deployment target. Today builds that mental map.

## 💡 What You'll Gain

## 📚 Dictionary for Developers

*   **`target`:** The ECMAScript version TypeScript emits.
*   **`lib`:** Built-in type definitions included (DOM, ES2022, etc.).
*   **`rootDir` / `outDir`:** Source root and output directory for compiled files.
*   **`extends`:** Inheriting options from another tsconfig file.
*   **`skipLibCheck`:** Skipping type checking of declaration files for faster builds.
