# Day 23: Advanced Types (I) - ReturnType and Parameters

## 📝 Today's Tasks

1.  **Extract ReturnType:** Infer a function's return type with `ReturnType<typeof myFn>` instead of duplicating it.
2.  **Extract Parameters:** Use `Parameters<typeof myFn>` to type wrappers delegating to an existing function.
3.  **Type a Higher-Order Function:** Wrap an async API call and preserve its return type automatically.
4.  **Handle Overloads:** Observe how `ReturnType` behaves with overloaded functions and document the result.

## 📖 Task Explanations

Functions are types too, and TypeScript can extract their signatures. `ReturnType<typeof fetchUser>` gives you the resolved type without manually copying it—when `fetchUser` changes, dependents update automatically. `Parameters<typeof fn>` captures the argument tuple, useful for middleware, decorators, and test mocks.

These utilities shine in higher-order functions: a `withRetry` wrapper should return the same type as the function it wraps. Hardcoding return types creates drift; extracting them keeps wrappers honest.

Overloaded functions resolve to the last overload signature for `ReturnType`—a subtlety worth knowing when reading advanced library types.

## 💡 What You'll Gain

By the end of today, you will extract function return and parameter types to build wrappers and mocks that stay synchronized with the functions they depend on.

## 📚 Dictionary for Developers

*   **`ReturnType<T>`:** Extracts the return type of a function type.
*   **`Parameters<T>`:** Extracts parameter types as a tuple from a function type.
*   **`typeof` (value context):** Obtains the type of a value, e.g. `typeof myFn` for function typing.
*   **Higher-Order Function:** A function that takes or returns another function.
*   **Signature Extraction:** Deriving types from an existing function instead of duplicating them.
