# Day 20: Generics - Practice

## 📝 Today's Tasks

1.  **Code Challenge 1: Typed Event Emitter:** Build `EventEmitter<Events>` where `Events` maps event names to payload types.
2.  **Code Challenge 2: Safe JSON Parse:** Create `parseJson<T>(text: string, guard: (u: unknown) => u is T): T` with a runtime guard.
3.  **Code Challenge 3: Generic Pipeline:** Write a `pipe` function chaining two generic transformations with preserved types.
4.  **Review:** List three places in real projects where generics prevent bugs that `any` would hide.

## 📖 Task Explanations

Practice day for generics. The event emitter challenge is a classic: `on('click', handler)` should infer payload types from a central `Events` map—mirroring Node's `EventEmitter` and browser custom events. The safe JSON parse combines generics with type guards from Day 9, foreshadowing Zod on Days 51–55.

The pipeline challenge tests composing generics so output of step one becomes input of step two without widening to `any`. If you get stuck, sketch types on paper before writing implementation.

The review step connects exercises to professional impact: generics catch mistakes in API clients, state management, and data layers at compile time.

## 💡 What You'll Gain

By the end of today, you will have built generic abstractions—a typed event system, safe parser, and pipeline—and articulate why generics matter in production TypeScript.

## 📚 Dictionary for Developers

*   **Typed Event Map:** An object type mapping event names to their payload shapes.
*   **Runtime Type Guard:** A function narrowing `unknown` to a specific type at runtime.
*   **Generic Pipeline:** A chain of transformations preserving types between stages.
*   **Type Widening:** When TypeScript expands a type (often to `any`) losing precision.
*   **Compile-Time Safety:** Catching invalid event names, keys, or shapes before deployment.
