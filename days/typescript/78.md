# Day 78: Strictness, Lint & CI - Prettier and Formatting Consistency

## 📝 Today's Tasks

1.  **Add Prettier:** Configure Prettier alongside ESLint without rule conflicts.
2.  **Format on Save:** Enable editor format-on-save for TypeScript files.
3.  **Add format Script:** Create `npm run format` and optionally `format:check` for CI.
4.  **Resolve Style Debates:** Document that formatting is automated, not argued in review.

## 📖 Task Explanations

Formatting should never consume review energy. Prettier enforces consistent style—semicolons, quotes, line width—automatically. Use `eslint-config-prettier` to disable ESLint stylistic rules that conflict.

`format:check` in CI fails PRs with unformatted code—cheap enforcement. Format-on-save keeps local files clean before commit.

Teams ship faster when style is invisible and automatic.

## 💡 What You'll Gain

By the end of today, you will integrate Prettier with ESLint, automate formatting, and add CI-checkable format scripts.

## 📚 Dictionary for Developers

*   **Prettier:** An opinionated code formatter for consistent style.
*   **eslint-config-prettier:** Disables ESLint rules that conflict with Prettier.
*   **Format on Save:** Editor setting applying Prettier when files are saved.
*   **`format:check`:** A script verifying files are formatted without modifying them.
*   **Style Automation:** Removing manual formatting debates from code review.
