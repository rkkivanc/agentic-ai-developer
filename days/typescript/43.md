# Day 43: DOM & Browser TypeScript - Forms and Input Handling

## 📝 Today's Tasks

1.  **Type Form Elements:** Access `HTMLInputElement`, `HTMLSelectElement`, and `HTMLFormElement` correctly.
2.  **Handle FormData:** Extract typed values from forms with validation before use.
3.  **Prevent Default Safely:** Type `SubmitEvent` handlers calling `preventDefault()`.
4.  **Build a Typed Form State:** Model form state as an interface and sync with inputs.

## 📖 Task Explanations

Forms are where untyped JavaScript causes the most runtime surprises—empty strings, unchecked checkboxes, missing fields. TypeScript plus validation closes the gap. Each input type has a specific element interface with typed properties (`value`, `checked`, `selectedIndex`).

`FormData` returns `FormDataEntryValue | null`—still requiring validation before treating values as domain types. Modeling form state as a typed interface (`SignupForm`) and updating fields explicitly connects UI to domain types cleanly.

This day foreshadows Zod validation (Days 51–55) at the browser boundary.

## 💡 What You'll Gain

By the end of today, you will type form elements and events, extract values safely, and model form state with typed interfaces.

## 📚 Dictionary for Developers

*   **`HTMLInputElement`:** Typed interface for input elements with `value` and `checked`.
*   **`FormData`:** A browser API collecting form field values; types require narrowing.
*   **`SubmitEvent`:** The event fired on form submission with `preventDefault`.
*   **Form State Model:** A typed object representing current form field values.
*   **Input Validation:** Checking form values before treating them as domain data.
