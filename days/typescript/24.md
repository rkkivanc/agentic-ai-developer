# Day 24: Advanced Types (I) - Mapped Types Basics

## 📝 Today's Tasks

1.  **Read a Mapped Type:** Study `{ [K in keyof T]: T[K] }` and confirm it reconstructs `T`.
2.  **Build `Nullable<T>`:** Map every property to `T[K] | null` using a mapped type.
3.  **Add Modifiers:** Experiment with `+?`, `-?`, `+readonly`, `-readonly` in mapped types.
4.  **Connect to Utilities:** Explain how `Partial` is implemented with `-?` on each property.

## 📖 Task Explanations

Mapped types iterate over keys—`[K in keyof T]`—and transform each property. This is the machinery behind `Partial`, `Readonly`, and countless custom utilities in libraries. Once you read the syntax, built-in utilities become predictable instead of magical.

Modifier prefixes add or remove `optional` and `readonly` during the mapping. `Partial` maps `T[K]` to `T[K]` but adds `?` (via `-?` removal of required). Understanding implementation helps you debug complex type errors and write domain-specific transforms.

You are not expected to write elaborate mapped types daily, but recognizing the pattern unlocks advanced library typings and your own reusable transforms.

## 💡 What You'll Gain

By the end of today, you will read and write basic mapped types, understand modifier prefixes, and see how built-in utility types are implemented.

## 📚 Dictionary for Developers

*   **Mapped Type:** A type iterating `keyof T` to transform each property.
*   **Key Remapping:** Using `as` in mapped types to rename or filter keys (preview for later).
*   **Optional Modifier (`?`):** Marks properties optional; `-?` removes optionality during mapping.
*   **`Nullable<T>`:** A custom mapped type adding `| null` to every property.
*   **Type-Level Iteration:** Looping over keys at compile time, similar to runtime `for...in`.
