# Day 17: Generics - Generic Classes and Interfaces

## 📝 Today's Tasks

1.  **Create a Generic Class:** Build `Stack<T>` or `Box<T>` with push, pop, and typed storage.
2.  **Generic Interface:** Define `Cache<K, V>` with `get` and `set` methods using two type parameters.
3.  **Implement the Interface:** Provide a concrete in-memory cache class implementing your generic interface.
4.  **Instantiate with Types:** Create `Cache<string, User>` and verify incorrect key/value types error at compile time.

## 📖 Task Explanations

Generic classes and interfaces extend the same ideas from functions to stateful abstractions. A `Stack<number>` and `Stack<string>` share implementation logic but maintain distinct element types. This pattern appears in collections, caches, repositories, and state containers.

Multiple type parameters model relationships between keys and values, inputs and outputs, or entity types and identifiers. The compiler ensures you cannot store a `User` in a `Cache<string, Product>` without an explicit mistake.

Interfaces with type parameters define contracts implementors must satisfy at any instantiation. This is how libraries expose flexible APIs while remaining type-safe for consumers.

## 💡 What You'll Gain

By the end of today, you will design generic classes and interfaces, implement them concretely, and instantiate with precise type arguments for safe, reusable data structures.

## 📚 Dictionary for Developers

*   **Generic Class:** A class parameterized by one or more type variables.
*   **Multiple Type Parameters:** Generics like `<K, V>` relating keys to values or inputs to outputs.
*   **Instantiation:** Creating a concrete type from a generic, e.g. `Stack<number>`.
*   **Generic Interface:** An interface with type parameters defining a flexible contract.
*   **Type-Safe Collection:** A container where element types are enforced by the compiler.
