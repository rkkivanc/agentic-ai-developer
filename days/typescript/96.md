# Day 96: Capstone & Professional Delivery - Capstone Planning

## 📝 Today's Tasks

1.  **Choose Capstone Scope:** Pick a full-stack typed project (API + client, or CLI + library) demonstrating curriculum skills.
2.  **Write Requirements:** Document features, types, validation, tests, and CI expectations.
3.  **Architecture Diagram:** Sketch layers, shared packages, and trust boundaries.
4.  **Milestone Schedule:** Plan Days 96–100 deliverables (plan, build, test, polish, ship).

## 📖 Task Explanations

The capstone proves everything together—not a tutorial exercise but a coherent product. Choose scope you can finish: a task API with typed React or vanilla client, or a CLI with published typed library. Requirements should explicitly list Zod boundaries, shared types, tests, and CI—evaluators look for professional completeness.

Architecture diagram forces thinking before coding—where do schemas live, how do errors flow, what is exported. Milestones prevent Day 100 panic—each day has a clear deliverable.

This is your portfolio centerpiece—plan accordingly.

## 💡 What You'll Gain

By the end of today, you will have a scoped capstone plan with requirements, architecture, and a day-by-day delivery schedule.

*   **Capstone Project:** A culminating project demonstrating end-to-end TypeScript mastery.
*   **Requirements Doc:** Written feature and technical expectations for the capstone.
*   **Architecture Diagram:** Visual map of layers, packages, and data flow.
*   **Milestone:** A checkpoint deliverable on the path to completion.
