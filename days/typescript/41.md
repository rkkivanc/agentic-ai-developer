# Day 41: DOM & Browser TypeScript - lib.dom and Global Types

## 📝 Today's Tasks

1.  **Explore lib.dom:** Inspect types for `document`, `window`, and `HTMLElement` in your editor.
2.  **Type DOM Queries:** Use type assertions or generics with `querySelector` for specific elements.
3.  **Narrow Nullable Elements:** Handle `null` returns from DOM queries before accessing properties.
4.  **Configure lib:** Adjust `lib` in tsconfig when targeting Node vs browser environments.

## 📖 Task Explanations

Browser TypeScript relies on `lib.dom`—thousands of types for DOM APIs, events, and fetch. Understanding these types prevents fighting the compiler when manipulating elements. `document.querySelector` returns `Element | null`; narrowing is mandatory before calling element-specific methods.

Generic overloads like `querySelector<HTMLInputElement>('#email')` improve ergonomics when selectors are stable. Separating browser and Node `tsconfig` projects (different `lib` settings) prevents DOM types leaking into server code.

Frontend TypeScript is mostly applying narrowing and correct element types to APIs you already know from JavaScript.

## 💡 What You'll Gain

By the end of today, you will navigate `lib.dom` types, query elements with appropriate typings, and handle nullability in browser code safely.

## 📚 Dictionary for Developers

*   **`lib.dom`:** TypeScript's built-in DOM type definitions for browser APIs.
*   **`HTMLElement`:** The base type for HTML elements in the DOM.
*   **`querySelector`:** A method returning `Element | null` requiring narrowing.
*   **Environment-Specific lib:** Choosing DOM vs Node libs per project target.
*   **Null Narrowing:** Refining `T | null` to `T` before property access.
