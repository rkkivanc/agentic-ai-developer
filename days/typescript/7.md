# Day 7: Objects, Unions & Narrowing - Arrays and Tuples

## 📝 Today's Tasks

1.  **Type Arrays:** Use `Type[]` and `Array<Type>` syntax for homogeneous collections.
2.  **Work with Tuples:** Define fixed-length arrays where each position has a specific type.
3.  **Use `readonly` Arrays:** Prevent accidental mutation of array data.
4.  **Iterate with Type Safety:** Loop and transform arrays while preserving element types.

## 📖 Task Explanations

Arrays and tuples are the workhorses of collection typing. A `string[]` holds any number of strings; a tuple like `[string, number]` holds exactly one string and one number in order. Tuples are ideal for function returns that bundle multiple values—think `[Error | null, User]`.

Readonly arrays signal that consumers should not mutate the underlying data—a useful pattern for shared configuration or API results. When you `map` or `filter`, TypeScript tracks element types through the transformation, catching mistakes before runtime.

## 💡 What You'll Gain

By the end of today, you will type arrays and tuples confidently, choose between them appropriately, and iterate over collections without losing type information.

## 📚 Dictionary for Developers

*   **Array Type:** A type representing a collection of elements sharing the same type.
*   **Tuple:** A fixed-length array where each index has a specific, declared type.
*   **Homogeneous:** Containing elements of the same type.
*   **`readonly` Array:** An array type that disallows mutating methods like `push`.
*   **Element Type:** The type of individual items within an array.
