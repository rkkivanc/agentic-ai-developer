# Day 80: Strictness, Lint & CI - Practice

## 📝 Today's Tasks

1.  **Harden a Project:** Apply full strict flags, typescript-eslint, and Prettier to an existing repo.
2.  **Fix All CI Checks:** Ensure typecheck, lint, format:check, test, and build pass locally.
3.  **Add Pre-commit Hooks:** Optionally use husky/lint-staged to run checks before commits.
4.  **Document Standards:** Write `CONTRIBUTING.md` explaining how to run checks locally.

## 📖 Task Explanations

Practice day for strictness and CI. Hardening an existing project surfaces real debt—fix it systematically: types first, then lint, then format. Local `npm run ci` script mirroring pipeline commands lets developers verify before push.

Pre-commit hooks catch issues early but should not replace CI—developers can bypass hooks. CONTRIBUTING.md reduces onboarding friction—new contributors know exactly what to run.

Green CI is a deliverable, not an afterthought.

## 💡 What You'll Gain

By the end of today, you will have a fully hardened project with local and CI checks documented for contributors.

## 📚 Dictionary for Developers

*   **Hardening:** Applying strict compiler, lint, and format rules to existing code.
*   **Pre-commit Hook:** Scripts running lint/format/tests before a git commit.
*   **lint-staged:** Running checks only on staged files for speed.
*   **CONTRIBUTING.md:** Guide for contributors including local check commands.
*   **CI Parity Script:** A local npm script mirroring CI steps.
