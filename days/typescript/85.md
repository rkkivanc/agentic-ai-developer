# Day 85: Full-stack Contracts & Shared Types - Practice

## 📝 Today's Tasks

1.  **Monorepo Exercise:** Build a minimal monorepo with `apps/web`, `apps/api`, and `packages/shared`.
2.  **Shared Zod Schemas:** Validate API input on server and parse responses on client from shared schemas.
3.  **Typed Client SDK:** Export a typed client from shared package used by the web app.
4.  **CI Workspace Build:** Configure CI to build packages in dependency order.

## 📖 Task Explanations

Practice day for full-stack contracts. The three-package monorepo is the smallest realistic full-stack TypeScript setup. Shared Zod schemas are the contract; typed client is the consumer DX; CI build order proves it works in automation.

Breakage should surface when you add a required field to a schema—both apps fail until updated. That is success, not annoyance.

This exercise mirrors professional teams shipping typed products at scale.

## 💡 What You'll Gain

By the end of today, you will have a working monorepo with shared schemas, typed client, and CI building the full workspace.

## 📚 Dictionary for Developers

*   **Monorepo Layout:** Repository structure with `apps/` and `packages/` directories.
*   **Cross-Package Import:** Importing shared types via workspace package name.
*   **Workspace CI:** Pipeline building all packages in correct dependency order.
*   **Contract Breakage:** Intentional compile failures when shared types change.
*   **Full-Stack Monorepo:** Frontend and backend apps sharing types in one repository.
