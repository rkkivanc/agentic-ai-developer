# Day 44: DOM & Browser TypeScript - fetch and Typed Responses

## 📝 Today's Tasks

1.  **Type fetch Responses:** Parse JSON into typed interfaces with runtime validation awareness.
2.  **Handle HTTP Errors:** Check `response.ok` and model error bodies with discriminated unions.
3.  **Use Headers and Methods:** Type `RequestInit` for POST requests with JSON bodies.
4.  **Combine with AbortSignal:** Add cancellable fetch to a typed API client function.

## 📖 Task Explanations

`fetch` returns `Promise<Response>`—the first step in a typed HTTP client. `response.json()` returns `Promise<any>` by default; you must assert, validate, or parse into typed shapes. Checking `response.ok` separates success from HTTP error statuses before parsing bodies.

`RequestInit` types method, headers, body, and signal—enabling fully typed POST helpers. Combining fetch with `AbortSignal` from Day 39 makes clients cancellable, essential for UI responsiveness.

Browser fetch patterns mirror Node HTTP work on Days 48–50, reinforcing cross-environment TypeScript skills.

## 💡 What You'll Gain

By the end of today, you will build typed fetch wrappers handling success and error responses, POST JSON safely, and support cancellation.

## 📚 Dictionary for Developers

*   **`Response`:** The fetch API result type with `ok`, `status`, and `json()`.
*   **`RequestInit`:** Typed options for configuring fetch requests.
*   **HTTP Error Body:** A typed structure representing API error responses.
*   **JSON Parsing:** Converting response text to typed objects—ideally with validation.
*   **API Client:** A typed wrapper around fetch for a specific backend.
