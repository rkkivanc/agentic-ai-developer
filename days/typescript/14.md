2.  **Merge Interface Declarations:** Declare the same interface twice and observe merged members in your editor.
3.  **Augment a Module:** Use declaration merging to extend a third-party interface in a `.d.ts` file.
4.  **Contrast with Nominal Thinking:** Write a short comment explaining why TypeScript is structural, not nominal.

## 📖 Task Explanations

Structural typing is TypeScript's defining characteristic: if it looks like a duck, it is a duck. Two types with the same shape are compatible even if named differently. This enables flexible APIs but surprises developers from nominally typed languages where names must match explicitly.

Declaration merging lets you split interface definitions across files—powerful for augmenting libraries. When you `declare module 'express'` and add fields to `Request`, you are using merging. Understanding this prevents confusion when reading DefinitelyTyped packages and writing ambient declarations.

Today connects object types, classes, and real-world library integration. You stop thinking of types as labels and start thinking of them as shapes the compiler verifies.

## 💡 What You'll Gain

## 📚 Dictionary for Developers

*   **Structural Typing:** Compatibility based on shape, not explicit type names.
*   **Nominal Typing:** A system where type names determine compatibility—TypeScript is not nominal.
*   **Declaration Merging:** Combining multiple declarations of the same interface into one type.
*   **Ambient Declaration:** A `.d.ts` file describing types without emitting JavaScript.
*   **Module Augmentation:** Extending types exported by an existing module via merging.
