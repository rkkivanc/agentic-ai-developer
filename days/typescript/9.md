# Day 9: Objects, Unions & Narrowing - Type Narrowing and Guards

## 📝 Today's Tasks

1.  **Use `typeof` Guards:** Narrow primitives inside `if` blocks.
2.  **Use `in` Operator Guards:** Distinguish object shapes in unions.
3.  **Write a Custom Type Guard:** Create `function isUser(x): x is User` predicates.
4.  **Handle `never` and Exhaustiveness:** Use `never` to catch unhandled union cases at compile time.

## 📖 Task Explanations

Narrowing is how you turn broad types into actionable ones. After `typeof value === 'string'`, TypeScript knows `value` is a string in that branch. The `in` operator checks for property presence to distinguish object variants.

Custom type guards are functions that return `value is SomeType`—a boolean that also tells the compiler the type if true. They encapsulate validation logic reusable across your codebase. Exhaustiveness checking with `never` ensures your `switch` handles every case; if you add a new union member, the compiler flags missing branches.

## 💡 What You'll Gain

By the end of today, you will eliminate unsafe casts by narrowing types with built-in and custom guards. Your code will handle every case explicitly, with the compiler verifying completeness.

## 📚 Dictionary for Developers

*   **Type Guard:** A runtime check that refines a type for TypeScript in a conditional branch.
*   **`typeof`:** An operator returning a string describing a primitive's runtime type.
*   **`in` Operator:** Checks whether an object property exists—useful for narrowing object unions.
*   **`never`:** A type representing values that should never occur; used for exhaustiveness checks.
*   **Exhaustiveness Check:** Ensuring all cases of a union are handled, often via `never` in a default branch.
