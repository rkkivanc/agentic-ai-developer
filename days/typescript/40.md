# Day 40: Async, Promises & Errors - Async Integration Patterns

## 📝 Today's Tasks

1.  **Retry with Backoff:** Implement a typed retry helper for transient failures.
2.  **Debounce an Async Call:** Prevent duplicate in-flight requests with a debounce wrapper.
3.  **Graceful Shutdown:** In a small Node script, handle `SIGINT` and drain pending promises.
4.  **Compose Patterns:** Combine Result + retry + abort in one realistic fetch workflow.

## 📖 Task Explanations

Production async code layers patterns: retry with exponential backoff for flaky networks, debounce for search inputs, graceful shutdown for servers finishing in-flight work. Each pattern must preserve types—retries return the same `T`, not `any`.

Composing Result types with retry and abort produces realistic workflows mirroring production API clients. Today ties Days 36–39 into integration patterns you will reuse in Node (Days 46–50) and full-stack work.

Async TypeScript is not just syntax—it is reliability engineering with types.

## 💡 What You'll Gain

By the end of today, you will implement retry, debounce, and shutdown patterns with preserved types, composing them into realistic async workflows.

## 📚 Dictionary for Developers

*   **Exponential Backoff:** Increasing delay between retries to reduce load on failing services.
*   **Debounce:** Delaying execution until input stops changing for a period.
*   **Graceful Shutdown:** Finishing in-flight work before process exit.
*   **Transient Failure:** A temporary error worth retrying (timeout, 503).
*   **Async Composition:** Combining multiple async patterns in one typed workflow.
