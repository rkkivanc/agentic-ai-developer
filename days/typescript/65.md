# Day 65: Advanced Types (II) - Practice

## 📝 Today's Tasks

1.  **Code Challenge 1: Type-Safe Path:** Build `Get<T, Path>` extracting nested properties via string path unions.
2.  **Code Challenge 2: Event Map:** Generate listener method types from an event map using template literals.
3.  **Code Challenge 3: Deep Readonly:** Implement a one-level or recursive `DeepReadonly<T>` type.
4.  **Review:** Add expect-type tests for each challenge asserting correct inference.

## 📖 Task Explanations

Practice day for advanced types. Path extraction types (`Get<User, 'address.city'>`) appear in configuration and form libraries. Event map generation combines template literals with keyof—mirroring typed event emitters from Day 20 at the type level.

Deep Readonly teaches recursive conditional and mapped patterns. Type tests are mandatory here—advanced types are too easy to get subtly wrong without `expect-type` assertions.

Do not rush—draw type expansions on paper when the compiler errors confuse you.

## 💡 What You'll Gain

By the end of today, you will have built advanced type utilities with type tests proving correctness—library-author-level type-level programming practice.

## 📚 Dictionary for Developers

*   **Path Union:** A union of dot-notation strings representing nested object paths.
*   **Type-Level Get:** A conditional type extracting a nested property type by path.
*   **Generated Methods:** Method types derived from keys via template literals.
*   **Deep Readonly:** A recursive type making all nested properties readonly.
*   **Type-Level Test:** expect-type assertions verifying advanced type behavior.
