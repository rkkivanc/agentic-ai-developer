# Day 2: TypeScript Fundamentals - Types and Annotations

## 📝 Today's Tasks

1.  **Learn Primitive Types:** Work with `string`, `number`, `boolean`, `null`, and `undefined` in typed variables.
2.  **Add Explicit Annotations:** Declare variable types explicitly and compare them to inferred types.
3.  **Explore `any` vs `unknown`:** Understand why `any` disables checking and why `unknown` is the safer escape hatch.
4.  **Use Type Assertions Sparingly:** Learn `as` syntax and when narrowing is preferable to casting.

## 📖 Task Explanations

Today you learn how TypeScript attaches meaning to data. Primitive types mirror JavaScript values but let the compiler catch mistakes early—like passing a string where a number is expected. Explicit annotations make contracts visible in your code; inference lets TypeScript deduce types when the value is obvious.

The distinction between `any` and `unknown` is a professional habit worth building early. `any` turns off the type checker and hides bugs. `unknown` forces you to narrow or validate before use, keeping safety intact. Reach for `unknown` at system boundaries; avoid `any` unless you are incrementally migrating legacy code.

Type assertions (`value as string`) tell the compiler you know more than it does. They are occasionally necessary but can mask real errors. Prefer control-flow narrowing—which you will deepen on Day 9—over blind casting.

## 💡 What You'll Gain

By the end of today, you will confidently annotate variables and understand when TypeScript infers types for you. You will know why `unknown` is safer than `any` and when assertions are appropriate. This is the vocabulary for every typed function and object you write next.

## 📚 Dictionary for Developers

*   **Type Annotation:** Explicit syntax (`let age: number`) that declares what type a variable may hold.
*   **Type Inference:** The compiler's ability to deduce a type from context without an explicit annotation.
*   **Primitive Type:** A basic built-in type such as `string`, `number`, or `boolean`.
*   **`any`:** A type that disables static checking for a value—use sparingly.
*   **`unknown`:** A type-safe counterpart to `any` that requires narrowing before use.
*   **Type Assertion:** A compile-time hint (`as Type`) telling TypeScript to treat a value as a specific type.
