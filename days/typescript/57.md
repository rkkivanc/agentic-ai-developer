# Day 57: Testing (runtime + type tests) - Mocking and Test Organization

## 📝 Today's Tasks

1.  **Mock Dependencies:** Replace `fetch` or filesystem modules with typed mocks in tests.
2.  **Use vi.fn / jest.fn:** Create spy functions and assert call arguments with types.
3.  **Structure Describe Blocks:** Group tests by behavior with clear `describe` and `it` names.
4.  **Share Test Fixtures:** Build typed factory functions for common test data.

## 📖 Task Explanations

Isolating units requires mocks—fake `fetch` responses, in-memory filesystems, stubbed clocks. TypeScript mocks should preserve signatures so tests fail when APIs change. Spy functions verify not just that calls happened, but with what typed arguments.

Good test names read like specifications: "returns ValidationError when email is invalid." Typed fixtures (`createTestUser()`) reduce boilerplate and keep tests focused on behavior.

Organization prevents test suites becoming unmaintainable—group by feature, not by file structure alone.

## 💡 What You'll Gain

By the end of today, you will mock dependencies with type safety, organize descriptive test suites, and use typed fixtures for readable tests.

## 📚 Dictionary for Developers

*   **Mock:** A test double replacing real dependencies with controlled behavior.
*   **Spy Function:** A function recording calls and arguments for assertions.
*   **Test Fixture:** Reusable setup data or objects for tests.
*   **Describe Block:** A grouped section of related tests.
*   **Isolation:** Testing one unit without real network, disk, or database.
