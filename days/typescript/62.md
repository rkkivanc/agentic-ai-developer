# Day 62: Advanced Types (II) - infer Keyword

## 📝 Today's Tasks

1.  **Extract with infer:** Write `type Element<T> = T extends (infer U)[] ? U : never`.
2.  **Infer Return Types:** Build `MyReturnType<T>` using `infer R` in a conditional.
3.  **Infer Parameters:** Extract function argument tuples with `infer A`.
4.  **Study Built-ins:** Trace how TypeScript's `ReturnType` and `Parameters` use `infer`.

## 📖 Task Explanations

`infer` declares a type variable within a conditional's true branch—TypeScript infers it from the matched pattern. This enables extracting return types, parameters, promise internals, and nested generic structures from complex types.

`infer` only works in conditional type contexts. It is the key to unpacking function signatures, array elements, and promise wrappers without manual duplication.

Understanding `infer` demystifies advanced library types and enables your own reusable type utilities.

## 💡 What You'll Gain

By the end of today, you will use `infer` to extract types from functions, arrays, and promises, and understand built-in utility implementations.

## 📚 Dictionary for Developers

*   **`infer`:** Declares a type variable inferred from a matched pattern in conditionals.
*   **Pattern Matching:** Matching a type shape to extract inner types.
*   **Return Type Extraction:** Pulling a function's return type via `infer R`.
*   **Argument Tuple Extraction:** Pulling parameters as a tuple via `infer A`.
*   **Unwrapping:** Extracting inner types from wrappers like `Promise<T>` or `T[]`.
