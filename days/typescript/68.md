# Day 68: Architecture & Patterns - Dependency Injection Lite

## 📝 Today's Tasks

1.  **Manual Constructor Injection:** Pass dependencies through constructors with typed parameters.
2.  **Build a Composition Root:** Wire all dependencies in one `main` or `bootstrap` module.
3.  **Avoid Service Locator:** Compare injection vs global singleton registries.
4.  **Type a Factory:** Create a factory function assembling services with correct dependencies.

## 📖 Task Explanations

Dependency injection means supplying dependencies from outside rather than creating them inside classes. Constructor injection with TypeScript types is explicit and testable—pass a mock repository in tests, real one in production.

The composition root is the single place wiring happens—`main.ts` constructs repositories, services, and handlers. No other module should instantiate concrete infrastructure. Service locators (global `getDb()`) hide dependencies and hurt testability.

You do not need a DI framework for small projects—typed manual injection is often enough.

## 💡 What You'll Gain

By the end of today, you will wire dependencies via constructor injection, centralize composition in a root module, and avoid service locator antipatterns.

## 📚 Dictionary for Developers

*   **Constructor Injection:** Passing dependencies as constructor parameters.
*   **Composition Root:** The single module where object graphs are assembled.
*   **Service Locator:** A global registry fetching dependencies—generally discouraged.
*   **Factory Function:** A function creating fully wired instances of a service.
*   **Testability:** Ease of substituting mocks via injected interfaces.
