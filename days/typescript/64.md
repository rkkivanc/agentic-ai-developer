## 📖 Task Explanations

Recursive types reference themselves—essential for JSON, ASTs, trees, and nested config. `type Json = string | number | boolean | null | Json[] | { [key: string]: Json }` is a classic pattern.

TypeScript allows reasonable recursion but hits depth limits on extreme nesting—usually a sign to simplify types or use interfaces for recursion. Deep utility types (`DeepPartial`) appear in libraries; understanding recursion helps you use them safely.

Recursive types are powerful for modeling unbounded structures at the type level.

## 💡 What You'll Gain

## 📚 Dictionary for Developers

*   **Recursive Type:** A type that references itself in its definition.
*   **`JsonValue`:** A common recursive union representing valid JSON data.
*   **Tree Type:** A node type with children of the same type.
*   **Recursion Depth Limit:** Compiler guard against infinitely expanding recursive types.
*   **Deep Utility:** A mapped type applying transforms recursively to nested properties.
