# Day 76: Strictness, Lint & CI - Strict Compiler Flags Deep Dive

## 📝 Today's Tasks

1.  **Enable Additional Flags:** Turn on `noUncheckedIndexedAccess`, `noImplicitOverride`, or `exactOptionalPropertyTypes`.
2.  **Fix Surfaced Errors:** Address each new error with proper narrowing or type fixes.
3.  **Document Flag Choices:** Record which strict flags you enable and why in `tsconfig` comments.
4.  **Compare CI vs Local:** Ensure CI uses the same strict config as local development.

## 📖 Task Explanations

Beyond `strict: true`, individual flags tighten specific holes. `noUncheckedIndexedAccess` adds `| undefined` to index access—catching missing array elements and record keys. `exactOptionalPropertyTypes` distinguishes missing properties from explicitly `undefined` ones.

Each flag has a migration cost but pays dividends in bug prevention. Document choices so teammates understand why certain patterns are required. CI must use identical tsconfig—drift between local and pipeline erodes trust.

Strictness is a team policy expressed in compiler configuration.

## 💡 What You'll Gain

By the end of today, you will enable and fix additional strict flags, document your strictness policy, and align CI with local compiler settings.

## 📚 Dictionary for Developers

*   **`noUncheckedIndexedAccess`:** Adds `undefined` to indexed access results.
*   **`exactOptionalPropertyTypes`:** Distinguishes missing vs explicitly undefined optional properties.
*   **Strict Migration:** Incrementally enabling flags and fixing resulting errors.
*   **Config Parity:** Identical TypeScript settings locally and in CI.
*   **Compiler Policy:** Team-agreed rules encoded in tsconfig flags.
