4.  **Wrap Callback APIs:** Convert a callback-style function to a typed `Promise` wrapper.

## 📖 Task Explanations

`async/await` is syntactic sugar over promises—it does not change types. An `async function` always returns `Promise<T>` even if you return `T` directly. This consistency lets you mix styles while keeping signatures predictable.

Sequential `await` in a loop is correct for dependent steps; `Promise.all` wins for independent parallel work. Mixing them poorly is a performance smell worth catching in review.

Many Node APIs still expose callbacks. Wrapping them in `new Promise<T>((resolve, reject) => ...)` produces typed promises your async code can await—bridging legacy and modern patterns.

## 📚 Dictionary for Developers

*   **`async` Function:** A function that implicitly returns a `Promise`.
*   **`await`:** Pauses async function execution until a promise settles.
*   **Parallel Execution:** Running independent async tasks concurrently via `Promise.all`.
*   **Promisify:** Wrapping a callback API to return a typed `Promise`.
*   **Async Control Flow:** Structuring dependent and independent async steps clearly.
