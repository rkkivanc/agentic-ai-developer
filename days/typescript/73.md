## 📝 Today's Tasks

4.  **Read strictFunctionTypes:** Enable and observe safer function type checking.

## 📖 Task Explanations

Variance describes how subtype relationships flow through generic positions. Return types are covariant (more specific is OK); parameters are contravariant (broader is OK) under `strictFunctionTypes`. Mutable arrays are invariant—`Dog[]` cannot substitute for `Animal[]` because you could push a Cat.

`readonly` arrays enable covariant reading—important for API design when consumers should not mutate. Library authors designing callback types must understand variance or consumers get unsound assignments.

This is advanced but explains mysterious assignability errors in real codebases.

## 📚 Dictionary for Developers

*   **Covariance:** Preserving subtype direction (more specific assignable where broader expected).
*   **Contravariance:** Reversing subtype direction in function parameter positions.
*   **Invariance:** No subtype substitution—exact type required.
*   **`strictFunctionTypes`:** Enables contravariant checking on function parameters.
*   **Readonly Covariance:** readonly arrays allowing safer subtype assignment for readers.
