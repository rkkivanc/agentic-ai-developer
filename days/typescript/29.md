4.  **Configure package.json exports:** Sketch `exports` field entries for a library targeting dual consumers.

## 📖 Task Explanations

Modern TypeScript rarely ends at `tsc` alone. Bundlers (Vite, esbuild, webpack) strip types, tree-shake, and bundle for browsers. Dev runners (`tsx`) execute TypeScript directly without a separate emit step. Knowing when to use each tool prevents over-engineering.

ESM and CommonJS coexist in npm. `esModuleInterop` and `allowSyntheticDefaultImports` smooth default imports from CJS modules. Library authors use `package.json` `exports` to route ESM and CJS consumers to correct entry points.

Today is about the full pipeline from `.ts` source to running code—not just type checking.

## 💡 What You'll Gain

By the end of today, you will choose appropriate dev and build tools, handle CJS/ESM interop, and understand how `package.json` exports route consumers to the right module format.

## 📚 Dictionary for Developers

*   **Bundler:** A tool combining modules for deployment (Vite, webpack, esbuild).
*   **`esModuleInterop`:** Improves compatibility importing CommonJS modules as ES modules.
*   **Tree Shaking:** Removing unused exports during bundling for smaller output.
*   **`package.json` exports:** Conditional entry points routing import vs require consumers.
*   **Dual Package Hazard:** Subtle bugs when ESM and CJS copies of a library diverge.
