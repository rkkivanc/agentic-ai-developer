# Day 5: Project - Typed Command-Line Greeting Utility

## 📝 Today's Tasks

1.  **Project Goal:** Build a CLI tool that greets a user by name and reports the current date.
2.  **Input:** Accept a name argument from the command line (`process.argv`).
3.  **Processing:** Validate input, apply a typed greeting function, and format today's date.
4.  **Output:** Print a friendly message to the console.
5.  **Error Handling:** Handle missing or empty name input with a clear typed error message.

## 📖 Task Explanations

Today, you'll build your first project! This is where you apply everything you've learned so far. You'll read command-line arguments, validate user input, call typed functions, and print results. The project ties together variables, function signatures, strict null checks, and basic error handling.

Keep the scope small but professional: separate parsing logic from formatting logic. Even a 40-line CLI benefits from clear function boundaries. This habit scales directly to larger TypeScript applications.

## 💡 What You'll Gain

By the end of today, you will have built a complete, albeit simple, TypeScript program from scratch. You will have practical experience applying fundamentals to a real task and a solid foundation for the composite types ahead.

## 📚 Dictionary for Developers

*   **CLI (Command-Line Interface):** A text-based interface for running programs and passing arguments via the terminal.
*   **`process.argv`:** A Node.js array containing command-line arguments passed to a script.
*   **Input Validation:** Checking that user-provided data meets expected shape and constraints before use.
*   **Error Handling:** Anticipating failure cases and responding with clear, typed feedback.
