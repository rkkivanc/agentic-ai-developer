# Day 79: Strictness, Lint & CI - CI Pipelines for TypeScript

## 📝 Today's Tasks

1.  **Create a GitHub Actions Workflow:** Run install, typecheck, lint, test, and build on push/PR.
2.  **Cache Dependencies:** Speed up CI with npm/pnpm cache actions.
3.  **Fail Fast:** Order jobs so cheap checks (typecheck, lint) run before expensive tests.
4.  **Add Status Badges:** Document CI status in your README.

## 📖 Task Explanations

CI is your team's safety net. A standard TypeScript pipeline: install → `tsc --noEmit` → `eslint` → `vitest` → `build`. Caching `node_modules` or package manager store cuts minutes off every run.

Fail-fast ordering saves CI minutes—type errors do not need full test suite runs. Branch protection requiring green CI prevents broken main branches.

Badges communicate health to contributors and portfolio viewers.

## 💡 What You'll Gain

By the end of today, you will have a working CI pipeline running typecheck, lint, tests, and build with dependency caching.

## 📚 Dictionary for Developers

*   **CI Pipeline:** Automated steps running on each push or pull request.
*   **`tsc --noEmit`:** Type-checking without emitting JavaScript files.
*   **Dependency Cache:** Storing installed packages between CI runs for speed.
*   **Fail Fast:** Running cheapest checks first to surface errors quickly.
*   **Branch Protection:** Requiring passing CI before merging to main.
