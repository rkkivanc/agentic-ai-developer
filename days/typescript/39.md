# Day 39: Async, Promises & Errors - AbortController and Cancellation

## 📝 Today's Tasks

1.  **Wire AbortSignal:** Pass `signal` through `fetch` and async functions supporting cancellation.
2.  **Propagate Cancellation:** Thread `AbortSignal` through layered async calls in your project.
3.  **Handle Abort Errors:** Distinguish `AbortError` from network failures with typed checks.
4.  **Timeout Pattern:** Implement `withTimeout(promise, ms)` using `AbortController`.

## 📖 Task Explanations

Long-running async work should be cancellable—users navigate away, requests time out, servers shut down. `AbortController` provides a standard `signal` browsers and Node fetch understand. Passing signals through your call stack lets deep functions respect cancellation.

`AbortError` is a distinct failure mode—handle it without treating it as a generic network error. Timeouts via `AbortController` are cleaner than `Promise.race` hacks with untyped results.

Cancellation is often overlooked in tutorials but essential in production UIs and services.

## 💡 What You'll Gain

By the end of today, you will propagate `AbortSignal` through async layers, handle abort errors distinctly, and implement timeout patterns for cancellable operations.

## 📚 Dictionary for Developers

*   **`AbortController`:** A Web/Node API for signalling cancellation to async operations.
*   **`AbortSignal`:** The signal object passed to `fetch` and other cancellable APIs.
*   **`AbortError`:** The error thrown when an operation is aborted.
*   **Cancellation Propagation:** Passing abort signals through nested async function calls.
*   **Timeout Wrapper:** A utility aborting a promise after a specified duration.
