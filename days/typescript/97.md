# Day 97: Capstone & Professional Delivery - Implementation Sprint

## 📝 Today's Tasks

1.  **Build Core Features:** Implement main use cases with typed services and repositories.
2.  **Wire Validation:** Add Zod schemas at every external boundary.
3.  **Shared Types Package:** Extract types and schemas to a shared module if full-stack.
4.  **Keep CI Green:** Commit incrementally with passing typecheck and tests.

## 📖 Task Explanations

Implementation day—focus on working software. Build vertically: one feature from API through persistence (or CLI through output) before sprawling horizontally. Validation at boundaries from Day 1 of implementation, not as an afterthought.

Shared package early prevents diverging client/server types. Incremental commits with green CI catch regressions immediately—do not pile up broken code for Day 99.

Shipping beats perfection; cut scope before cutting quality (types, validation, tests).

## 💡 What You'll Gain

By the end of today, you will have core capstone features implemented with validation, shared types, and continuously green checks.

## 📚 Dictionary for Developers

*   **Vertical Slice:** One complete feature built through all layers end-to-end.
*   **Incremental Commit:** Small commits each keeping the project in a working state.
*   **Boundary-First Implementation:** Adding validation before business logic depends on raw input.
*   **Scope Management:** Cutting features to deliver quality within time constraints.
*   **Green CI:** Continuous integration passing at each commit.
