# Day 45: DOM & Browser TypeScript - Practice

## 📝 Today's Tasks

1.  **Mini App Goal:** Build a typed todo list or weather widget using DOM APIs and `fetch`.
2.  **Type All Events:** Handle add, delete, and submit with correctly typed event listeners.
3.  **Fetch and Render:** Parse JSON into typed interfaces and render safely into the DOM.
4.  **Review:** Audit for `any`, unchecked `null`, and missing error states in the UI.

## 📖 Task Explanations

Practice day for browser TypeScript. A small todo or weather app exercises event typing, form handling, and typed `fetch` together—mirroring real frontend tasks without framework complexity.

Every interaction should have typed handlers: `click` on buttons, `submit` on forms, `input` on fields. Fetch responses should flow through typed interfaces before touching the DOM—never `innerHTML` with unvalidated data.

The review pass targets common frontend TypeScript gaps: nullable elements, swallowed fetch errors, and `any` from `document.querySelector`. Fix them before considering the exercise complete.

## 💡 What You'll Gain

By the end of today, you will have built a small typed browser app with safe event handling, typed fetch, and reviewed DOM code free of common type-safety gaps.

## 📚 Dictionary for Developers

*   **DOM Manipulation:** Creating, updating, and removing browser document elements.
*   **Typed Event Handler:** A listener using specific event types (`SubmitEvent`, `MouseEvent`).
*   **Render Pipeline:** Flow from typed data to DOM updates.
*   **Null-Safe Query:** Handling `null` from `querySelector` before use.
*   **Frontend Type Audit:** Reviewing UI code for `any`, null risks, and missing error UI.
