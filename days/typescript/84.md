# Day 84: Full-stack Contracts & Shared Types - End-to-End Type Safety

## 📝 Today's Tasks

1.  **Trace a Request Path:** Follow a typed request from client form → API → database → response.
2.  **Eliminate Casting Gaps:** Find any `as` assertions in the path and replace with validation.
3.  **Type Error Propagation:** Ensure API errors map to typed client error handling.
4.  **Add Contract Tests:** Verify client parser accepts server response fixtures.

## 📖 Task Explanations

End-to-end type safety means no untyped gaps from browser to database. Every boundary parses; every layer uses shared types; errors propagate as discriminated unions, not strings. Tracing one request exposes weak points—usually `as` casts at JSON boundaries.

Contract tests with fixture JSON verify runtime agreement even when both sides typecheck—servers can emit extra fields, clients can mis-parse. Combine compile-time shared types with runtime fixture tests for belt-and-suspenders safety.

This is the payoff of the full curriculum—types, Zod, shared packages, and tests working together.

## 💡 What You'll Gain

By the end of today, you will trace and harden a full request path with shared types, validation, and contract tests eliminating casting gaps.

## 📚 Dictionary for Developers

*   **End-to-End Type Safety:** Continuous typing from client UI through server to persistence.
*   **Casting Gap:** A point where `as` bypasses validation between typed layers.
*   **Contract Test:** Verifying client and server agree on serialized JSON fixtures.
*   **Error Propagation:** Carrying typed errors from server through client handling.
*   **Fixture JSON:** Sample serialized responses used to test parsing compatibility.
