# Day 19: Generics - keyof, Defaults, and Reusable Abstractions

## 📝 Today's Tasks

1.  **Build a Typed pick Helper:** Implement `pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K>` before using the built-in.
2.  **Default Type Parameters:** Add defaults like `<T = string>` and observe when they apply.
3.  **Compose Small Utilities:** Chain generic helpers (`pick` → `partial`) on a domain object.
4.  **Read Library Source:** Inspect how a popular library uses generics in one exported function.

## 📖 Task Explanations

Today connects generics to the utility types you will study formally on Days 21–24. A hand-rolled `pick` teaches how `keyof` and `Pick` cooperate: keys must exist on `T`, and the result type contains only those properties. Default type parameters reduce verbosity when most callers use the same type—`Array<T = any>` is a familiar example.

Composing generics builds abstractions that feel magical but are just functions at the type level. Understanding implementation demystifies lodash, Zod, and React's `useState` inference.

Reading library source cements that professional TypeScript is mostly well-chosen generics, constraints, and inference—not black magic.

## 💡 What You'll Gain

By the end of today, you will implement keyof-based utilities, use default type parameters, compose generic helpers, and read real library generic signatures with confidence.

## 📚 Dictionary for Developers

*   **`Pick<T, K>`:** A utility type selecting specific keys from an object type.
*   **Default Type Parameter:** A fallback type used when callers omit an explicit type argument.
*   **Generic Composition:** Combining multiple generic functions or types into higher-level abstractions.
*   **keyof T:** The union of all keys on type `T`, used to validate property name arguments.
*   **Library Typing:** How published packages expose generic APIs for consumer type safety.
