# Day 93: Team Practices & Migration - Monorepo Team Workflows

## 📝 Today's Tasks

1.  **Define Ownership:** Assign package owners and CODEOWNERS paths in a monorepo.
2.  **Coordinate Releases:** Practice versioning shared packages with changesets or similar.
3.  **Run Affected Tasks:** Configure turborepo/nx to test only changed packages in CI.
4.  **Document Workspace Commands:** Standardize `dev`, `build`, `test` from repository root.

## 📖 Task Explanations

Monorepos scale teams but need workflow discipline. CODEOWNERS routes PR reviews to package experts. Changesets coordinate semver bumps across interdependent packages—consumers know when shared types break.

Affected-only CI saves minutes—do not test unchanged packages. Root-level scripts (`pnpm -r test`) standardize contributor experience.

Monorepo tooling is team infrastructure—invest early when repos grow past one app.

## 💡 What You'll Gain

By the end of today, you will configure monorepo ownership, coordinated releases, and affected-only CI for efficient team workflows.

## 📚 Dictionary for Developers

*   **CODEOWNERS:** GitHub file mapping paths to responsible reviewers.
*   **Changesets:** Tooling for versioning and changelog generation in monorepos.
*   **Affected Tasks:** Running CI only on packages changed in a PR.
*   **Package Owner:** A person or team accountable for a workspace package.
*   **Root Scripts:** Standard commands run from monorepo root for all packages.
