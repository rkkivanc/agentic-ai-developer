# Day 3: TypeScript Fundamentals - Functions and Inference

## 📝 Today's Tasks

1.  **Type Function Parameters:** Add parameter and return type annotations to standalone functions.
2.  **Explore Optional and Default Parameters:** Use `?` and default values while keeping types accurate.
3.  **Write Arrow Functions with Types:** Apply the same typing rules to arrow function expressions.
4.  **Understand Contextual Typing:** See how TypeScript infers callback parameter types from usage.

## 📖 Task Explanations

Functions are where TypeScript earns its keep. Parameter types document what callers must provide; return types document what callers receive. Together they form a contract that editors enforce as you type.

Optional parameters (`name?: string`) and default values (`count = 0`) are common in real APIs. TypeScript models them so callers know which arguments are required. Arrow functions follow identical rules—syntax changes, typing does not.

Contextual typing is a powerful convenience: when you pass a callback to `array.map`, TypeScript often infers the element type for you. Learning to read these inferred signatures in your editor accelerates everyday development.

## 💡 What You'll Gain

By the end of today, you will write functions with clear parameter and return types, use optional and default parameters correctly, and recognize when the compiler infers types for you. Functions become self-documenting contracts instead of guesswork.

## 📚 Dictionary for Developers

*   **Function Signature:** The combination of parameter types and return type that defines a function's contract.
*   **Optional Parameter:** A parameter marked with `?` that callers may omit.
*   **Default Parameter:** A parameter with a fallback value used when the caller omits it.
*   **Return Type:** The type of value a function produces.
*   **Contextual Typing:** When TypeScript infers a type from the context in which an expression appears.
