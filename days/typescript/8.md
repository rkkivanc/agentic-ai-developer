# Day 8: Objects, Unions & Narrowing - Union and Intersection Types

## 📝 Today's Tasks

1.  **Create Union Types:** Combine types with `|` to represent values that may be one of several shapes.
2.  **Use Intersection Types:** Merge types with `&` to require all properties from multiple sources.
3.  **Model Discriminated Unions:** Add a shared literal field to distinguish between union members.
4.  **Narrow with `typeof` and `in`:** Refine union types inside conditional branches.

## 📖 Task Explanations

Union types express "this OR that"—a result might be a `string` or a `number`. Intersection types express "this AND that"—combining `Person` and `Employee` into a single shape. Together they let you model flexible APIs without resorting to `any`.

Discriminated unions add a tag field (like `kind: 'success' | 'error'`) so TypeScript can narrow reliably in `switch` statements. This pattern appears everywhere: Redux actions, API results, parser outputs. Learning it now saves pain later.

## 💡 What You'll Gain

By the end of today, you will model alternative shapes with unions, combine types with intersections, and begin narrowing unions safely in conditional code.

## 📚 Dictionary for Developers

*   **Union Type (`|`):** A type representing a value that may be one of several types.
*   **Intersection Type (`&`):** A type combining multiple types into one shape with all properties.
*   **Discriminated Union:** A union sharing a literal discriminator field for reliable narrowing.
*   **Narrowing:** Refining a broad type to a more specific type via checks.
*   **Literal Type:** A type representing an exact value, such as `'success'` or `42`.
