# Day 6: Objects, Unions & Narrowing - Object Types

## 📝 Today's Tasks

1.  **Define Object Type Literals:** Type objects inline with known property names and value types.
2.  **Add Optional and Readonly Properties:** Use `?` and `readonly` to model partial and immutable data.
3.  **Use Index Signatures:** Type objects with dynamic keys when property names are not fixed.
4.  **Nest Object Types:** Combine object types to represent structured records like users or addresses.

## 📖 Task Explanations

Most real-world data is structured, not primitive. Object types let you describe the shape of that data: a `User` has a `name` string and an `age` number. TypeScript uses structural typing—it cares about the shape, not a particular class name.

Optional properties model fields that may be absent; `readonly` prevents reassignment after creation. Index signatures (`[key: string]: number`) handle dictionaries and maps where keys are dynamic. Together these tools let you model API responses, configuration objects, and domain records accurately.

## 💡 What You'll Gain

By the end of today, you will model structured data with object types, optional fields, and readonly constraints. You can represent real records instead of loose bags of properties.

## 📚 Dictionary for Developers

*   **Object Type:** A type describing the shape of an object—its property names and value types.
*   **Structural Typing:** TypeScript's approach: compatibility is based on shape, not explicit inheritance.
*   **Optional Property:** An object field marked with `?` that may be absent.
*   **`readonly`:** A modifier preventing reassignment of a property after initialization.
*   **Index Signature:** A type pattern for objects with dynamic string or number keys.
