# Day 30: Modules, Tooling & tsconfig - Practice

## 📝 Today's Tasks

1.  **Code Challenge 1: Multi-Package Layout:** Create `src/core`, `src/utils`, and `src/cli` with path aliases connecting them.
2.  **Code Challenge 2: Dual Output:** Configure a build emitting both ESM and CJS (or document your chosen strategy).
3.  **Code Challenge 3: Strict Flags Audit:** Enable `noUncheckedIndexedAccess` or `exactOptionalPropertyTypes` and fix resulting errors.
4.  **Review:** Write a `TOOLING.md` explaining your module resolution and build choices for future teammates.

## 📖 Task Explanations

Practice day for modules and tooling. The multi-package layout simulates how real repos organize code—aliases prevent `../../../` import chains that break on refactors. Dual output matters when publishing libraries consumed by both ESM-first bundlers and legacy CJS tools.

Strict flag audits surface hidden assumptions. `noUncheckedIndexedAccess` adds `| undefined` to indexed access, catching off-by-one and missing-key bugs. Documenting choices in `TOOLING.md` is professional practice: tooling decisions affect every contributor.

Today consolidates Days 26–29 into a coherent project configuration you can reuse as a template.

## 💡 What You'll Gain

By the end of today, you will have a working multi-folder TypeScript project with path aliases, a deliberate module strategy, stricter flags, and documentation explaining your toolchain decisions.

## 📚 Dictionary for Developers

*   **Path Alias:** An import shortcut like `@core/*` mapped in `tsconfig` paths.
*   **Dual Package:** Publishing both ESM and CJS entry points for library consumers.
*   **Strict Flag Audit:** Enabling additional compiler checks and fixing surfaced issues.
*   **TOOLING.md:** Documentation explaining build, module, and compiler choices for a project.
*   **Import Hygiene:** Clean, stable import paths that survive refactors.
