# Day 49: Node.js TypeScript - Environment Variables and Structured Projects

## 📝 Today's Tasks

1.  **Type process.env:** Model environment variables with an interface and required keys.
2.  **Validate at Startup:** Fail fast when required env vars are missing or malformed.
3.  **Organize Folders:** Structure `src/` into `config`, `services`, `handlers`, and `types`.
4.  **Export a Clean Entry:** Keep `index.ts` minimal—wire config and start the server.

## 📖 Task Explanations

`process.env` values are `string | undefined`—never trust them as valid domain types without validation. A `loadConfig()` function parsing and validating env vars into a typed `Config` object fails fast at boot, preventing mysterious runtime failures.

Folder structure communicates architecture before anyone reads code. Config loads first; services receive typed config; handlers stay thin. This layout scales from scripts to services and previews Days 66–70 architecture topics.

Structured Node projects are easier to test, review, and onboard—professional habits start here.

## 💡 What You'll Gain

By the end of today, you will validate environment variables into typed config, organize a Node project into clear layers, and keep entry points minimal.

## 📚 Dictionary for Developers

*   **`process.env`:** Node's environment variable map—values are strings or undefined.
*   **Fail Fast:** Exiting early when configuration is invalid rather than failing later.
*   **Config Module:** A dedicated module loading and validating environment settings.
*   **Folder Conventions:** Standard directory layout (`handlers`, `services`, `types`).
*   **Bootstrap:** Startup logic wiring config before serving requests.
