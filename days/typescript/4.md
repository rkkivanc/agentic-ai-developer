# Day 4: TypeScript Fundamentals - Control Flow and Strict Mode

## 📝 Today's Tasks

1.  **Enable `strict` Mode:** Turn on `strict` in `tsconfig.json` and fix resulting errors in a small program.
2.  **Use `if/else` with Narrowing:** See how conditions refine types inside branches.
3.  **Work with `switch` and Loops:** Apply control flow while keeping variables correctly typed.
4.  **Handle `null` and `undefined`:** Use strict null checks to prevent runtime null-reference bugs.

## 📖 Task Explanations

Today is about making your programs reliable. Control flow statements—`if`, `switch`, loops—are not new, but TypeScript uses them to understand your code better. When you check `if (value !== null)`, the compiler knows `value` is non-null inside that block. This is the beginning of type narrowing.

Enabling `strict` in `tsconfig.json` activates a bundle of safety flags, including `strictNullChecks`. It may surface errors you did not expect—that is the point. Fixing them now prevents entire classes of production bugs. Professional TypeScript projects almost always run with `strict` enabled.

Strict null checking forces you to decide how missing values are represented. Instead of assuming a variable always exists, you model absence explicitly and handle it at the boundary.

## 💡 What You'll Gain

By the end of today, you will run TypeScript in strict mode and use control flow to satisfy the compiler. You will treat `null` and `undefined` as first-class concerns rather than surprises at runtime.

## 📚 Dictionary for Developers

*   **`strict`:** A `tsconfig` flag enabling a set of rigorous type-checking options.
*   **`strictNullChecks`:** A flag requiring explicit handling of `null` and `undefined`.
*   **Control Flow:** The order in which statements execute; TypeScript uses it to refine types.
*   **Type Narrowing:** When the compiler infers a more specific type after a conditional check.
*   **Nullish Value:** A value that is `null` or `undefined`.
