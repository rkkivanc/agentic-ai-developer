# Day 46: Node.js TypeScript - Project Setup and Execution

## 📝 Today's Tasks

1.  **Initialize a Node Project:** Set up `package.json`, TypeScript, and Node-appropriate `tsconfig`.
2.  **Choose a Runner:** Compare `tsx`, `node --import`, and compile-then-run workflows.
3.  **Configure Node moduleResolution:** Use `nodenext` or `node16` for accurate Node ESM resolution.
4.  **Script npm Commands:** Add `dev`, `build`, and `start` scripts with typed entry points.

## 📖 Task Explanations

Node.js TypeScript requires deliberate toolchain choices unlike browser bundler defaults. `moduleResolution: nodenext` aligns with Node's actual ESM resolution rules—including extension requirements and `package.json` `exports`.

Runners like `tsx` accelerate development; compiled output via `tsc` suits production deploys. Scripting `npm run dev` and `npm run build` standardizes workflows for teammates and CI.

Today establishes the Node project template you will extend through Day 50.

## 💡 What You'll Gain

By the end of today, you will have a correctly configured Node TypeScript project with appropriate module resolution and npm scripts for dev and production.

## 📚 Dictionary for Developers

*   **`nodenext`:** Module resolution matching Node.js ESM/CJS behavior.
*   **`tsx`:** A TypeScript execution runner for Node development.
*   **Entry Point:** The main file Node executes (`src/index.ts` → `dist/index.js`).
*   **npm Scripts:** package.json commands standardizing dev, build, and start.
*   **Node ESM:** Using `import`/`export` in Node with `"type": "module"`.
