# Day 15: Interfaces, Types & Classes - Practice

## 📝 Today's Tasks

1.  **Code Challenge 1: Plugin System:** Design a `Plugin` interface with `name`, `version`, and `execute()`; implement two plugins in classes.
2.  **Code Challenge 2: Repository Contract:** Define a `Repository<T>` interface with `findById` and `save`; implement an in-memory version.
3.  **Code Challenge 3: Merge and Extend:** Augment a fake third-party `Config` interface via declaration merging to add `debug?: boolean`.
4.  **Review:** Refactor one implementation to use a type alias instead of an interface and document the trade-off.

## 📖 Task Explanations

Today is practice day for interfaces, types, and classes. The plugin challenge tests `implements`, method signatures, and polymorphism—you should call plugins through the interface type, not concrete classes. The repository challenge introduces a generic-shaped contract you will formalize with generics on Day 16.

Declaration merging in the third challenge mirrors real library work: extending Express request types, Jest matchers, or environment globals. The review step forces intentional choice between interfaces and type aliases rather than arbitrary preference.

Take your time with compiler errors—they are feedback on contract violations, not obstacles.

## 💡 What You'll Gain

By the end of today, you will have hands-on experience designing contracts, implementing classes against them, and augmenting external types. These patterns appear in every professional TypeScript codebase.

## 📚 Dictionary for Developers

*   **Polymorphism:** Using a shared interface type to work with multiple implementations.
*   **Repository Pattern:** An abstraction over data access, often expressed as an interface.
*   **Plugin Architecture:** A design where components implement a common contract to extend behavior.
*   **Refactoring:** Restructuring code without changing behavior—here, swapping interface for type alias.
*   **Contract Violation:** When an implementation's shape does not satisfy its declared interface.
