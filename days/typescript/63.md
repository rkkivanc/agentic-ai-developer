# Day 63: Advanced Types (II) - Template Literal Types

## 📝 Today's Tasks

1.  **Create String Patterns:** Type event names like `` `on:${string}` `` with template literals.
2.  **Build Route Types:** Generate union of paths like `'/users' | '/users/${string}'`.
3.  **Combine with Unions:** Distribute template literals over string union keys.
4.  **Type CSS Properties:** Experiment with `` `${Property}-${Size}` `` style combinations.

## 📖 Task Explanations

Template literal types construct string types at the type level—`` `get${Capitalize<Key>}` `` generates method names from keys. They power typed event systems, CSS-in-JS libraries, and routing frameworks.

When a template literal type parameter is a string union, TypeScript distributes—`` `prefix_${A | B}` `` becomes `` `prefix_A` | `prefix_B` ``. This enables exhaustive string combinations from finite sets.

These features feel magical but underpin modern TypeScript DX in libraries like tRPC and typed routers.

## 💡 What You'll Gain

By the end of today, you will create template literal types, combine them with unions, and apply them to routes, events, and generated string APIs.

## 📚 Dictionary for Developers

*   **Template Literal Type:** A string type built with backtick pattern syntax at the type level.
*   **String Distribution:** Template literals distributing over string union inputs.
*   **`Capitalize<T>`:** A built-in intrinsic transforming string literal types.
*   **Typed Route:** A path string enforced by union or template literal types.
*   **String API Generation:** Deriving method or key names from type-level string templates.
