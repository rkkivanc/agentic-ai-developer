# Day 88: Performance & Modern TS Features - Incremental Builds

## 📝 Today's Tasks

1.  **Enable incremental:** Turn on `incremental` and `composite` for faster repeat compiles.
2.  **Use Project References:** Build a multi-package project with `tsc --build --watch`.
3.  **Measure Build Time:** Compare clean vs incremental build duration on your project.
4.  **Configure tsBuildInfo:** Understand `.tsbuildinfo` cache files and gitignore them.

## 📖 Task Explanations

Large TypeScript projects need fast feedback. `incremental` caches graph information between compiles; `composite` enables project references with cached outputs. `tsc --build` compiles only changed packages and their dependents.

`.tsbuildinfo` files should be gitignored—they are machine-local caches. Measuring build times justifies investment in project references vs single giant tsconfig.

Incremental builds are infrastructure—neglect them and teams disable strict checks to cope with slow CI.

## 💡 What You'll Gain

By the end of today, you will configure incremental compilation and project references, measuring improved rebuild performance.

## 📚 Dictionary for Developers

*   **Incremental Compilation:** Caching analysis between TypeScript builds for speed.
*   **`composite`:** Enabling a project for references and incremental builds.
*   **`.tsbuildinfo`:** Cache file storing incremental build state.
*   **`tsc --build`:** Solution-style build for referenced projects.
*   **Rebuild Performance:** Time to recompile after small changes.
