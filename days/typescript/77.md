# Day 77: Strictness, Lint & CI - ESLint and typescript-eslint

## 📝 Today's Tasks

1.  **Install typescript-eslint:** Configure flat or legacy config with recommended TypeScript rules.
2.  **Enable Type-Aware Rules:** Use `parserOptions.project` for rules needing type information.
3.  **Fix Key Violations:** Address `no-floating-promises`, `no-misused-promises`, and unsafe `any` usage.
4.  **Integrate with Editor:** Ensure ESLint errors appear in your IDE on save.

## 📖 Task Explanations

ESLint catches patterns TypeScript alone misses—especially async misuse and unsafe escapes. `typescript-eslint` provides hundreds of rules; type-aware rules need your tsconfig path to analyze promises and template expressions accurately.

`no-floating-promises` prevents fire-and-forget async bugs. `no-misused-promises` stops passing async functions where void returns are expected—common in event handlers. These rules require type information.

Lint and types complement each other: TypeScript proves shapes; ESLint enforces usage discipline.

## 💡 What You'll Gain

By the end of today, you will configure typescript-eslint with type-aware rules and fix high-impact lint violations in your project.

## 📚 Dictionary for Developers

*   **typescript-eslint:** ESLint tooling with TypeScript-specific parsing and rules.
*   **Type-Aware Rule:** An ESLint rule requiring TypeScript's type checker.
*   **`no-floating-promises`:** Requires promises to be awaited, returned, or explicitly handled.
*   **`no-misused-promises`:** Prevents async functions in void-return contexts like event handlers.
*   **Lint Policy:** Agreed ESLint rules enforcing code quality beyond types.
