# Day 22: Advanced Types (I) - Pick, Omit, and Record

## 📝 Today's Tasks

1.  **Derive a DTO with Pick:** Select only public fields from a domain type for an API response.
2.  **Hide Internals with Omit:** Remove `passwordHash` and internal IDs from a user type exposed to clients.
3.  **Model Dictionaries with Record:** Type `Record<Role, Permission[]>` for a role-to-permissions map.
4.  **Compare Manual vs Utility:** Rewrite a hand-rolled subset type using `Pick`/`Omit` and delete the duplicate.

## 📖 Task Explanations

`Pick` and `Omit` slice object types along property boundaries—essential for DTOs, public APIs, and form models that expose subsets of domain entities. Instead of maintaining parallel type definitions, you derive them from one canonical shape.

`Record<Keys, ValueType>` types dictionaries and lookup tables with known key sets—often `Record<string, T>` for open keys or `Record<Role, string>` for enumerated keys. Combined with literal unions from Day 10, `Record` enforces exhaustive mappings.

These utilities appear in every layer: API serializers, React props, configuration maps. Mastering them eliminates entire categories of copy-paste type maintenance.

## 💡 What You'll Gain

By the end of today, you will derive DTOs and lookup tables with `Pick`, `Omit`, and `Record`, keeping internal fields private and dictionaries exhaustively typed.

## 📚 Dictionary for Developers

*   **`Pick<T, K>`:** Selects a subset of properties from `T`.
*   **`Omit<T, K>`:** Removes specified properties from `T`.
*   **`Record<K, V>`:** An object type with keys `K` and values `V`.
*   **DTO (Data Transfer Object):** A type representing data crossing a boundary, often a `Pick`/`Omit` of domain types.
*   **Derived Type:** A type computed from another rather than declared independently.
