# Day 89: Performance & Modern TS Features - Type-Check Performance

## 📝 Today's Tasks

1.  **Profile Slow Types:** Identify files with heavy conditional or recursive types causing slowdowns.
2.  **Simplify Hot Types:** Refactor complex inferred types to named aliases or interfaces.
3.  **Limit Generic Depth:** Reduce deeply nested generics in public APIs.
4.  **Use skipLibCheck Judiciously:** Understand the trade-off of skipping `.d.ts` checking.

## 📖 Task Explanations

Type-level complexity has a cost—massive conditional types can slow `tsc` to a crawl. Profiling involves watching which files dominate `tsc --extendedDiagnostics` or `--generateTrace` output. Simplifying public API types helps consumers too.

Interfaces with simpler expansion behavior sometimes outperform complex mapped/conditional types. Avoid exporting deeply computed types when a named interface communicates the same contract.

`skipLibCheck` speeds CI by skipping node_modules `.d.ts` validation—a common trade-off with small risk.

## 💡 What You'll Gain

By the end of today, you will diagnose type-check slowdowns, simplify expensive types, and make informed `skipLibCheck` decisions.

## 📚 Dictionary for Developers

*   **Type-Check Performance:** How long the compiler spends analyzing types.
*   **Heavy Conditional Type:** Complex type logic that slows compilation.
*   **Type Simplification:** Refactoring types for faster checking and clearer APIs.
*   **`skipLibCheck`:** Skipping type checking of declaration files in dependencies.
*   **Extended Diagnostics:** Compiler flags reporting timing and performance data.
