# Day 31: First Typed Project - Planning and API Surface

## 📝 Today's Tasks

1.  **Choose a Project:** Pick a typed CLI utility or small library (e.g., task tracker, markdown linter).
2.  **Define the Public API:** List exported functions, types, and constants consumers will use.
3.  **Sketch Domain Types:** Model core entities with interfaces and utility-derived DTOs.
4.  **Write a Project Brief:** Document scope, non-goals, and success criteria in `PROJECT.md`.

## 📖 Task Explanations

Days 31–35 are your first substantial typed project—the MVP milestone of this curriculum. Today is planning, not coding frenzy. Professional work starts with a clear API surface: what do consumers import, and what stays internal?

Choosing a constrained scope matters. A CLI that manages tasks or validates markdown files fits the week—building a full SaaS does not. Define domain types early; they become the spine connecting parsing, business logic, and output.

`PROJECT.md` captures decisions so you do not scope-creep mid-week. Non-goals are as important as goals—they protect delivery.

## 💡 What You'll Gain

By the end of today, you will have a scoped project plan, a drafted public API, core domain types, and documented success criteria ready for implementation.

## 📚 Dictionary for Developers

*   **API Surface:** The set of exported types and functions consumers depend on.
*   **MVP (Minimum Viable Product):** The smallest version delivering real value.
*   **Domain Type:** A type representing a core business entity or concept.
*   **Scope:** The boundaries of what a project will and will not include.
*   **Project Brief:** A short document aligning goals, constraints, and deliverables.
