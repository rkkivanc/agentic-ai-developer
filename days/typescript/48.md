# Day 48: Node.js TypeScript - HTTP Servers and Request Handling

## 📝 Today's Tasks

1.  **Create a Typed Server:** Build an HTTP server with `node:http` or a lightweight framework.
2.  **Type Request Handlers:** Define handler signatures with typed request/response objects.
3.  **Parse JSON Bodies:** Read POST bodies into typed interfaces with validation.
4.  **Return Typed Errors:** Send consistent JSON error responses with proper status codes.

## 📖 Task Explanations

HTTP servers expose your types to the world. Whether using raw `node:http` or frameworks like Fastify or Express with types, handlers should be thin: parse, validate, delegate to services, respond.

Request body parsing returns `unknown` until validated—treat it like browser fetch JSON. Typed error responses (`{ error: { code, message } }`) let clients handle failures predictably.

This day bridges DOM fetch (Day 44) with server-side counterparts—the two sides of a typed HTTP contract.

## 💡 What You'll Gain

By the end of today, you will build a typed HTTP server with structured handlers, JSON parsing, and consistent error responses.

## 📚 Dictionary for Developers

*   **Request Handler:** A function processing an HTTP request and producing a response.
*   **Status Code:** Numeric HTTP outcome (200, 400, 500) sent with responses.
*   **JSON Body Parsing:** Reading request payloads into typed objects.
*   **Error Response Shape:** A consistent JSON structure for API failures.
*   **Transport Layer:** HTTP concerns separated from business logic.
