# Day 16: Generics - Generic Functions

## 📝 Today's Tasks

1.  **Write an Identity Function:** Implement `<T>(value: T) => T` and call it with string, number, and object types.
2.  **Build Array Helpers:** Create generic `first`, `last`, or `unique` helpers preserving element types.
3.  **Observe Inference:** Call generics without explicit type arguments and read inferred signatures in your editor.
4.  **Pass Explicit Type Args:** Use `helper<string>(...)` when inference cannot decide the correct type.

## 📖 Task Explanations

Generics parameterize types the way functions parameterize values. An identity function that accepts and returns `T` preserves the relationship between input and output—something `any` destroys. This is the core of reusable typed utilities in libraries from lodash-style helpers to React hooks.

TypeScript infers generic type parameters from arguments in most cases. When you pass a `string[]` to `first`, `T` becomes `string` automatically. Explicit type arguments are the escape hatch when inference fails—common with empty arrays or complex overloads.

Today establishes mental models you will extend with constraints (Day 18), `keyof` (Day 19), and generic classes (Day 17). Generics are not advanced trivia; they are everyday TypeScript.

## 💡 What You'll Gain

By the end of today, you will write and call generic functions, rely on inference by default, and supply explicit type arguments when needed. Reusable typed utilities become natural, not intimidating.

## 📚 Dictionary for Developers

*   **Generic Type Parameter:** A placeholder like `<T>` filled in by callers or inference.
*   **Type Argument:** The concrete type provided for a generic parameter, e.g. `string` in `identity<string>`.
*   **Inference from Usage:** The compiler deducing `T` from values passed to a generic function.
*   **Type Preservation:** Keeping input-output type relationships across function boundaries.
*   **Reusable Utility:** A helper function typed to work across many shapes safely.
