# Day 11: Interfaces, Types & Classes - Interfaces vs Type Aliases

## 📝 Today's Tasks

1.  **Define an Interface:** Create an interface describing a `User` with `id`, `name`, and `email` properties.
2.  **Create a Type Alias:** Rewrite the same shape using `type User = { ... }` and compare ergonomics.
3.  **Extend Both Forms:** Extend your user type with `AdminUser` using `extends` (interface) and intersection (`&`) for types.
4.  **Choose Intentionally:** Document when you would pick an interface over a type alias in a small README note.

## 📖 Task Explanations

Today you learn the two primary ways to name object shapes in TypeScript. Interfaces and type aliases overlap heavily for object types, but they differ in extension syntax, declaration merging, and team conventions. Understanding both lets you read any codebase and choose the right tool deliberately.

Interfaces use `extends` for inheritance and can be merged if declared twice in the same scope—useful for augmenting third-party types. Type aliases use intersections (`&`) and support unions, primitives, and tuples in one declaration. Many teams use interfaces for public object contracts and type aliases for unions, utilities, and computed shapes.

Neither choice affects runtime behavior; both erase completely at compile time. The professional habit is consistency within a module: pick a convention, document it, and apply it predictably.

## 💡 What You'll Gain

By the end of today, you will define object shapes with interfaces and type aliases, extend them confidently, and explain when each form fits best. This vocabulary is the foundation for every class, generic, and utility type ahead.

## 📚 Dictionary for Developers

*   **Interface:** A named contract describing the shape of an object, extendable with `extends`.
*   **Type Alias:** A named type created with `type`, supporting objects, unions, and more.
*   **Extension (`extends`):** Interface syntax for inheriting properties from another interface.
*   **Intersection (`&`):** Combining multiple types into one shape that has all members.
*   **Declaration Merging:** When multiple interface declarations with the same name merge into one.
