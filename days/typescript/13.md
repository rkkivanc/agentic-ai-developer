# Day 13: Interfaces, Types & Classes - Classes and Access Modifiers

## 📝 Today's Tasks

1.  **Build a Typed Class:** Define a class with constructor parameters, fields, and methods with explicit types.
2.  **Apply Access Modifiers:** Use `public`, `private`, and `protected` to control visibility.
3.  **Use `readonly` Fields:** Mark immutable properties and observe constructor initialization rules.
4.  **Compare Class vs Plain Objects:** Refactor a small example to show when a class adds value over a typed object.

## 📖 Task Explanations

TypeScript classes add type annotations and access modifiers to familiar JavaScript class syntax. Constructor parameter properties (`constructor(public name: string)`) reduce boilerplate while keeping types visible. Access modifiers exist only at compile time—they are erased in emitted JavaScript but guide API design.

`readonly` fields signal immutability after construction—a pattern common in domain models and DTOs. `private` and `protected` document intent: internal helpers stay internal. Not every TypeScript project leans on classes; many prefer functions and plain objects. Still, classes remain essential for frameworks, ORMs, and object-oriented libraries.

Learning classes with proper typing prepares you for generics on classes (Day 17) and architectural patterns later in the curriculum.

## 💡 What You'll Gain

By the end of today, you will write typed classes with access modifiers and readonly fields, and articulate when classes beat plain object types in TypeScript projects.

## 📚 Dictionary for Developers

*   **Access Modifier:** `public`, `private`, or `protected`—controls visibility at compile time.
*   **Parameter Property:** A constructor parameter that also declares and initializes a class field.
*   **`readonly`:** A modifier preventing reassignment after initialization.
*   **Class Method:** A function on a class instance or prototype with typed parameters and return.
*   **Erased Modifier:** Access modifiers are removed in emitted JavaScript; they guide developers only.
