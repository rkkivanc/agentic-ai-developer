# Day 12: Interfaces, Types & Classes - Implementing and Extending Contracts

## 📝 Today's Tasks

1.  **Implement an Interface:** Create a `class` or object factory that `implements` a typed interface.
2.  **Satisfy Multiple Interfaces:** Implement two interfaces on one class and verify structural compatibility.
3.  **Extend Interfaces:** Chain `extends` across `Entity`, `Timestamped`, and `Auditable` interfaces.
4.  **Validate at Compile Time:** Introduce a deliberate mismatch and fix it using the compiler error as guidance.

## 📖 Task Explanations

The `implements` keyword connects classes to type contracts without affecting runtime. A class that implements `Serializable` promises specific methods exist with correct signatures—the compiler enforces that promise. This is how you design replaceable components: depend on interfaces, not concrete classes.

Structural typing means TypeScript checks shape, not declaration keywords. An object literal can satisfy an interface even without `implements`, as long as properties match. Extending interfaces builds hierarchies of shared fields—`Auditable` might add `createdBy` atop `Timestamped`. These patterns mirror domain modeling in services and libraries.

Today reinforces that types are promises checked at build time. When a class drifts from its contract, you learn immediately—not in production.

## 💡 What You'll Gain

By the end of today, you will implement and extend interfaces on classes, combine multiple contracts, and trust the compiler to keep implementations aligned with declared shapes.

## 📚 Dictionary for Developers

*   **`implements`:** A class clause promising conformance to an interface's shape.
*   **Contract:** A type-level agreement about required properties and methods.
*   **Structural Compatibility:** TypeScript accepts values whose shape matches, regardless of explicit declaration.
*   **Interface Hierarchy:** A chain of interfaces extending one another to share common fields.
*   **Compile-Time Enforcement:** Errors surfaced during `tsc` rather than at runtime.
