# Day 36: Async, Promises & Errors - Promise Types

## 📝 Today's Tasks

1.  **Type Promise Returns:** Annotate functions returning `Promise<User>` and `Promise<void>`.
2.  **Chain with Thenables:** Use `.then` and `.catch` while preserving typed results through chains.
3.  **Handle Promise.all:** Type `Promise.all` tuples and understand `Promise.allSettled` result shapes.
4.  **Avoid Floating Promises:** Enable `noFloatingPromises` (if available) and handle all async calls.

## 📖 Task Explanations

Asynchronous TypeScript begins with `Promise<T>`—the type of a value available later. Return types document what callers await; `Promise<void>` signals side-effect-only work. Chaining must preserve types: mapping a `Promise<User>` to `Promise<string>` requires explicit transformation.

`Promise.all` returns a tuple when given a tuple of promises—a typed pattern for parallel fetches. `Promise.allSettled` never rejects; its result type reflects fulfilled and rejected outcomes per element.

Floating promises (unhandled async calls) are a common bug source. Linters flag them; professional code always awaits, returns, or voids promises intentionally.

## 💡 What You'll Gain

By the end of today, you will type promise-returning functions, chain async operations safely, and use `Promise.all` with correct tuple types.

## 📚 Dictionary for Developers

*   **`Promise<T>`:** A type representing an asynchronous value of type `T`.
*   **Thenable:** An object with `.then` compatible with Promise chaining.
*   **`Promise.all`:** Waits for all promises; typed tuple in, typed tuple out.
*   **Floating Promise:** An unhandled promise that may reject silently.
*   **Async Return Type:** The `T` in `Promise<T>` a caller receives after `await`.
