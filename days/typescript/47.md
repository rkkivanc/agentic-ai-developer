# Day 47: Node.js TypeScript - fs, path, and Core APIs

## 📝 Today's Tasks

1.  **Type File Operations:** Use `fs/promises` with typed paths and error handling.
2.  **Work with path:** Combine paths with `path.join` and type directory traversal results.
3.  **Read JSON Config:** Parse a JSON file into a typed interface with validation.
4.  **Handle ENOENT:** Narrow Node system errors with `instanceof` and `code` checks.

## 📖 Task Explanations

Node's core modules ship with TypeScript types via `@types/node`. `fs/promises` returns promises—fitting naturally with Days 36–40 patterns. Path operations are strings but should be validated before filesystem access.

Reading JSON config is a daily backend task: read file, parse, validate, use typed result. Node system errors carry `code` properties like `'ENOENT'`—narrow with type guards instead of assuming all errors are `Error`.

These fundamentals underpin CLIs, build tools, and services you will architect later.

## 💡 What You'll Gain

By the end of today, you will perform typed filesystem operations, handle Node-specific errors, and load JSON configuration into typed structures.

## 📚 Dictionary for Developers

*   **`fs/promises`:** Promise-based Node filesystem API.
*   **`path.join`:** Cross-platform safe path concatenation.
*   **Node System Error:** Errors with `code` like `ENOENT` or `EACCES`.
*   **Config Loading:** Reading and parsing configuration files at startup.
*   **`@types/node`:** TypeScript definitions for Node.js built-in modules.
