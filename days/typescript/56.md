# Day 56: Testing (runtime + type tests) - Unit Testing with Vitest or Jest

## 📝 Today's Tasks

1.  **Set Up a Test Runner:** Configure Vitest or Jest with TypeScript support in your project.
2.  **Write Unit Tests:** Test pure functions with typed inputs and `expect` assertions.
3.  **Organize Test Files:** Use `*.test.ts` colocated or in `__tests__` folders consistently.
4.  **Run in Watch Mode:** Use watch mode during development for fast feedback loops.

## 📖 Task Explanations

Tests prove behavior; types prove shape—professional TypeScript uses both. Vitest and Jest run TypeScript tests with minimal config via native ESM or ts-jest/tsx transforms. Unit tests target pure functions first: parsers, mappers, validators—the easiest high-value targets.

File organization matters for discoverability. Colocated tests encourage maintenance; separate folders suit larger repos. Watch mode keeps feedback under a second—essential for flow.

Today starts a testing mindset that continues through type tests on Day 58 and practice on Day 60.

## 💡 What You'll Gain

By the end of today, you will configure a TypeScript test runner, write unit tests for pure functions, and organize tests for maintainability.

## 📚 Dictionary for Developers

*   **Vitest:** A fast Vite-native test runner with TypeScript support.
*   **Jest:** A widely used JavaScript test framework with TS presets.
*   **Unit Test:** A test verifying a single function or module in isolation.
*   **Watch Mode:** Re-running tests automatically on file changes.
*   **Test Colocation:** Placing tests beside the source files they cover.
