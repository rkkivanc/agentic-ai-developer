# Day 42: DOM & Browser TypeScript - Event Typing

## 📝 Today's Tasks

1.  **Type Click Handlers:** Write `(event: MouseEvent) => void` listeners with correct targets.
2.  **Handle Keyboard Events:** Use `KeyboardEvent` for keydown handlers with typed `key` access.
3.  **Use Event Delegation:** Type delegated listeners on parent elements with narrowing `event.target`.
4.  **Custom Events:** Dispatch and listen for `CustomEvent<T>` with typed `detail` payloads.

## 📖 Task Explanations

DOM events carry rich type information. `MouseEvent`, `KeyboardEvent`, and `SubmitEvent` expose properties specific to each interaction—using generic `Event` loses that precision. Typed handlers catch typos in property names at compile time.

Event delegation attaches one listener to a parent, inspecting `event.target` inside—narrowing targets from `EventTarget` to `HTMLElement` is essential. Custom events with `CustomEvent<T>` bring typed payloads to browser-native pub/sub.

These patterns transfer directly to React and other frameworks where underlying DOM types still matter.

## 💡 What You'll Gain

By the end of today, you will write precisely typed event handlers, implement delegation safely, and use custom events with typed detail payloads.

## 📚 Dictionary for Developers

*   **`MouseEvent`:** A DOM event type for pointer interactions.
*   **`KeyboardEvent`:** A DOM event type with `key`, `code`, and modifier properties.
*   **Event Delegation:** Handling events on a parent instead of individual children.
*   **`CustomEvent<T>`:** A browser event carrying a typed `detail` payload.
*   **`EventTarget`:** The base type requiring narrowing to specific element types.
