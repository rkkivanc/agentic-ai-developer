# Day 1: TypeScript Fundamentals - Setup and Your First Program

## 📝 Today's Tasks

1.  **Set up your TypeScript Environment:** Install Node.js LTS and add TypeScript as a project dependency with `npm i -D typescript`.
2.  **Initialize a Project:** Run `npx tsc --init` to generate `tsconfig.json` and create a `src/` folder for your source files.
3.  **Create and Run "Hello, TypeScript!":** Write `src/hello.ts` that prints a greeting, compile with `npx tsc`, and run the output with Node.
4.  **Understand the Compile Loop:** Practice the edit → type-check → emit → run workflow that every TypeScript project uses.

## 📖 Task Explanations

Today is about laying the groundwork for your TypeScript journey. A proper development environment is essential for a smooth learning experience. Installing TypeScript locally in your project ensures every teammate and CI pipeline uses the same compiler version—a best practice called a reproducible toolchain.

Your "Hello, TypeScript!" program is a classic first step. It confirms your environment works and introduces the core idea that TypeScript compiles to plain JavaScript. Types exist at compile time to help you and your editor; they are erased before the code runs. Understanding this separation is fundamental to everything that follows.

The `tsconfig.json` file is your project's control panel. Options like `target`, `module`, `rootDir`, `outDir`, and `strict` shape how TypeScript behaves. You do not need to memorize every flag today—just know where configuration lives and that `strict` mode will become your friend.

## 💡 What You'll Gain

By the end of today, you will have a functional TypeScript development setup and understand the basic compile loop. You will know how to create, compile, and run a simple TypeScript program. This knowledge is the bedrock for all future concepts in the curriculum.

## 📚 Dictionary for Developers

*   **TypeScript (TS):** A statically typed superset of JavaScript that compiles to plain JavaScript.
*   **`tsc`:** The TypeScript compiler that type-checks source files and emits JavaScript.
*   **`tsconfig.json`:** The configuration file that controls compiler options for a TypeScript project.
*   **Type Erasure:** The process of removing type annotations during compilation so emitted JavaScript runs in any JS runtime.
*   **SDK / Toolchain:** The set of tools (Node.js, TypeScript, your editor) used to write, compile, and run code.
