# Day 32: Relations, Lists & Pagination - Nested Lists and Field Selection

## 📝 Today's Tasks

1.  **Model List Fields:** Add `[Post!]!` or similar list return types on parent objects.
2.  **Resolve Lists Efficiently:** Fetch related collections in parent resolvers when appropriate.
3.  **Query Nested Lists:** Write client queries selecting subsets of list items and their fields.
4.  **Empty List Behavior:** Return `[]` instead of null for absent collections when non-null lists are declared.

## 📖 Task Explanations

Lists are everywhere in GraphQL APIs—comments on posts, items in orders. Non-null list elements (`[Post!]!`) communicate that clients always receive an array, possibly empty, never null.

Consider where list resolution lives: on the parent object resolver for small data, or on a child field resolver when lazy loading is needed.

## 💡 What You'll Gain

By the end of today, you will model and resolve nested lists with correct nullability and efficient queries.

## 📚 Dictionary for Developers

*   **List Field:** A schema field whose type is a list of another type.
*   **Non-null Element:** Each list item must be present when using `!` inside brackets.
*   **Empty Array:** Valid response for a non-null list with no items.
*   **Collection Resolver:** Logic returning an array of child objects for a parent.
*   **Lazy Resolution:** Deferring list fetches until the client requests the list field.
