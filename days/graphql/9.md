# Day 9: Schema Definition Language (SDL) - Interfaces, Unions, and Custom Scalars

## 📝 Today's Tasks

1.  **Define an Interface:** Create a `Node` or `SearchResult` interface with shared fields.
2.  **Implement Interface Types:** Make concrete types implement the interface in SDL.
3.  **Create a Union Type:** Define a union like `SearchResult = User | Post | Comment`.
4.  **Introduce a Custom Scalar:** Sketch a `DateTime` or `Email` scalar and note server-side serialization needs.

## 📖 Task Explanations

Interfaces and unions enable polymorphism in GraphQL. An interface lists fields common to several types; a union represents a value that could be one of several unrelated types. Both require inline fragments in queries.

Custom scalars extend the type system beyond built-ins. They need server-side parsing and serialization logic—Apollo Server and GraphQL Yoga both support custom scalar definitions.

## 💡 What You'll Gain

By the end of today, you will understand polymorphic types in GraphQL and how custom scalars extend the type system.

## 📚 Dictionary for Developers

*   **Interface:** An abstract type listing fields that implementing types must provide.
*   **Union:** A type representing one of several possible object types, selected at runtime.
*   **Inline Fragment:** A query syntax `... on TypeName { fields }` for accessing type-specific fields.
*   **Custom Scalar:** A user-defined scalar with custom parse and serialize functions on the server.
*   **Polymorphism:** The ability to treat different concrete types through a shared abstract type.
