# Day 10: Schema Definition Language (SDL) - Practice

## 📝 Today's Tasks

1.  **Build a Library Schema:** Write complete SDL for Book, Author, Genre, and a LibraryQuery root.
2.  **Add Enums and Interfaces:** Include a `BookFormat` enum and a `Publishable` interface.
3.  **Validate with a Schema Linter:** Use GraphQL ESLint or your IDE to catch syntax errors.
4.  **Review Type Design:** Check naming consistency, nullability choices, and field descriptions.

## 📖 Task Explanations

Today consolidates SDL skills from Days 6–9. Building a library schema exercises scalars, objects, enums, interfaces, and modifiers together.

Run your SDL through a linter or schema validation tool. Catching design issues now prevents resolver rework later.

## 💡 What You'll Gain

By the end of today, you will have designed a complete, documented GraphQL schema in SDL covering core type system features.

## 📚 Dictionary for Developers

*   **Schema Linter:** A tool that checks SDL for syntax errors, naming rules, and best practices.
*   **Type Consistency:** Using uniform naming and nullability patterns across related types.
*   **Abstract Type:** A type that cannot be instantiated directly: interfaces and unions.
*   **Concrete Type:** An object type with a defined set of fields that can be returned directly.
*   **SDL Review:** A manual pass over schema definitions for clarity and correctness.
