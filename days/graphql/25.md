# Day 25: Resolvers & Execution - Practice

## 📝 Today's Tasks

1.  **Resolver Map:** Implement resolvers for a three-type schema with nested field resolvers.
2.  **Context Wiring:** Pass an in-memory data store through context instead of importing globals.
3.  **Execution Trace:** Add temporary logging to observe resolver call order for a nested query.
4.  **Self-Review:** Confirm default resolvers work for simple fields and custom resolvers handle relations.

## 📖 Task Explanations

Today consolidates resolver execution from Days 21–24. Tracing resolver order demystifies GraphQL's tree walk and prepares you for performance tuning.

Using context for data access is the pattern you'll keep through MVP and production—practice it now with a small in-memory store.

## 💡 What You'll Gain

By the end of today, you will wire resolvers with context and understand execution order through hands-on tracing.

## 📚 Dictionary for Developers

*   **Resolver Map:** The object mapping type fields and root operations to functions.
*   **Execution Trace:** Logging that shows which resolvers run and in what sequence.
*   **In-Memory Store:** A simple list or map standing in for a database during learning.
*   **Field Resolver:** A function resolving a single field on an object type.
*   **Tree Walk:** How the engine traverses the selection set depth-first.
