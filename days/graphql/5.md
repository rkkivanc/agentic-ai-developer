# Day 5: GraphQL Fundamentals - Practice

## 📝 Today's Tasks

1.  **Query Explorer Challenge:** Write five queries against a public API, each requesting different nested fields.
2.  **REST vs GraphQL Comparison:** Document one REST endpoint and the equivalent GraphQL query for the same screen.
3.  **Schema Sketch:** Draw a simple schema for a blog (Post, Author, Comment) on paper or a whiteboard.
4.  **Self-Review:** Verify each query runs without errors and returns only the fields you selected.

## 📖 Task Explanations

Today is about putting Days 1–4 into practice. Running varied queries against a live API solidifies how selection sets and nested fields work better than reading alone.

Your REST vs GraphQL comparison forces you to think about data requirements from a product perspective. The schema sketch prepares you for SDL work starting tomorrow.

## 💡 What You'll Gain

By the end of today, you will have hands-on experience querying live GraphQL APIs and a clear mental model of GraphQL's advantages over REST for specific use cases.

## 📚 Dictionary for Developers

*   **Nested Fields:** Requesting fields on related types within a single query, e.g., `author { name }`.
*   **Schema Sketch:** An informal diagram of types and relationships before writing formal SDL.
*   **Self-Review:** Checking your own work for correctness, clarity, and completeness.
*   **Public API:** A GraphQL endpoint open for exploration, often with read-only access.
*   **Happy Path:** The expected successful flow when inputs and conditions are normal.
