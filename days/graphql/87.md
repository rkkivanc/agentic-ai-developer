# Day 87: Federation & Multi-Graph - Apollo Federation and Subgraphs

## 📝 Today's Tasks

1.  **Define an Entity:** Mark a type with `@key` in a federated subgraph schema.
2.  **Extend Remote Types:** Use `extend type Product @key(fields: "id")` from another team's subgraph.
3.  **Run Rover or Apollo Tools:** Compose subgraph schemas in a local dev supergraph.
4.  **Plan Ownership:** Decide which team owns which types before splitting services.

## 📖 Task Explanations

Apollo Federation lets each service publish a subgraph; the gateway stitches them into one supergraph for clients. Entity keys link the same logical object across services.

Start with one subgraph locally before splitting production services—learn federation mechanics on a monorepo first.

## 💡 What You'll Gain

By the end of today, you will define federated entities and extend types across subgraph schemas.

## 📚 Dictionary for Developers

*   **Subgraph:** A service-owned GraphQL schema participating in federation.
*   **@key Directive:** Marks the fields used to identify and fetch an entity across subgraphs.
*   **Entity:** A type that can be resolved by multiple subgraphs in federation.
*   **Supergraph:** The unified schema composed from all subgraphs.
*   **Rover CLI:** Apollo's CLI for composing and checking federated schemas.
