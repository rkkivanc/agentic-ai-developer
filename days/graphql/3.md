## 📖 Task Explanations

A GraphQL request is typically an HTTP POST with a JSON payload containing the query string and optional variables. The server parses the document, validates it against the schema, executes resolvers, and returns JSON whose shape mirrors the query. This predictable mapping is one of GraphQL's biggest strengths.

Over-fetching happens when an endpoint returns more data than the client needs; under-fetching happens when multiple round trips are required. GraphQL lets clients specify exactly what they need, reducing both problems.

## 📚 Dictionary for Developers

*   **Over-fetching:** Receiving more data from an API than the client actually needs.
*   **Under-fetching:** Needing multiple API calls to assemble the data a screen requires.
*   **Document:** The GraphQL source text containing one or more operations.
*   **Resolver:** A function that returns data for a specific field in the schema.
*   **Payload:** The JSON body sent to or returned from a GraphQL server.
