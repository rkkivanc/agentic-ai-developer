# Day 93: Team Practices & Operations - Monitoring, Logging, and Tracing

## 📝 Today's Tasks

1.  **Log Operation Names:** Record named operations and duration for each request.
2.  **Track Error Rates:** Metric GraphQL errors versus HTTP 4xx/5xx separately.
3.  **Add Trace IDs:** Propagate correlation IDs from HTTP middleware into error extensions.
4.  **Dashboard Sketch:** Draft panels for p95 latency, error rate, and top operations.

## 📖 Task Explanations

Observability for GraphQL means logging operation names—not just POST /graphql. Errors array entries should increment metrics even when HTTP status stays 200.

Trace IDs in error extensions help support teams correlate user reports with server logs across microservices.

## 💡 What You'll Gain

By the end of today, you will instrument GraphQL APIs with operation logging, metrics, and trace correlation.

## 📚 Dictionary for Developers

*   **Operation Metric:** Latency or count grouped by named GraphQL operation.
*   **Error Rate:** Ratio of operations returning one or more GraphQL errors.
*   **Trace ID:** Correlation identifier linking logs and spans for one request.
*   **p95 Latency:** 95th percentile response time for operations.
*   **Observability:** Logs, metrics, and traces making production behavior visible.
