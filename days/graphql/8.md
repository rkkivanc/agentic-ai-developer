# Day 8: Schema Definition Language (SDL) - Enums and Literal Values

## 📝 Today's Tasks

1.  **Define an Enum Type:** Create an `OrderStatus` enum with values like PENDING, SHIPPED, DELIVERED.
2.  **Use Enums in Object Fields:** Add enum-typed fields to existing object types.
3.  **Query Enum Values:** Write queries that request enum fields and observe serialized string output.
4.  **Avoid Enum Anti-patterns:** Learn why enums should represent fixed sets, not open-ended strings.

## 📖 Task Explanations

Enums restrict a field to a fixed set of named values. They make APIs safer and more discoverable than free-form strings. Clients see valid options in documentation and get validation errors for invalid values.

Use enums when the set of values is stable and finite. For extensible or user-defined categories, consider a string field or a dedicated lookup type instead.

## 💡 What You'll Gain

By the end of today, you will know when and how to use enum types to represent constrained value sets in your schema.

## 📚 Dictionary for Developers

*   **Enum Type:** A scalar-like type restricted to a defined set of named constants.
*   **Enum Value:** One of the allowed constants in an enum definition, e.g., SHIPPED.
*   **Serialization:** How a GraphQL server converts internal values to JSON for the response.
*   **Validation Error:** An error returned when a query or input violates schema constraints.
*   **Closed Set:** A fixed collection of values that cannot be extended by clients at runtime.
