# Day 83: Caching & Performance - HTTP Caching and CDN Integration

## 📝 Today's Tasks

1.  **Identify Cacheable Queries:** Find public, non-personalized queries safe for CDN caching.
2.  **Use POST vs GET:** Evaluate persisted GET queries for cacheable read operations.
3.  **Set Cache-Control:** Apply `max-age` only on responses without private user data.
4.  **CDN Caveats:** Document why authenticated GraphQL POST requests usually bypass CDNs.

## 📖 Task Explanations

Most GraphQL traffic is personalized POST requests—not CDN-friendly. Exceptions include public persisted queries or GET-based read operations with shared responses.

Never cache responses containing private fields at shared edge caches—a common security mistake.

## 💡 What You'll Gain

By the end of today, you will identify when HTTP and CDN caching apply safely to GraphQL operations.

## 📚 Dictionary for Developers

*   **Cache-Control:** HTTP header directing browsers and CDNs how long to cache responses.
*   **Persisted GET:** Serving allow-listed queries via GET enabling CDN cache keys.
*   **Shared Cache:** CDN edge cache reused across users—unsafe for personalized data.
*   **Private Response:** Data varying per user that must not be stored at the edge.
*   **max-age:** Directive specifying seconds a response may be considered fresh.
