# Day 7: Schema Definition Language (SDL) - Object Types and Fields

## 📝 Today's Tasks

1.  **Define Object Types:** Create SDL for `Product`, `Category`, and `Review` with meaningful fields.
2.  **Add Field Descriptions:** Use description strings to document each field's purpose.
3.  **Model Relationships:** Connect types with object fields, e.g., `Product.category: Category`.
4.  **Plan Root Types:** Identify which types appear on Query, Mutation, and Subscription roots.

## 📖 Task Explanations

Object types are the building blocks of GraphQL schemas. Each type groups related fields, and object-typed fields express relationships between entities. Good naming and descriptions make schemas self-documenting.

Think about which types clients access directly from Query versus those reachable only through relationships. This distinction shapes your API's entry points.

## 💡 What You'll Gain

By the end of today, you will be able to model domain entities as GraphQL object types with documented fields and relationships.

## 📚 Dictionary for Developers

*   **Object Type:** A composite type with named fields, each with its own return type.
*   **Field:** A named property on an object type defined in SDL with a return type.
*   **Description:** A string in SDL that documents a type or field for developers and tools.
*   **Root Type:** The entry-point type (Query, Mutation, or Subscription) where operations begin.
*   **Domain Model:** A representation of business entities and their relationships in your schema.
