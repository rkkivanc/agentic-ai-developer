# Day 40: Errors, Validation & Nullability - Practice Project & Review

## 📝 Today's Tasks

1.  **Validation Layer:** Add input validation with clear user-facing error extensions.
2.  **Nullability Audit:** Review which fields should be `!` versus nullable.
3.  **Error Scenarios:** Test not-found, forbidden, and invalid input paths.
4.  **Client Messages:** Map extension codes to friendly UI strings in a frontend or doc table.

## 📖 Task Explanations

Today hardens error behavior on your MVP. GraphQL clients rely on consistent extension codes—define a small set and use them everywhere.

Nullability decisions affect every client. Overusing `!` causes cascading nulls; underusing it forces unnecessary null checks in UI code.

## 💡 What You'll Gain

By the end of today, you will have consistent validation, nullability, and error messaging across your API.

## 📚 Dictionary for Developers

*   **Validation Layer:** Business rules checking inputs before mutations persist data.
*   **Nullability Audit:** A systematic review of `!` markers on fields and inputs.
*   **Extension Code:** Machine-readable error category for clients and logs.
*   **Cascading Null:** When a non-null child error nullifies parent fields.
*   **User Message:** Human-readable text derived from structured errors.
