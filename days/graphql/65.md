# Day 65: Schema Design at Scale - Practice Project & Review

## 📝 Today's Tasks

1.  **Naming Pass:** Rename any inconsistent types or fields to match team conventions.
2.  **Deprecate a Field:** Mark an old field `@deprecated` with reason and alternative.
3.  **Domain Boundaries:** Group related types in SDL files or modules by bounded context.
4.  **Schema README:** Document naming rules and deprecation policy for your project.

## 📖 Task Explanations

Today applies schema-at-scale practices to your growing API. Deprecation rehearsal prepares you for real clients—you cannot delete fields overnight.

File organization by domain keeps SDL maintainable as types multiply.

## 💡 What You'll Gain

By the end of today, you will have a cleaner, documented schema with deprecation and naming consistency.

## 📚 Dictionary for Developers

*   **@deprecated:** SDL directive marking fields scheduled for removal.
*   **Naming Convention:** Consistent rules for PascalCase types and camelCase fields.
*   **Bounded Context:** A business area owning a subset of schema types.
*   **SDL Modules:** Multiple files composing one schema by domain.
*   **Deprecation Reason:** Human-readable explanation and migration path in SDL.
