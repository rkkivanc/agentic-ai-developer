# Day 66: Testing GraphQL - Unit Testing Resolvers

## 📝 Today's Tasks

1.  **Mock Context:** Pass fake `user` and `db` objects to resolver unit tests.
2.  **Test Happy and Sad Paths:** Assert resolver output and thrown errors for authorized and forbidden viewers.
3.  **Isolate from HTTP:** Test resolver functions directly without starting the full server.
4.  **Cover Edge Cases:** Empty lists, null parents, and missing IDs in field resolvers.

## 📖 Task Explanations

Unit tests target resolver logic in isolation—authorization rules, mapping, validation—without HTTP or database integration. Mocks keep tests fast and deterministic.

Table-driven tests work well for permission matrices with multiple roles and resources.

## 💡 What You'll Gain

By the end of today, you will unit test resolvers with mocked context and cover authorization edge cases.

## 📚 Dictionary for Developers

*   **Resolver Unit Test:** Test calling a resolver function with controlled inputs.
*   **Mock Context:** Fake per-request dependencies for isolated resolver tests.
*   **Table-Driven Test:** Parameterized tests looping scenarios with expected outcomes.
*   **Sad Path:** Test verifying failures and errors behave correctly.
*   **Deterministic Test:** Test without network or database variability.
