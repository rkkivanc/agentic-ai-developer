# Day 81: Caching & Performance - Client-Side Cache Policies

## 📝 Today's Tasks

1.  **Configure `typePolicies`:** Set `keyFields` for types identified differently than default `id`.
2.  **Merge Paginated Lists:** Define field policies merging connection pages in Apollo cache.
3.  **Manual Cache Eviction:** Call `cache.evict` after deletes to remove stale entities.
4.  **Inspect with DevTools:** Verify normalized storage after navigation between screens.

## 📖 Task Explanations

Apollo Client and urql caches need hints for paginated lists and composite keys. Field policies describe how incoming query results merge with cached data.

Eviction after deletes prevents ghost items appearing when users navigate back to a list view.

## 💡 What You'll Gain

By the end of today, you will configure client cache type and field policies for pagination and eviction.

## 📚 Dictionary for Developers

*   **typePolicies:** Apollo Client rules customizing cache behavior per GraphQL type.
*   **keyFields:** Fields used as cache identity when `id` is insufficient.
*   **Field Policy:** Merge or read function controlling how field data combines.
*   **cache.evict:** API removing specific entities or fields from normalized cache.
*   **Normalized Store:** Client cache deduplicating objects by identity across queries.
