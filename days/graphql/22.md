# Day 22: Resolvers & Execution - Context, Parent, and Arguments

## 📝 Today's Tasks

1.  **Trace the Parent Value:** Log `parent` in a field resolver and see data passed from the parent resolver.
2.  **Read Arguments:** Access `args` in resolvers for field arguments and mutation inputs.
3.  **Build Request Context:** Attach `user`, `db`, and headers to `context` in Apollo Server or GraphQL Yoga.
4.  **Avoid Context Abuse:** Store only per-request services in context, not global mutable state.

## 📖 Task Explanations

Resolver functions receive four parameters: parent, args, context, and info. The parent is the object containing the current field; args carry query parameters; context is your per-request dependency bucket.

Context is how authentication, database connections, and DataLoader instances flow into every resolver without global singletons.

## 💡 What You'll Gain

By the end of today, you will use parent, args, and context correctly in resolver functions.

## 📚 Dictionary for Developers

*   **Parent:** The resolved object containing the field currently being resolved.
*   **Context:** A per-request object shared across all resolvers for auth and data access.
*   **Args:** The arguments supplied to the current field in the operation.
*   **GraphQLContext:** The typed object you define for context in TypeScript servers.
*   **Per-Request Scope:** Lifecycle limited to one HTTP request, preventing data leaks.
