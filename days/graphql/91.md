# Day 91: Team Practices & Operations - Schema Review and Governance Process

## 📝 Today's Tasks

1.  **Write a Review Checklist:** Cover naming, nullability, auth, pagination, and deprecation.
2.  **Practice a Review:** Evaluate a sample schema change as if approving a teammate's PR.
3.  **Define Owners:** Assign schema reviewers or a platform team for shared graphs.
4.  **Block Criteria:** List changes requiring explicit approval—breaking, auth, performance.

## 📖 Task Explanations

Schema review prevents death by a thousand optional fields. A checklist makes reviews consistent whether you have one maintainer or twenty.

Governance scales trust—client teams rely on the graph when they know changes are reviewed for compatibility and security.

## 💡 What You'll Gain

By the end of today, you will run schema reviews using a governance checklist and clear approval criteria.

## 📚 Dictionary for Developers

*   **Schema Governance:** Processes controlling how a shared GraphQL schema evolves.
*   **Review Checklist:** Standard questions every schema change must answer.
*   **Schema Owner:** Team or role accountable for graph quality and compatibility.
*   **Approval Gate:** Required review before merging risky schema changes.
*   **Compatibility:** Whether existing client operations continue working after a change.
