# Day 28: First GraphQL API (MVP) - Connecting a Data Source

## 📝 Today's Tasks

1.  **Choose a Data Source:** Connect an in-memory array, SQLite, or REST wrapper behind resolvers.
2.  **Repository Pattern:** Hide database or HTTP calls in repository functions accessed via context.
3.  **Map Records to Types:** Transform storage rows into objects matching your GraphQL schema shape.
4.  **Handle Not Found:** Return null or throw `GraphQLError` with clear extensions for missing IDs.

## 📖 Task Explanations

Resolvers should stay thin—delegate persistence to repositories or data sources injected through context. This separation makes testing and swapping databases easier later.

Mapping layers translate snake_case columns or REST JSON into camelCase GraphQL fields without leaking storage details into the schema.

## 💡 What You'll Gain

By the end of today, you will connect a real or in-memory data source to your GraphQL resolvers via context.

## 📚 Dictionary for Developers

*   **Data Source:** Database, REST service, or store backing resolver logic.
*   **Repository:** Module encapsulating queries and commands for one aggregate.
*   **Context Injection:** Passing data access objects through GraphQL context per request.
*   **Record Mapping:** Converting storage format to GraphQL object shape.
*   **Not Found:** Missing entity case handled with null or a typed error.
