# Day 75: Security Hardening - Practice Project & Review

## 📝 Today's Tasks

1.  **Depth Limit:** Reject queries deeper than an agreed nesting level.
2.  **Complexity Budget:** Assign costs to fields and block overly expensive queries.
3.  **Disable Introspection:** Turn off introspection in a production-like config profile.
4.  **Persisted Query Experiment:** Register one operation hash and require it in production mode.

## 📖 Task Explanations

Today applies security controls from Days 71–74 to your running server. Tune limits using real queries from your client integration.

Try attacks yourself—absurdly deep queries or scripted bursts—to verify limits trigger.

## 💡 What You'll Gain

By the end of today, you will have depth, complexity, and introspection policies protecting your GraphQL API.

## 📚 Dictionary for Developers

*   **Depth Limit:** Maximum allowed nesting depth for a query.
*   **Complexity Budget:** Upper bound on summed field costs per operation.
*   **Introspection Toggle:** Environment flag enabling or disabling schema discovery.
*   **Operation Hash:** Fingerprint of a query document used for persisted queries.
*   **Abuse Test:** Deliberate attempt to exceed limits and verify blocking.
