# Day 85: Caching & Performance - Practice Project & Review

## 📝 Today's Tasks

1.  **Client Cache Policy:** Set field policies for a paginated list in Apollo Client or urql.
2.  **Server TTL Cache:** Cache one public, non-personalized query with Redis or in-memory TTL.
3.  **Invalidate on Write:** Clear server cache when relevant mutations run.
4.  **Measure Latency:** Compare p95 response times before and after caching.

## 📖 Task Explanations

Today applies caching strategies from Days 81–84. Cache only what is safe—public catalog queries yes, personalized dashboards maybe client-side only.

Measure to avoid premature optimization; cache the slowest resolver you identified in logs.

## 💡 What You'll Gain

By the end of today, you will have client and server caching with mutation invalidation on your API.

## 📚 Dictionary for Developers

*   **Field Policy:** Client cache rules for specific schema fields like paginated lists.
*   **TTL Cache:** Server cache expiring entries after a fixed duration.
*   **p95 Latency:** 95th percentile response time—a common SLA metric.
*   **Cache Hit:** Serving a response from cache without hitting origin resolvers.
*   **Invalidation Hook:** Logic clearing cache entries after mutations.
