# Day 90: Federation & Multi-Graph - Practice Project & Review

## 📝 Today's Tasks

1.  **Two Subgraphs Local:** Split your schema into two federated subgraphs with shared entities.
2.  **Compose Supergraph:** Run federation composition and fix any `@key` errors.
3.  **Gateway Query:** Execute a query spanning both subgraphs through the gateway.
4.  **Ownership Doc:** Write which team or module owns each subgraph type.

## 📖 Task Explanations

Today practices federation from Days 86–89 on a small scale. Even splitting users and posts into two services teaches entity keys and composition errors.

Ownership docs prevent two subgraphs defining the same type differently—a common federation pitfall.

## 💡 What You'll Gain

By the end of today, you will have composed a two-subgraph supergraph and queried it through a gateway.

## 📚 Dictionary for Developers

*   **Composition Error:** Failure when subgraph schemas cannot merge into a supergraph.
*   **Entity Key:** Fields uniquely identifying an object across subgraph boundaries.
*   **Gateway Query:** A client operation resolved by multiple subgraphs via the router.
*   **Subgraph Ownership:** Clear assignment of types to services or modules.
*   **Local Federation:** Practicing federation with processes on one machine.
