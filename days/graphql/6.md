# Day 6: Schema Definition Language (SDL) - Scalar Types and Type Modifiers

## 📝 Today's Tasks

1.  **Learn Built-in Scalars:** Study String, Int, Float, Boolean, and ID and when to use each.
2.  **Understand Type Modifiers:** Practice non-null (`!`) and list (`[]`) syntax in schema definitions.
3.  **Write Scalar Field Definitions:** Add scalar fields to a sample `User` type in SDL.
4.  **Validate Modifier Combinations:** Experiment with `[String!]!` and understand what each `!` guarantees.

## 📖 Task Explanations

Today you begin writing Schema Definition Language (SDL). Scalars are the leaves of your type tree—they hold actual values. The built-in scalars cover most cases, and modifiers express nullability and collections.

Non-null (`!`) means a field must always return a value; lists wrap types in square brackets. Combining them precisely communicates your data guarantees to clients and code generators.

## 💡 What You'll Gain

By the end of today, you will be able to define scalar fields with correct nullability and list modifiers in SDL.

## 📚 Dictionary for Developers

*   **SDL (Schema Definition Language):** The syntax used to define GraphQL types, fields, and operations.
*   **ID Scalar:** A string-like scalar meant for unique identifiers, serialized as a string.
*   **Non-null Type:** A type marked with `!` indicating the field or list element cannot be null.
*   **List Type:** A type wrapped in `[]` indicating the field returns an array of values.
*   **Leaf Field:** A field whose type is a scalar or enum, with no sub-selection.
