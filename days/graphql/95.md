# Day 95: Team Practices & Operations - Practice Project & Review

## 📝 Today's Tasks

1.  **Schema Review Checklist:** Run a self-review using governance criteria from Days 91–94.
2.  **Ops Dashboard:** Add basic logging for operation names and error rates.
3.  **Breaking Change Drill:** Propose a schema change and write the client migration plan.
4.  **Runbook Stub:** Document steps for rollback, incident response, and on-call contacts.

## 📖 Task Explanations

Today practices operating your API like a team would. Dashboards and runbooks matter even for solo capstones—they show production thinking.

Writing a migration plan for a hypothetical breaking change exercises communication skills with client developers.

## 💡 What You'll Gain

By the end of today, you will have governance checklists, basic observability, and operational docs for your GraphQL API.

## 📚 Dictionary for Developers

*   **Schema Review:** Structured checklist before merging schema changes.
*   **Operation Logging:** Recording named operations and outcomes for monitoring.
*   **Migration Plan:** Steps clients take when breaking schema changes ship.
*   **Runbook:** Operational guide for deploys, rollbacks, and incidents.
*   **Error Rate:** Percentage of operations returning GraphQL or HTTP errors.
