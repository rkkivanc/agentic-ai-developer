# Day 4: GraphQL Fundamentals - Tools, Sandboxes, and Developer Workflow

## 📝 Today's Tasks

1.  **Set Up a Local Playground:** Install Node.js and try Apollo Sandbox or GraphiQL against a demo server.
2.  **Use Schema Documentation Panels:** Explore auto-generated docs from the schema in your IDE or sandbox.
3.  **Save and Organize Queries:** Create a folder of `.graphql` files for reusable query documents.
4.  **Learn Keyboard Shortcuts:** Practice running queries, formatting documents, and viewing variables panels.

## 📖 Task Explanations

A proper tooling setup makes GraphQL development enjoyable. Sandboxes like Apollo Sandbox or GraphiQL provide syntax highlighting, autocomplete, and schema documentation. Saving queries as `.graphql` files keeps your learning organized and mirrors professional workflows.

Take time to explore the documentation panel generated from the schema. It is the same introspection data clients rely on, and reading it fluently is a core GraphQL skill.

## 💡 What You'll Gain

By the end of today, you will have a functional GraphQL exploration environment and a workflow for writing and saving query documents.

## 📚 Dictionary for Developers

*   **Apollo Sandbox:** A browser-based GraphQL IDE for exploring schemas and running operations.
*   **Autocomplete:** IDE feature that suggests fields and arguments based on the live schema.
*   **Query Document:** A saved `.graphql` file containing one or more named operations.
*   **Variables Panel:** A UI section for supplying runtime values to parameterized queries.
*   **Developer Experience (DX):** How pleasant and productive a toolchain feels for day-to-day work.
