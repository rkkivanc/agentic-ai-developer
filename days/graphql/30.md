# Day 30: First GraphQL API (MVP) - Practice Project & Review

## 📝 Today's Tasks

1.  **Project Goal:** Deliver a working GraphQL MVP with CRUD for one resource and GraphiQL enabled.
2.  **README:** Document how to install, run, and example queries/mutations.
3.  **Error Pass:** Ensure not-found and validation cases return sensible GraphQL errors.
4.  **Retrospective:** Note what was easy, what was confusing, and what to refactor next block.

## 📖 Task Explanations

Today you ship the MVP from Days 26–29. It should start with one command, respond on `/graphql`, and support create/read/update/delete for at least one type.

A short retrospective builds learning habits—note where resolver wiring tripped you up and which Apollo Server or GraphQL Yoga APIs you want to learn deeper.

## 💡 What You'll Gain

By the end of today, you will have delivered a documented GraphQL MVP ready for relationship and pagination features.

## 📚 Dictionary for Developers

*   **MVP Delivery:** Shipping the simplest API that demonstrates core GraphQL value.
*   **CRUD:** Create, read, update, and delete operations exposed as mutations and queries.
*   **Local Sandbox:** GraphiQL or Apollo Sandbox for exploring your running API.
*   **Retrospective:** Structured reflection on what worked and what to improve.
*   **README:** Setup and usage instructions for other developers or future you.
