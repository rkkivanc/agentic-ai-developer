# Day 21: Resolvers & Execution - Introduction to Resolvers

## 📝 Today's Tasks

1.  **Write Your First Resolver:** Map a Query field to a function returning static or fetched data.
2.  **Understand the Resolver Signature:** Learn `(parent, args, context, info)` parameters.
3.  **Attach Resolvers to Schema:** Wire resolvers in Apollo Server or GraphQL Yoga alongside type definitions.
4.  **Return Shapes Matching Schema:** Ensure resolver return values align with declared field types.

## 📖 Task Explanations

Resolvers are functions that produce values for schema fields. The Query root resolver runs first; field resolvers on object types run as the execution engine walks the selection set.

Start with simple resolvers returning hard-coded data, then connect real data sources once the wiring is clear.

## 💡 What You'll Gain

By the end of today, you will implement and wire basic resolvers that fulfill schema fields.

## 📚 Dictionary for Developers

*   **Parent Object:** The resolved object containing the field currently being resolved.
*   **Context:** A per-request object shared across resolvers for auth, data sources, and loaders.
*   **Execution:** The process of running resolvers to produce a response for an operation.
*   **Field Path:** The dot-separated path to a field, e.g., `Query.user.email`.
*   **Default Resolver:** Built-in behavior that returns a property from the parent when no custom resolver exists.
