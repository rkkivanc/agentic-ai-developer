# Day 56: DataLoader & N+1 - Understanding the N+1 Problem

## 📝 Today's Tasks

2.  **Measure Query Count:** Log database calls per GraphQL request and observe the spike.
3.  **Identify Resolver Patterns Causing N+1:** Find field resolvers that fetch one record at a time.
4.  **Plan a Batching Solution:** Outline how DataLoader will consolidate lookups in upcoming days.

## 📖 Task Explanations

The N+1 problem is GraphQL's most common performance pitfall. A query returning N items with a nested field can trigger N additional fetches if each field resolver queries independently.

Recognizing N+1 in logs is a critical skill. You'll fix it with DataLoader, but first you must see the problem clearly.

## 💡 What You'll Gain

By the end of today, you will recognize and demonstrate the N+1 problem in a GraphQL API.

## 📚 Dictionary for Developers

*   **N+1 Problem:** Performance issue where one query triggers N additional queries for related items.
*   **DataLoader:** A utility that batches and caches loads within a single request.
*   **Batch Function:** A function DataLoader calls with a list of keys to fetch in one round trip.
*   **Per-Request Cache:** A cache scoped to one HTTP request, preventing cross-user data leaks.
*   **Eager Loading:** Fetching related data proactively to avoid repeated round trips.
