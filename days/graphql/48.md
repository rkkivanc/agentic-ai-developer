# Day 48: Clients & Frontend Integration - Mutations and Optimistic Updates

## 📝 Today's Tasks

1.  **Run a Mutation Hook:** Use `useMutation` in Apollo Client or urql's `useMutation` with updates.
2.  **Optimistic Response:** Provide temporary data so the UI updates before the server responds.
3.  **Rollback on Error:** Revert optimistic state when the mutation returns errors.
4.  **Refetch or Cache Modify:** Choose between automatic list updates and manual cache writes.

## 📖 Task Explanations

Optimistic updates make apps feel instant—likes, todos, and cart changes appear immediately. GraphQL clients need a shaped optimistic response matching the mutation selection set.

Always handle failure: network loss after optimism confuses users if items disappear without explanation.

## 💡 What You'll Gain

By the end of today, you will implement mutations with optimistic updates and error rollback in a GraphQL client.

## 📚 Dictionary for Developers

*   **Optimistic Response:** Temporary data injected into cache before server confirmation.
*   **useMutation:** Client hook executing write operations and exposing loading state.
*   **Rollback:** Reverting optimistic cache changes when a mutation fails.
*   **Cache Update:** Function adjusting stored queries after a successful write.
*   **Mutation Hook:** Client API tying UI events to GraphQL write operations.
