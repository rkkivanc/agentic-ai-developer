# Day 73: Security Hardening - Introspection and Production Hardening

## 📝 Today's Tasks

1.  **Toggle Introspection:** Disable schema introspection in production configuration.
2.  **Environment Profiles:** Maintain `development` with introspection on and `production` with it off.
3.  **Hide Schema Details:** Ensure GraphiQL is not exposed on public production URLs.
4.  **Document Exceptions:** Note if admin-only introspection is required for specific tooling.

## 📖 Task Explanations

Introspection helps development but aids attackers mapping your API. Disable it in production unless a trusted admin tool requires it behind auth.

Disabling GraphiQL on production endpoints is a simple hardening step often forgotten after local development.

## 💡 What You'll Gain

By the end of today, you will harden production GraphQL endpoints by controlling introspection and IDE exposure.

## 📚 Dictionary for Developers

*   **Introspection:** Querying the schema itself for types and fields.
*   **Production Profile:** Environment configuration with stricter security defaults.
*   **GraphiQL Exposure:** Publicly serving an interactive schema IDE on production.
*   **Attack Surface:** Features attackers can abuse to explore or stress an API.
*   **Admin Tooling:** Trusted internal apps that may need introspection behind VPN or auth.
