# Day 26: First GraphQL API (MVP) - Project Setup — Apollo Server or GraphQL Yoga

## 📝 Today's Tasks

1.  **Initialize a Node Project:** Create a TypeScript project with `package.json` and install a GraphQL server library.
2.  **Choose Apollo Server or GraphQL Yoga:** Set up a minimal HTTP server exposing `/graphql`.
3.  **Enable GraphiQL or Sandbox:** Configure the interactive IDE for local development.
4.  **Add a Health Check Route:** Include a simple non-GraphQL endpoint for uptime monitoring.

## 📖 Task Explanations

Today begins your first GraphQL API project. Both Apollo Server and GraphQL Yoga offer excellent TypeScript support; pick one and stay consistent through the MVP block.

A health check endpoint is a small production-minded touch that deployment platforms expect alongside your GraphQL route.

## 💡 What You'll Gain

By the end of today, you will have a running local GraphQL server ready for schema and resolver development.

## 📚 Dictionary for Developers

*   **Apollo Server:** A popular Node.js GraphQL server with plugins, subscriptions, and federation support.
*   **GraphQL Yoga:** A lightweight, standards-focused GraphQL server for Node and other runtimes.
*   **MVP (Minimum Viable Product):** The simplest version of an API that demonstrates core value.
*   **Data Source:** Any database, REST service, or in-memory store backing your resolvers.
*   **CRUD:** Create, Read, Update, Delete—the basic data operations most APIs support.
