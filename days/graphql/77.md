# Day 77: Tooling, Codegen & DX - IDE Plugins and Schema Navigation

## 📝 Today's Tasks

1.  **Install GraphQL VS Code Extension:** Enable syntax highlighting, validation, and go-to-definition.
2.  **Configure Schema Pointer:** Point the plugin at your local SDL or introspection endpoint.
3.  **Autocomplete in Queries:** Write `.graphql` files with field completion from the live schema.
4.  **Share Settings:** Commit editor config so teammates get the same GraphQL IDE experience.

## 📖 Task Explanations

IDE plugins turn SDL and query documents into a navigable, validated workspace. Linking to your schema catches typos before runtime.

Pointing at a running dev server's introspection keeps client queries in sync as the schema evolves.

## 💡 What You'll Gain

By the end of today, you will configure IDE plugins for schema-aware GraphQL editing and navigation.

## 📚 Dictionary for Developers

*   **GraphQL VS Code:** Popular extension adding language support for GraphQL files.
*   **Schema Pointer:** Config telling the IDE where to load schema for validation.
*   **Go to Definition:** Jump from a query field to its SDL definition.
*   **Inline Validation:** Squiggles on unknown fields or type errors in query documents.
*   **Editor Config:** Shared settings ensuring consistent tooling across the team.
