# Day 67: Testing GraphQL - Integration Testing GraphQL APIs

## 📝 Today's Tasks

1.  **Spin Up Test Server:** Boot Apollo Server or GraphQL Yoga in-process for tests.
2.  **Execute Operations:** Send HTTP POST requests or use `execute` with query documents.
3.  **Assert JSON Shape:** Compare `data` and `errors` to expected fixtures.
4.  **Test Auth Paths:** Verify protected fields fail without context and succeed with tokens.

## 📖 Task Explanations

Integration tests hit the real GraphQL execution pipeline—not isolated resolver functions. They catch wiring mistakes between schema, resolvers, and middleware.

Use a clean database or in-memory store per test to avoid order-dependent failures.

## 💡 What You'll Gain

By the end of today, you will write integration tests that execute operations against a live test GraphQL server.

## 📚 Dictionary for Developers

*   **Integration Test:** Test exercising multiple layers of the GraphQL stack together.
*   **execute():** GraphQL-js function running an operation against a schema in tests.
*   **Test Server:** An in-process server instance used only in the test suite.
*   **Fixture:** Saved expected response JSON for comparison.
*   **Test Isolation:** Independent data and context per test case.
