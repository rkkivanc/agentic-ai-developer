# Day 45: Authentication & Authorization - Practice Project & Review

## 📝 Today's Tasks

1.  **Protect Mutations:** Require authentication for all write operations.
2.  **Field Guards:** Hide or null sensitive fields based on roles in context.
3.  **Negative Tests:** Call protected operations without tokens and with wrong roles.
4.  **Auth Documentation:** Document how clients obtain and send credentials.

## 📖 Task Explanations

Today secures your MVP end to end. Authorization belongs as close to the data as possible—field-level checks beat coarse middleware alone.

Negative tests prove unauthorized users cannot read private emails or admin-only fields.

## 💡 What You'll Gain

By the end of today, you will have authentication and authorization enforced across your GraphQL MVP.

## 📚 Dictionary for Developers

*   **Field Guard:** Resolver logic allowing or denying access per field.
*   **Negative Test:** Verifying forbidden actions fail as expected.
*   **Role Claim:** A JWT or session value indicating user permissions.
*   **Protected Mutation:** A write operation requiring a valid authenticated context.
*   **Auth Documentation:** Instructions for obtaining and attaching access tokens.
