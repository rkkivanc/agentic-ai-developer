# Day 46: Clients & Frontend Integration - Apollo Client Setup and Configuration

## 📝 Today's Tasks

1.  **Install Apollo Client:** Add `@apollo/client` to a React or vanilla TypeScript frontend project.
2.  **Configure HttpLink and Cache:** Point the client at your GraphQL endpoint with an InMemoryCache.
3.  **Run a Query with useQuery:** Fetch data in a component and render loading and error states.
4.  **Inspect Apollo DevTools:** Explore cache contents and active queries in browser extensions.

## 📖 Task Explanations

Client integration brings your schema to life in user interfaces. Apollo Client handles networking, caching, and React hooks out of the box.

Always handle loading and error states explicitly. GraphQL errors can be partial—some fields succeed while others fail.

## 💡 What You'll Gain

By the end of today, you will configure Apollo Client and fetch data in a frontend application.

## 📚 Dictionary for Developers

*   **Apollo Client:** A comprehensive GraphQL client with cache, local state, and React integrations.
*   **urql:** A lightweight GraphQL client emphasizing simplicity and extensible exchanges.
*   **Cache Policy:** Rules determining when cached query results are reused or refetched.
*   **Optimistic Update:** Updating the UI before the server confirms a mutation succeeded.
*   **Normalized Cache:** Storage that deduplicates objects by ID across queries.
