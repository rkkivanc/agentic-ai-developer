# Day 20: Mutations & Inputs - Practice

## 📝 Today's Tasks

1.  **Mutation Suite:** Implement create, update, and delete mutations for a `Task` type with input objects.
2.  **Payload Wrappers:** Return `TaskPayload` types including `task` and a list of user-facing errors.
3.  **Sandbox Testing:** Run mutations in Apollo Sandbox with variables for each operation.
4.  **Schema Review:** Verify mutation names are verbs, inputs are separate types, and descriptions are clear.

## 📖 Task Explanations

Today consolidates mutation skills from Days 16–19. Building a small task API exercises input types, payloads, and idempotency thinking in one cohesive schema.

Test invalid inputs deliberately—empty titles, duplicate IDs—and confirm your payload pattern communicates failures without opaque server errors.

## 💡 What You'll Gain

By the end of today, you will have designed and tested a coherent set of mutations with input objects and payload types.

## 📚 Dictionary for Developers

*   **Input Object:** A structured mutation argument type grouping multiple fields.
*   **Verb Naming:** Naming mutations after business actions, e.g., `createTask`.
*   **User-Facing Error:** A validation message safe to display in the UI.
*   **Write Path:** The set of mutations clients use to change server state.
*   **Mutation Test:** Executing write operations against a schema to verify behavior.
