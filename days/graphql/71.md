# Day 71: Security Hardening - Query Depth and Complexity Limits

## 📝 Today's Tasks

1.  **Calculate Query Depth:** Measure nesting levels in a few real client queries.
2.  **Set Depth Limit:** Reject queries exceeding a configured maximum depth in server plugins.
3.  **Assign Field Costs:** Give expensive fields higher complexity weights.
4.  **Block Absurd Queries:** Try a deeply nested introspection-style query and confirm rejection.

## 📖 Task Explanations

Depth and complexity limits protect servers from malicious or accidentally expensive queries. Apollo Server and GraphQL Yoga support plugins or middleware enforcing both.

Tune limits using production query shapes—too aggressive breaks legitimate mobile clients; too loose invites DoS.

## 💡 What You'll Gain

By the end of today, you will configure query depth and complexity limits based on real operation shapes.

## 📚 Dictionary for Developers

*   **Query Depth:** Maximum nesting level of fields in an operation.
*   **Complexity Score:** Numeric cost summed from field weights in a query.
*   **Cost Analysis:** Assigning weights reflecting database or CPU expense per field.
*   **DoS Query:** Malicious deeply nested request intended to overload the server.
*   **Validation Rule:** Server plugin rejecting operations exceeding configured limits.
