# Day 41: Authentication & Authorization - Authentication Fundamentals in GraphQL

## 📝 Today's Tasks

1.  **Authenticate at HTTP Layer:** Parse cookies or Authorization headers before GraphQL execution.
2.  **Attach Identity to Context:** Store `userId` or full user on context for resolvers.
3.  **Public vs Private Fields:** Keep public catalog queries open; guard private profile fields.
4.  **Document Auth Flow:** Write how clients obtain and refresh tokens for your API.

## 📖 Task Explanations

GraphQL has no built-in auth—HTTP middleware establishes identity, then resolvers and schema directives enforce rules. This stack-agnostic pattern works in Apollo Server, GraphQL Yoga, and other runtimes.

Authenticate once per request, authorize per field. Mixing the two leads to duplicated checks and inconsistent behavior.

## 💡 What You'll Gain

By the end of today, you will authenticate GraphQL requests at the HTTP layer and expose identity via context.

## 📚 Dictionary for Developers

*   **Authentication:** Verifying who is making the request.
*   **Authorization:** Deciding whether they may access specific fields or mutations.
*   **HTTP Middleware:** Code running before the GraphQL executor receives the request.
*   **Anonymous Access:** Allowed operations for unauthenticated clients.
*   **Identity:** Stable user identifier attached to context after authentication.
