# Day 2: GraphQL Fundamentals - Core Concepts — Schema, Types, and Operations

## 📝 Today's Tasks

1.  **Identify the Three Operation Types:** Distinguish queries (read), mutations (write), and subscriptions (realtime).
2.  **Study the Type System:** Learn scalar types, object types, and how fields map to resolvers.
3.  **Inspect a Schema with Introspection:** Run an introspection query or use IDE tools to list types and fields.
4.  **Trace a Query to Its Response:** Follow how a selection set maps to JSON output field by field.

## 📖 Task Explanations

GraphQL is built on a strong type system. Every API exposes a schema that acts as a contract between server and client. Queries fetch data, mutations change it, and subscriptions push updates over a persistent connection. Understanding these three operations is essential before you implement any of them.

Introspection is a powerful learning tool. GraphQL schemas are designed to be discoverable, so tools can list every type and field automatically. Spend time connecting schema definitions to the JSON responses you receive.

## 💡 What You'll Gain

By the end of today, you will understand GraphQL's type system, the three operation types, and how introspection helps you explore unfamiliar APIs.

## 📚 Dictionary for Developers

*   **Query:** A read-only GraphQL operation that requests data without side effects.
*   **Mutation:** A GraphQL operation that creates, updates, or deletes data on the server.
*   **Subscription:** A long-lived operation that streams realtime updates to the client.
*   **Scalar Type:** A leaf type representing a single value, such as String, Int, or Boolean.
*   **Introspection:** The ability to query a GraphQL schema for its own types and fields.
*   **Selection Set:** The set of fields requested inside a query or mutation operation.
