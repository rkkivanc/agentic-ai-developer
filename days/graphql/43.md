# Day 43: Authentication & Authorization - Field-Level Authorization

## 📝 Today's Tasks

1.  **Guard Sensitive Fields:** Return null or throw `FORBIDDEN` for `email` when viewer lacks permission.
2.  **Centralize Rules:** Extract `canViewEmail(viewer, user)` helpers used by multiple resolvers.
3.  **Test Matrix:** Table-test owner, admin, stranger roles against protected fields.
4.  **Avoid Leaks via Errors:** Do not reveal existence of hidden records through different error messages.

## 📖 Task Explanations

Field-level authorization is GraphQL's fine-grained strength—hide columns without separate REST endpoints. Implement checks in resolvers or schema directives consistently.

Be careful with error messages that confirm a hidden user exists. Sometimes returning null for unauthorized lookups is safer than forbidden errors.

## 💡 What You'll Gain

By the end of today, you will enforce field-level authorization with reusable permission helpers.

## 📚 Dictionary for Developers

*   **Field-Level Auth:** Allowing or denying access to individual schema fields.
*   **Permission Helper:** Shared function encoding who may see or edit data.
*   **FORBIDDEN:** Common extension code when a caller lacks permission.
*   **Information Leak:** Unintentionally revealing private data through error differences.
*   **Viewer:** The authenticated subject making the current request.
