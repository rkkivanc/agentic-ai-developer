# Day 58: DataLoader & N+1 - Batching, Caching, and Per-Request Scope

## 📝 Today's Tasks

1.  **Enable DataLoader Cache:** Call `load` twice for the same key and observe one batch fetch.
2.  **Clear Between Requests:** Instantiate fresh loaders per HTTP request in context setup.
3.  **Prime the Cache:** Use `prime` to preload values when you already fetched them eagerly.
4.  **Measure Batch Size:** Log how many keys arrive in each batch function invocation.

## 📖 Task Explanations

DataLoader caches within a request after batching—duplicate `load(1)` calls hit memory, not the database. Fresh loaders per request prevent cross-user leakage.

Priming is useful when a parent resolver already loaded children—avoid duplicate fetches by seeding the loader cache.

## 💡 What You'll Gain

By the end of today, you will use DataLoader batching and per-request caching safely and measure batch behavior.

## 📚 Dictionary for Developers

*   **Per-Request Cache:** DataLoader memoization scoped to one GraphQL operation.
*   **prime():** DataLoader method preloading the cache with a key-value pair.
*   **Batch Size:** Number of keys coalesced into one batch function call.
*   **Cache Hit:** Serving a value from DataLoader memory without calling the batch function.
*   **Loader Lifecycle:** Create at request start, discard after response completes.
