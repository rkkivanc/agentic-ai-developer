# Day 68: Testing GraphQL - Mocking Data Sources and Context

## 📝 Today's Tasks

1.  **Stub Repositories:** Replace database modules with in-memory fakes in tests.
2.  **Spy on Calls:** Assert repositories receive expected IDs and filters from resolvers.
3.  **Fixture Data:** Build reusable seed objects representing users and posts.
4.  **Reset Between Tests:** Clear in-memory stores after each test case.

## 📖 Task Explanations

Mocking data sources lets integration and unit tests run without Docker databases. Spies verify resolvers call persistence layers correctly.

Shared fixtures keep tests readable—import `testUser` instead of duplicating object literals in every file.

## 💡 What You'll Gain

By the end of today, you will mock data sources and context for fast, reliable GraphQL tests.

## 📚 Dictionary for Developers

*   **Test Double:** Fake or stub replacing a real data source in tests.
*   **Spy:** Mock recording how it was called for assertions.
*   **Fixture Data:** Reusable sample records shared across tests.
*   **In-Memory Fake:** Simple store simulating database behavior in tests.
*   **Test Reset:** Clearing state between tests to prevent cross-test pollution.
