# Day 70: Testing GraphQL - Practice Project & Review

## 📝 Today's Tasks

1.  **Resolver Unit Tests:** Cover three resolvers with mocked context and data sources.
2.  **Integration Test:** Execute a query and mutation against the test server with fixtures.
3.  **Schema Snapshot:** Save SDL or introspection JSON and fail CI on unexpected diffs.
4.  **Coverage Review:** Identify untested auth and error paths and add at least one test each.

## 📖 Task Explanations

Today bolsters confidence in your MVP before security hardening. Tests are the safety net for refactors during capstone week.

Schema snapshots catch accidental breaking changes when editing SDL quickly.

## 💡 What You'll Gain

By the end of today, you will have unit and integration tests plus schema snapshot protection on your API.

## 📚 Dictionary for Developers

*   **Resolver Unit Test:** Isolated test of one resolver with mocked dependencies.
*   **Schema Snapshot:** Committed baseline compared in CI for unexpected changes.
*   **Test Fixture:** Reusable setup data for predictable integration tests.
*   **Coverage Gap:** Critical code paths lacking automated tests.
*   **CI Failure:** Pipeline stopping merges when tests or snapshots fail.
