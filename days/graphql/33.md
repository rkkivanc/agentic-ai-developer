2.  **Implement `first` and `after`:** Support forward pagination arguments on list root fields.
3.  **Encode Cursors:** Base64-encode offsets or stable row IDs as opaque cursors.
4.  **Populate PageInfo:** Set `hasNextPage` and `endCursor` accurately for clients.

## 📖 Task Explanations

Relay cursor pagination is the most common pattern for large lists in GraphQL. Connections wrap edges and nodes so clients page without skipping items when data changes.

Cursors should be opaque—clients must not depend on internal encoding. Even simple base64 offsets teach the pattern before adopting stable keyset cursors.

## 💡 What You'll Gain

By the end of today, you will implement Relay-style cursor pagination with connections and PageInfo.

## 📚 Dictionary for Developers

*   **Connection:** Relay pattern with `edges` and `pageInfo` for paginated lists.
*   **Edge:** Wrapper containing a `node` and `cursor` for one item.
*   **PageInfo:** Object describing whether more pages exist and cursor boundaries.
*   **Opaque Cursor:** An encoded token clients pass back unchanged for the next page.
*   **first Argument:** Relay argument limiting how many edges to return forward.
