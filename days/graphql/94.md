## 📚 Dictionary for Developers

*   **Schema Review:** Team process for evaluating proposed schema changes before merge.
*   **Breaking Change Policy:** Documented rules for when and how incompatible changes ship.
*   **Observability:** Metrics, logs, and traces making API behavior visible in production.
*   **SLA (Service Level Agreement):** Commitment to availability or latency targets for an API.
*   **Runbook:** Operational guide for handling incidents, deploys, and common failures.
