# Day 27: First GraphQL API (MVP) - Writing Your First Schema and Resolvers

## 📝 Today's Tasks

1.  **Define SDL Types:** Write `Query`, `Mutation`, and domain types for your MVP in `.graphql` files.
2.  **Wire Resolvers:** Map each root field and object field to resolver functions.
3.  **Add Descriptions:** Document types and fields so GraphiQL displays helpful schema docs.
4.  **Smoke Test:** Run one query and one mutation end to end in your local sandbox.

## 📖 Task Explanations

With the server running, today you give it meaning. Start with a narrow schema—users and posts, or products and orders—and implement resolvers that return coherent data shapes.

Descriptions are cheap documentation that pays off immediately in Apollo Sandbox and for future codegen consumers.

## 💡 What You'll Gain

By the end of today, you will have a working schema with resolvers fulfilling queries and mutations locally.

## 📚 Dictionary for Developers

*   **Type Definitions:** SDL files or strings declaring your GraphQL types and roots.
*   **Resolver Wiring:** Connecting schema fields to JavaScript/TypeScript functions.
*   **Smoke Test:** A quick manual check that core operations succeed.
*   **Schema Documentation:** Description strings embedded in SDL for discoverability.
*   **MVP Schema:** The minimal type system supporting your first useful API.
