# Day 42: Authentication & Authorization - JWT and Request Context

## 📝 Today's Tasks

1.  **Parse Authorization Header:** Extract Bearer tokens in middleware before GraphQL execution.
2.  **Verify JWT:** Validate signature and expiry, then attach claims to context.
3.  **Nullable Viewer:** Expose `viewer: User` on Query that is null for anonymous requests.
4.  **Reject Invalid Tokens:** Return `UNAUTHENTICATED` errors for protected fields when tokens fail.

## 📖 Task Explanations

JWT authentication in GraphQL typically happens in HTTP middleware. The verified identity lands in context so resolvers and authorization rules read a trusted `user` object.

Anonymous access is common for public fields. Designing `viewer` as nullable keeps one schema for both signed-in and guest users.

## 💡 What You'll Gain

By the end of today, you will authenticate requests with JWT and expose the current user through context.

## 📚 Dictionary for Developers

*   **Bearer Token:** An access token sent in the `Authorization: Bearer` header.
*   **JWT Claims:** Payload fields such as `sub` or `role` embedded in the token.
*   **Middleware:** HTTP layer running before the GraphQL executor sees the request.
*   **Viewer:** The currently authenticated user exposed as a query field.
*   **Token Verification:** Cryptographic check that a JWT was issued by a trusted authority.
