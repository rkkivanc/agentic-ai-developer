# Day 15: Queries & Selection Sets - Practice

## 📝 Today's Tasks

1.  **Query Cookbook:** Build a personal library of five named queries with variables for a public API.
2.  **Fragment Refactor:** Take one large query and split it into two reusable fragments.
3.  **Directive Drill:** Use `@include` and `@skip` with a boolean variable in a live query.
4.  **Self-Review:** Check each operation is named, uses variables for dynamic values, and requests only needed fields.

## 📖 Task Explanations

Today consolidates query skills from Days 11–14. A query cookbook mirrors how teams store `.graphql` documents alongside application code.

Refactoring with fragments is a daily client-side task. Directives teach you that execution can be conditional without duplicating entire queries.

## 💡 What You'll Gain

By the end of today, you will have a reusable query cookbook demonstrating variables, fragments, aliases, and directives.

## 📚 Dictionary for Developers

*   **Query Cookbook:** A collection of tested, named GraphQL operations for common UI needs.
*   **@include Directive:** Executes a field only when the given Boolean argument is true.
*   **@skip Directive:** Omits a field when the given Boolean argument is true.
*   **Selection Discipline:** Requesting only fields the UI needs to minimize payload size.
*   **Operation Naming:** Labeling queries for logs, analytics, and persisted query registries.
