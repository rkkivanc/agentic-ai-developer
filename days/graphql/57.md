# Day 57: DataLoader & N+1 - DataLoader Fundamentals

## 📝 Today's Tasks

1.  **Install DataLoader:** Add the `dataloader` package to your GraphQL server project.
2.  **Write a Batch Function:** Load many users by ID in one query inside a batch loader.
3.  **Create Per-Request Loaders:** Instantiate loaders in context for each HTTP request.
4.  **Compare Before/After:** Log database queries with and without DataLoader for the same GraphQL query.

## 📖 Task Explanations

DataLoader batches multiple `load(id)` calls into one `batchLoad(ids)` invocation within a single tick of the event loop, then caches results for the rest of the request.

Never share a DataLoader across requests—the cache would leak private data between users.

## 💡 What You'll Gain

By the end of today, you will implement a DataLoader batch function and attach it to per-request context.

## 📚 Dictionary for Developers

*   **DataLoader:** Utility batching and caching key lookups within one request.
*   **Batch Function:** Receives an array of keys and returns values in the same order.
*   **load():** Queues a key for batching and returns a Promise for its value.
*   **Request Scope:** Instantiating loaders fresh for each incoming GraphQL request.
*   **Batch Window:** The brief window in which separate load calls are combined.
