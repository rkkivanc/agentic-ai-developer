# Day 1: GraphQL Fundamentals - What Is GraphQL and Why It Exists

## 📝 Today's Tasks

1.  **Understand GraphQL's Purpose:** Learn why GraphQL was created and what problems it solves compared to traditional REST APIs.
2.  **Compare REST and GraphQL:** Study how REST uses multiple endpoints while GraphQL exposes a single endpoint with a typed schema.
3.  **Explore a Public GraphQL API:** Try a sandbox like the SpaceX or GitHub GraphQL API in GraphiQL or Apollo Sandbox.
4.  **Read the GraphQL Specification Overview:** Skim the official spec introduction to learn core vocabulary: schema, types, and operations.

## 📖 Task Explanations

Today is about laying the groundwork for your GraphQL journey. Before writing a single resolver, you need to understand what GraphQL is and why teams adopt it. GraphQL is a query language for APIs and a runtime for fulfilling those queries with your existing data. Unlike REST, where each endpoint returns a fixed shape, GraphQL lets clients request exactly the fields they need in one round trip.

Your exploration of a public API is a classic first step. Running queries in GraphiQL or Apollo Sandbox confirms that GraphQL is interactive and self-documenting. You will see how a schema describes available data and how a query selects nested fields in a single request.

## 💡 What You'll Gain

By the end of today, you will understand what GraphQL is, how it differs from REST, and how to run your first exploratory queries in a sandbox. This foundation prepares you for schema design and server implementation.

## 📚 Dictionary for Developers

*   **GraphQL:** A query language for APIs and a server-side runtime that executes queries against a typed schema.
*   **Schema:** The contract that defines all types, fields, and operations available in a GraphQL API.
*   **REST (Representational State Transfer):** An architectural style where clients call resource-specific HTTP endpoints with fixed response shapes.
*   **Endpoint:** A URL where clients send requests; GraphQL typically uses one endpoint for all operations.
*   **GraphiQL:** An in-browser IDE for writing, validating, and exploring GraphQL queries against a live schema.
*   **Operation:** A GraphQL request: a query (read), mutation (write), or subscription (realtime stream).
