# Day 78: Tooling, Codegen & DX - GraphQL ESLint and Schema Rules

## 📝 Today's Tasks

1.  **Install graphql-eslint:** Add the plugin and parser for `.graphql` and SDL files.
2.  **Enable Recommended Rules:** Catch unknown fields, deprecated usage, and naming issues.
3.  **Lint in CI:** Fail pulls that introduce invalid operations against the schema.
4.  **Fix Violations:** Resolve lint errors in one query document and one schema file.

## 📖 Task Explanations

GraphQL ESLint validates queries against your schema at edit time and enforces style rules on SDL. It prevents typos in field names from reaching runtime.

Running lint in CI protects main branch quality—especially when multiple clients commit operation documents.

## 💡 What You'll Gain

By the end of today, you will lint GraphQL schema and operation documents with GraphQL ESLint in local and CI workflows.

## 📚 Dictionary for Developers

*   **GraphQL ESLint:** ESLint plugin validating queries and schema files.
*   **Operation Lint:** Static check that query fields exist in the schema.
*   **Schema Rule:** Lint enforcing naming, descriptions, or deprecation policies.
*   **CI Gate:** Pipeline step blocking merges on lint failures.
*   **Unknown Field:** Query field absent from schema caught before deploy.
