# Day 14: Queries & Selection Sets - Aliases, Fragments, and Reuse

## 📝 Today's Tasks

1.  **Use Field Aliases:** Request the same field twice with different arguments using aliases.
2.  **Create Fragments:** Define reusable selection sets on object types and spread them with `...FragmentName`.
3.  **Apply Inline Fragments:** Use `... on TypeName` for interface and union fields.
4.  **Combine Fragments:** Nest spreads to build maintainable query documents for complex UIs.

## 📖 Task Explanations

Aliases let you rename fields in the response, useful when fetching the same field with different arguments. Fragments reduce duplication when multiple components need overlapping data.

Inline fragments are required for abstract types. They tell GraphQL which fields to resolve for each concrete type.

## 💡 What You'll Gain

By the end of today, you will write DRY query documents using aliases, named fragments, and inline fragments.

## 📚 Dictionary for Developers

*   **Argument:** A named parameter on a field, specified in the query and defined in the schema.
*   **Variable:** A dynamic value passed separately from the query document using a `$` prefix.
*   **Fragment:** A reusable selection set that can be spread into multiple operations.
*   **Alias:** A query-time rename of a field in the response using `alias: fieldName` syntax.
*   **Directive:** An annotation such as `@include` or `@skip` that modifies query execution.
*   **Operation Name:** An optional label for a query that aids debugging and server logs.
