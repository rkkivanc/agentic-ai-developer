# Day 11: Queries & Selection Sets - Query Syntax and Root Fields

## 📝 Today's Tasks

1.  **Write Root Queries:** Define entry-point fields on the Query type that return your core object types.
2.  **Practice Selection Sets:** Request multiple scalar and object fields in a single query document.
3.  **Use Nested Selections:** Query related objects in one round trip, e.g., `posts { title author { name } }`.
4.  **Validate Against Schema:** Run queries in a sandbox and fix any unknown field or type errors.

## 📖 Task Explanations

Every GraphQL read operation starts at the Query root type. Clients build a tree of requested fields called a selection set. Nested selections follow object-typed fields deeper into the graph.

Practice writing queries that mirror what a real screen needs. This client-first mindset is central to GraphQL API design.

## 💡 What You'll Gain

By the end of today, you will write valid GraphQL queries with nested selection sets anchored at the Query root.

## 📚 Dictionary for Developers

*   **Argument:** A named parameter on a field, specified in the query and defined in the schema.
*   **Variable:** A dynamic value passed separately from the query document using a `$` prefix.
*   **Fragment:** A reusable selection set that can be spread into multiple operations.
*   **Alias:** A query-time rename of a field in the response using `alias: fieldName` syntax.
*   **Directive:** An annotation such as `@include` or `@skip` that modifies query execution.
*   **Operation Name:** An optional label for a query that aids debugging and server logs.
