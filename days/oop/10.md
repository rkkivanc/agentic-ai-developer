# Day 10: Encapsulation & Abstraction - Practice

## 📝 Today's Tasks

1.  **Harden Your Model:** Take your Day 5 domain and encapsulate all fields.
2.  **Invariant Suite:** Add validation to every mutating operation.
3.  **Abstraction Pass:** Rename/reshape methods so callers read like domain language.
4.  **Before/After Note:** Write 5 lines on what illegal actions are now impossible.

## 📖 Task Explanations

Practice day: turn a naive model into an encapsulated one. The before/after note proves you understand why encapsulation matters—not just how to add `private`.

## 💡 What You'll Gain

An encapsulated domain model with enforced invariants and clearer abstraction.

## 📚 Dictionary for Developers

*   **Hardening:** Strengthening a design against misuse.
*   **Domain Language:** Names that match how experts talk about the problem.
*   **Misuse Resistance:** APIs designed so wrong usage is hard.
