# Day 16: SOLID, Patterns & Practice - SRP and OCP

## 📝 Today's Tasks

1.  **SRP Refactor:** Split a class that has two reasons to change.
2.  **OCP Extension:** Add a new behavior by adding a type/strategy rather than editing a big switch everywhere.
3.  **Before/After:** Capture how many places you had to change in each approach.
4.  **Smell Check:** Identify a real or sample class that violates SRP.

## 📖 Task Explanations

Single Responsibility (SRP) and Open/Closed (OCP) are the practical backbone of SOLID for beginners. Split reasons to change; extend via new types when possible instead of endlessly editing core logic.

## 💡 What You'll Gain

You will apply SRP and OCP on a small example and measure how extension became safer.

## 📚 Dictionary for Developers

*   **SRP:** A class should have one reason to change.
*   **OCP:** Software entities should be open for extension, closed for modification.
*   **Reason to Change:** A stakeholder concern that would force edits to a class.
*   **Code Smell:** A surface indication of a deeper design problem.
