# Day 7: Encapsulation & Abstraction - Invariants and Validation

## 📝 Today's Tasks

1.  **List Invariants:** Write 3 rules your object must always satisfy.
2.  **Preserve on Every Change:** Update methods so no public operation breaks the invariants.
3.  **Fail Fast:** Throw/return errors when an operation would violate a rule.
4.  **Unit Mentality:** Write informal or real tests for the invariant edge cases.

## 📖 Task Explanations

Invariants are the object's promises (balance never negative, email always valid format). Every mutating method must protect them. Fail-fast validation stops corrupt state early.

## 💡 What You'll Gain

You will define invariants and enforce them on every state change with clear failure behavior.

## 📚 Dictionary for Developers

*   **Invariant:** A rule that must remain true for the object's lifetime.
*   **Fail Fast:** Rejecting invalid operations immediately.
*   **Mutator:** A method that changes object state.
*   **Edge Case:** Boundary input that often breaks naive validation.
