# Day 18: SOLID, Patterns & Practice - Useful Patterns (Factory & Strategy)

## 📝 Today's Tasks

1.  **Factory:** Create objects through a factory instead of scattering constructors.
2.  **Strategy:** Extract interchangeable algorithms behind a common interface.
3.  **Replace Conditionals:** Swap a switch/if-chain for strategies where it clarifies design.
4.  **Don't Pattern-Hunt:** Note a case where a plain class is better than a pattern.

## 📖 Task Explanations

Patterns are named solutions—not mandatory decorations. Factory centralizes creation; Strategy swaps algorithms cleanly. Use them when they reduce complexity; skip them when they add ceremony.

## 💡 What You'll Gain

You will implement Factory and Strategy thoughtfully and know when not to force a pattern.

## 📚 Dictionary for Developers

*   **Design Pattern:** A reusable named solution to a common design problem.
*   **Factory Pattern:** Centralizes object creation logic.
*   **Strategy Pattern:** Defines a family of interchangeable algorithms.
*   **Ceremony:** Extra structure that costs more than it helps.
