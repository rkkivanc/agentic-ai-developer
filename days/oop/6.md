# Day 6: Encapsulation & Abstraction - Hiding Internal State

## 📝 Today's Tasks

1.  **Make State Private:** Hide fields; expose behavior through methods.
2.  **Break External Writes:** Show that outside code can no longer corrupt fields directly.
3.  **Controlled Updates:** Provide methods like `rename` or `applyDiscount` that enforce rules.
4.  **Read Access:** Expose safe read-only views or getters where needed.

## 📖 Task Explanations

Encapsulation keeps an object's internals private so invariants cannot be broken from outside. Callers use a deliberate API. This is how large systems stay maintainable: change internals without rewriting all callers.

## 💡 What You'll Gain

You will encapsulate state, prevent illegal external mutation, and expose a controlled public API.

## 📚 Dictionary for Developers

*   **Encapsulation:** Bundling data with methods and restricting direct access to internals.
*   **Information Hiding:** Keeping implementation details private.
*   **Public API:** The members other code is allowed to use.
*   **Getter:** A method that returns state, often without allowing unrestricted writes.
