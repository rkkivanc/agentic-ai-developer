# Day 1: OOP Fundamentals - What Is Object-Oriented Programming

## 📝 Today's Tasks

1.  **Compare Styles:** Write the same tiny feature once procedurally and once with a class; list differences.
2.  **Identify Objects:** Pick a real-world domain (shop, library, game) and list candidate objects and their responsibilities.
3.  **Vocabulary Drill:** Define class, object, attribute, and method in your own words with one example each.
4.  **Language Choice:** Pick one language you know (TypeScript, Dart, Java, Python, C#, etc.) for the rest of this track.

## 📖 Task Explanations

Object-Oriented Programming organizes software around objects—bundles of state and behavior—rather than only free functions and data. Today you build intuition: when grouping related data with methods improves clarity. You will also choose a language to practice in so concepts stay concrete.

## 💡 What You'll Gain

By the end of today, you will explain OOP vs procedural style and identify natural objects in a domain. You will have a language ready for hands-on practice.

## 📚 Dictionary for Developers

*   **OOP:** A programming paradigm that models software as interacting objects with state and behavior.
*   **Object:** A concrete instance that holds data and can perform actions via methods.
*   **Class:** A blueprint that defines the structure and behavior of objects of a type.
*   **Procedural Programming:** A style focused on procedures/functions operating on separate data.
