# Day 13: Inheritance & Polymorphism - Interfaces and Abstract Contracts

## 📝 Today's Tasks

1.  **Define a Contract:** Create an interface or abstract class with required operations.
2.  **Implement Twice:** Write two different concrete classes satisfying the contract.
3.  **Depend on Contract:** Write a function that accepts the contract type, not a concrete class.
4.  **Swap Implementations:** Run the same caller with both implementations.

## 📖 Task Explanations

Interfaces/abstract contracts define what something can do without how. Depending on contracts enables testing and swapping implementations—core professional OOP.

## 💡 What You'll Gain

You will program to an interface/contract and swap implementations without changing callers.

## 📚 Dictionary for Developers

*   **Interface / Protocol:** A set of operations a type must support.
*   **Abstract Class:** A class that may include shared code and abstract methods; typically not instantiated.
*   **Program to an Interface:** Depending on abstractions rather than concrete classes.
*   **Dependency Inversion (idea):** High-level code depending on abstractions.
