# Day 2: OOP Fundamentals - Classes and Objects

## 📝 Today's Tasks

1.  **Define a Class:** Create a simple class (e.g., `Book` or `User`) with a few attributes.
2.  **Create Instances:** Construct at least two different objects from the same class.
3.  **Call Methods:** Add a method that uses the object's own data and invoke it on each instance.
4.  **Inspect State:** Print or log each object's state to see that instances are independent.

## 📖 Task Explanations

A class is a blueprint; objects are the living instances. Creating two books with different titles proves that each object has its own state. Methods act on that state—this is the core mental model for everything that follows.

## 💡 What You'll Gain

You will create classes, instantiate multiple independent objects, and invoke methods that use instance state.

## 📚 Dictionary for Developers

*   **Instance:** A specific object created from a class.
*   **Attribute / Field / Property:** Data stored on an object.
*   **Method:** A function that belongs to a class/object and typically uses its state.
*   **Identity:** The fact that two objects can hold equal values yet remain distinct instances.
