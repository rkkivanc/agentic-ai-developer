# Day 12: Inheritance & Polymorphism - Method Overriding

## 📝 Today's Tasks

1.  **Override a Method:** Change subclass behavior for a base method.
2.  **Call Super:** Optionally extend base behavior by calling the parent implementation.
3.  **Substitution Demo:** Put mixed subclass instances in a list typed as the base type and call the method.
4.  **Document Override:** Comment why the subclass needs different behavior.

## 📖 Task Explanations

Overriding customizes behavior while keeping a shared contract. The substitution demo is polymorphism in action: callers use a base type; the right implementation runs at runtime.

## 💡 What You'll Gain

You will override methods correctly and see polymorphic dispatch through a base type.

## 📚 Dictionary for Developers

*   **Override:** Providing a new implementation of an inherited method.
*   **super / base Call:** Invoking the parent class implementation.
*   **Dynamic Dispatch:** Selecting the method implementation based on the runtime type.
*   **Polymorphism:** Using one interface/base type to work with many concrete types.
