# Day 3: OOP Fundamentals - Constructors and Initialization

## 📝 Today's Tasks

1.  **Write a Constructor:** Require essential fields when an object is created.
2.  **Validate Inputs:** Reject invalid construction (empty name, negative price) with a clear error.
3.  **Default Values:** Provide sensible defaults for optional fields.
4.  **Factory Lite:** Add a named constructor or static factory method for a common creation case.

## 📖 Task Explanations

Constructors establish invariants early: if an object exists, it should be usable. Validation at creation prevents half-built objects from spreading bugs. Named factories improve readability when there are multiple valid ways to create an object.

## 💡 What You'll Gain

You will initialize objects safely with constructors, defaults, and basic validation so objects start in a valid state.

## 📚 Dictionary for Developers

*   **Constructor:** Special method/process that initializes a new object.
*   **Invariant:** A condition that must always be true for an object to be valid.
*   **Factory Method:** A method that creates objects, often with a clearer name than a raw constructor.
*   **Default Value:** A fallback used when the caller omits an optional input.
