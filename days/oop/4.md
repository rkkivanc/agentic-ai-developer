# Day 4: OOP Fundamentals - Behavior and Responsibilities

## 📝 Today's Tasks

1.  **Single Responsibility Sketch:** Split a god object into two classes with clear jobs.
2.  **Tell, Don't Ask:** Replace logic that pulls fields and decides outside with a method on the object.
3.  **Side Effects Note:** Mark which methods change state vs return derived info.
4.  **Collaboration:** Let one object call another (e.g., `Order` uses `Customer`).

## 📖 Task Explanations

Good objects have a clear job. 'Tell, don't ask' means asking an object to do work instead of extracting all its data for external logic. Distinguishing commands (change state) from queries (return info) keeps designs understandable.

## 💡 What You'll Gain

You will assign responsibilities cleanly, prefer behavior on the object that owns the data, and compose collaborating objects.

## 📚 Dictionary for Developers

*   **Responsibility:** The job a class is accountable for.
*   **Tell, Don't Ask:** Prefer calling methods that act over interrogating fields and deciding elsewhere.
*   **Command:** A method whose primary purpose is to change state.
*   **Query:** A method that returns information without changing state (ideally).
