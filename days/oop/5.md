# Day 5: OOP Fundamentals - Practice

## 📝 Today's Tasks

1.  **Mini Domain:** Model a small domain (todo, library loan, or cart item) with 2–3 classes.
2.  **Create Scenarios:** Instantiate objects and run 3–4 realistic operations.
3.  **Enforce Creation Rules:** Ensure invalid objects cannot be constructed.
4.  **Review Checklist:** Confirm each class has a clear responsibility and useful methods—not just public fields.

## 📖 Task Explanations

Today consolidates fundamentals into a tiny working model. Prefer a few strong classes over many empty ones. Practice explaining your design out loud: what each object owns and why.

## 💡 What You'll Gain

A small object model you can demo, with valid construction rules and clear responsibilities.

## 📚 Dictionary for Developers

*   **Domain Model:** Classes that represent real concepts in the problem space.
*   **Demo Scenario:** A short script exercising your objects like a real use case.
*   **Public Field Smell:** Exposing raw fields everywhere instead of intentional methods.
