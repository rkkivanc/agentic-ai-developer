# Day 19: SOLID, Patterns & Practice - Observer and Refactoring Legacy OOP

## 📝 Today's Tasks

1.  **Observer Lite:** Implement a simple subscription/notification flow between objects.
2.  **Legacy Refactor:** Take a messy procedural or god-class sample and extract 2–3 objects.
3.  **Characterization:** List current behaviors before changing so you do not break them.
4.  **Small Steps:** Refactor in tiny commits/steps, verifying after each.

## 📖 Task Explanations

Observer helps when many parts must react to an event. Refactoring legacy code is the real workplace skill: characterize behavior, extract objects, keep steps small. Patterns support refactoring—they are not the goal.

## 💡 What You'll Gain

You will build a simple Observer flow and refactor messy code into clearer objects safely.

## 📚 Dictionary for Developers

*   **Observer Pattern:** Objects subscribe to notifications when another object's state changes.
*   **God Class:** A class that knows/does too much.
*   **Characterization:** Documenting existing behavior before changing code.
*   **Incremental Refactoring:** Improving design in small, verified steps.
